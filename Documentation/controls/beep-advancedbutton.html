BeepAdvancedButton

Multi-style button with loading states, toggle mode, intent semantics, split layout, and FAB support.

Stable v1.0.164 TheTechIdea.Beep.Winform.Controls

Overview

BeepAdvancedButton is a versatile button control that unifies six distinct visual styles — Solid, Icon, Text, Toggle, FAB, and Ghost — in a single component. Each style adapts its rendering automatically while sharing a common API for colors, size, shape, icons, loading state, and semantic intent.

Built on BaseControl, it supports an optional loading spinner overlay that blocks re-clicks during async operations, a toggle mode that tracks pressed/depressed state, a Split shape that fires separate left and right click events, and a focus ring for keyboard accessibility.

Inheritance: Control → BaseControl → BeepAdvancedButton
Default size: 100 × 40 pixels (Medium)
Namespace: TheTechIdea.Beep.Winform.Controls

Key Features

🎨 Six Button Styles

Solid, Icon-only, Text-only, Toggle, Floating Action Button (FAB), and Ghost styles.

⏳ Loading State

Built-in spinner overlay; optionally suppresses clicks while loading to prevent duplicate submissions.

🔀 Toggle Mode

Toggle style tracks pressed/unpressed state and fires ToggleChanged with the new value.

📐 Shape Options

RoundedRectangle, Circle, Pill, and Split shapes — Split fires left/right area click events.

🎯 Semantic Intent

Intent enum (Primary, Secondary, Tertiary, Destructive, Success, Neutral) for consistent theming.

♿ Focus Ring

Configurable focus ring with custom thickness and offset for keyboard-accessible UIs.

Basic Usage

Solid Button (Primary Action)

var saveBtn = new BeepAdvancedButton
{
    Text        = "Save",
    ButtonStyle = AdvancedButtonStyle.Solid,
    ButtonSize  = ButtonSize.Medium,
    Shape       = ButtonShape.RoundedRectangle,
    Intent      = ButtonIntent.Primary,
    ImagePath   = "icons/save.png"  // optional left icon
};

saveBtn.Click += async (s, e) =>
{
    saveBtn.IsLoading = true;
    await SaveDataAsync();
    saveBtn.IsLoading = false;
};

Toggle Button

var boldBtn = new BeepAdvancedButton
{
    Text        = "B",
    ButtonStyle = AdvancedButtonStyle.Toggle,
    Shape       = ButtonShape.RoundedRectangle,
    intent      = ButtonIntent.Secondary
};

boldBtn.ToggleChanged += (s, e) =>
{
    richTextBox1.SelectionFont = new Font(
        richTextBox1.SelectionFont,
        e.IsToggled ? FontStyle.Bold : FontStyle.Regular
    );
};

Floating Action Button (FAB)

var fab = new BeepAdvancedButton
{
    ButtonStyle = AdvancedButtonStyle.FAB,
    Shape       = ButtonShape.Circle,
    ImagePath   = "icons/add.svg",
    Intent      = ButtonIntent.Primary,
    Size        = new Size(56, 56)
};

fab.Click += (s, e) => ShowNewItemDialog();

Split Button (left action + dropdown)

var splitBtn = new BeepAdvancedButton
{
    Text        = "Export",
    ButtonStyle = AdvancedButtonStyle.Solid,
    Shape       = ButtonShape.Split,
    IconRight   = "bi-chevron-down"
};

splitBtn.LeftAreaClicked  += (s, e) => ExportDefault();
splitBtn.RightAreaClicked += (s, e) => ShowExportMenu();

Properties

Property Type Default Description
ButtonStyle AdvancedButtonStyle Solid Visual rendering style: Solid, Icon, Text, Toggle, FAB, Ghost.
ButtonSize ButtonSize Medium Preset size — Small (80×32), Medium (100×40), Large (120×48). Can be overridden by setting Size directly.
Shape ButtonShape RoundedRectangle Outer shape: RoundedRectangle, Circle, Pill, Split.
Text string "" Button label. Not shown in Icon or FAB styles.
ImagePath string "" Path to an image file rendered to the left of the text.
IconLeft string "" Bootstrap Icon class name rendered to the left of the text (e.g., "bi-save").
IconRight string "" Bootstrap Icon class name rendered to the right of the text. Used in Split shape for the dropdown caret.
Intent ButtonIntent Primary Semantic colour intent: Primary, Secondary, Tertiary, Destructive, Success, Neutral.
SolidBackground Color (theme) Explicit fill color for Solid / FAB styles. Overrides theme intent color.
SolidForeground Color (theme) Explicit text/icon color. Overrides theme intent color.
IsToggled bool false Current pressed state for Toggle style buttons. Setting this fires ToggleChanged.
IsLoading bool false When true, shows a spinner overlay. If SuppressClickWhileLoading is true, Click events are blocked.
SuppressClickWhileLoading bool true Prevents Click, LeftAreaClicked, and RightAreaClicked from firing while IsLoading is true.
ShowFocusRing bool true Whether to draw a focus ring when the button has keyboard focus.
FocusRingThickness int 2 Stroke width in pixels of the focus ring.
FocusRingOffset int 3 Gap in pixels between the button edge and the focus ring.
ReduceMotion bool false When true, suppresses hover/press animations for accessibility (respects OS reduce-motion preference).

Inherited properties from BaseControl (Theme, BorderStyle, BorderThickness, etc.) are also available. See the BaseControl API reference.

Events

Event Signature Description
Click EventHandler Standard WinForms click. Not fired while IsLoading if SuppressClickWhileLoading is true.
ToggleChanged EventHandler<ToggleChangedEventArgs> Fired when IsToggled changes. ToggleChangedEventArgs.IsToggled holds the new state.
LeftAreaClicked EventHandler Fired when the left region of a Split shape button is clicked.
RightAreaClicked EventHandler Fired when the right region of a Split shape button is clicked (typically the dropdown arrow).

Button Style Reference

Style Typical Use Notes
Solid Primary/default action Filled background, contrasting text. Responds to Intent color.
Ghost Secondary action, less visual weight Transparent fill, colored border and text.
Text Tertiary action, link-like buttons No background or border; text only, underline on hover.
Icon Toolbar, compact action buttons Square aspect ratio, icon centered; text is hidden.
Toggle Formatting (Bold/Italic), feature on/off Tracks IsToggled state with visual depressed feedback.
FAB Floating action button (Material Design) Circle shape with drop shadow; usually docked bottom-right.

Code Examples

Async Operation with Loading Guard

btn.Click += async (s, e) =>
{
    btn.IsLoading = true;
    try
    {
        await UploadFileAsync(selectedPath);
        ShowSuccess("File uploaded successfully.");
    }
    catch (Exception ex)
    {
        ShowError(ex.Message);
    }
    finally
    {
        btn.IsLoading = false;
    }
};

Intent-Driven Delete Button

var deleteBtn = new BeepAdvancedButton
{
    Text        = "Delete",
    ButtonStyle = AdvancedButtonStyle.Solid,
    Shape       = ButtonShape.RoundedRectangle,
    Intent      = ButtonIntent.Destructive,
    IconLeft    = "bi-trash"
};

deleteBtn.Click += (s, e) =>
{
    if (ConfirmDelete())
        DeleteSelectedRecord();
};

Toolbar Icon Row

string[] icons = { "bi-bold", "bi-italic", "bi-underline" };
foreach (var icon in icons)
{
    var btn = new BeepAdvancedButton
    {
        ButtonStyle = AdvancedButtonStyle.Toggle,
        Shape       = ButtonShape.RoundedRectangle,
        IconLeft    = icon,
        ButtonSize  = ButtonSize.Small,
        Intent      = ButtonIntent.Secondary
    };
    btn.ToggleChanged += (s, e) => ApplyFormatting(icon, e.IsToggled);
    toolbarPanel.Controls.Add(btn);
}

Theming

BeepAdvancedButton maps its Intent to theme color tokens automatically. For example, ButtonIntent.Primary uses the theme's primary accent color, while ButtonIntent.Destructive uses the theme's error/danger color. 

// Apply a theme — Intent colors update automatically
BeepThemesManager.ApplyTheme(btn, "Ocean");

// Or explicit override (ignores Intent-based theming)
btn.SolidBackground = Color.FromArgb(0, 120, 215);
btn.SolidForeground = Color.White;

Best Practices

  • One primary action per area — Use Intent.Primary for the single most important action; use Secondary/Ghost for supporting actions.
  • Always guard async operations — Set IsLoading = true before awaiting and false in a finally block to prevent double-submission.
  • Use Split sparingly — The Split shape is powerful but unexpected; reserve it for actions where there is a natural primary + options pattern (e.g., Export + format picker).
  • Destructive intent requires confirmation — Always show a confirmation dialog before executing irreversible actions triggered by a ButtonIntent.Destructive button.
  • Icon-only buttons need tooltips — When using Icon style without text, set the inherited ToolTip property so screen readers and mouse-hover users understand the action.
  • ReduceMotion for accessibility — Check SystemInformation.HighContrast or expose a setting in your app and set ReduceMotion = true accordingly.
