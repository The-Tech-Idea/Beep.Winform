Filtering System

A complete, decoupled filter architecture: BeepFilter builds the filter UI and emits a FilterConfiguration event; consuming controls (BeepGridPro, BeepListBox) apply it through the generic FilterEngine<T>. Eight interaction styles, 17 operators, drag-drop, validation, smart suggestions, and full keyboard support.

Architecture Overview

The filtering system is split into two clearly separated responsibilities:

LayerClassResponsibility
UI BuilderBeepFilterRenders filter UI, accepts user input, emits FilterConfiguration via events — never touches data.
Data ApplierFilterEngine<T>Evaluates a FilterConfiguration against an IEnumerable<T> — no UI involved.
ConsumerBeepGridPro / BeepListBoxSubscribes to FilterApplied, calls FilterEngine<T>, updates visible rows/items.
Architecture Diagram
+-----------------------------+
|         BeepFilter          |  UI only: takes EntityStructure,
|       (UI Builder)          |  shows filter widgets (TagPills, etc.)
|                             |  Raises FilterApplied event -----+
+-----------------------------+                                  |
                                                                 | FilterConfiguration
                                              +------------------v-----------+
                                              |    BeepGridPro / BeepListBox |
                                              |  (Consumer)                  |
                                              |  Uses FilterEngine<T>        |
                                              +------------------------------+
Key Design Principle: BeepFilter never sees your data. It generates a FilterConfiguration and fires an event. The consuming control decides how and when to apply it. This makes BeepFilter reusable with any data source.

Filter Styles

FilterStyle controls the interaction pattern — how the user builds criteria. Visual colors, borders, and spacing always come from BeepStyling / BeepControlStyle, not from the style enum.

ValuePainter ClassVisual PatternBest For
TagPills TagPillsFilterPainter Horizontal row of rounded pill chips with × remove button; drag handles for reordering. Simple, at-a-glance active filters bar.
GroupedRows GroupedRowsFilterPainter One row per criterion with AND/OR connectors; Add Filter / Add Group buttons; indented subgroups. Multi-condition structured queries.
QueryBuilder QueryBuilderFilterPainter Visual logic tree with If/Else branches; Prompt vs Logic toggle. Complex nested conditions and business rules.
DropdownMultiSelect DropdownMultiSelectFilterPainter Field / Operator / Value dropdowns; multi-select checkboxes; active count badge. User-friendly quick filtering with fixed column sets.
InlineRow InlineRowFilterPainter Single compact row: Column | Operator | Value. Minimal spacing. Space-constrained panels; table header filter rows.
SidebarPanel SidebarPanelFilterPainter Vertical list of collapsible filter categories with checkboxes/toggles. Apply/Clear at bottom. E-commerce / catalog browsing style.
QuickSearch QuickSearchFilterPainter Single search bar with column selector. Real-time filtering. Filter count badge. Fast free-text search across all columns.
AdvancedDialog AdvancedDialogFilterPainter Modal or slide-in panel with tabs; Save/Load config; result count preview. Power users; reusable named filter sets.

Switching Styles at Runtime

C# — runtime style switch
// Painter is hot-swapped; existing ActiveFilter is preserved
beepFilter.FilterStyle = FilterStyle.GroupedRows;

// Get description from factory
string desc = FilterPainterFactory.GetStyleDescription(FilterStyle.QuickSearch);
// "Single search bar with instant filtering - quick and easy"

bool ready = FilterPainterFactory.IsFullyImplemented(FilterStyle.AdvancedDialog); // true

Data Models

FilterCriteria

One condition in the filter: ColumnName Operator Value.

PropertyTypeDescription
ColumnNamestringColumn / property to filter on.
OperatorFilterOperatorComparison operator (one of 17).
ValueobjectPrimary comparison value.
Value2objectUpper bound for Between / NotBetween.
CaseSensitiveboolCase-sensitive text comparison. Default false.
IsEnabledboolSkip this criterion when false without removing it. Default true.
C# — creating criteria
// Simple
var c1 = new FilterCriteria("Country", FilterOperator.Equals, "Germany");

// Range
var c2 = new FilterCriteria("Price", FilterOperator.Between, 10.0, 50.0);

// Case-sensitive text
var c3 = new FilterCriteria("Code", FilterOperator.StartsWith, "ABC", caseSensitive: true);

// Disabled (remembered but not evaluated)
c3.IsEnabled = false;

FilterConfiguration

A named, serialisable collection of FilterCriteria with AND / OR logic.

PropertyTypeDescription
NamestringHuman-readable name for saving / loading.
DescriptionstringOptional description.
CriteriaList<FilterCriteria>Ordered list of conditions.
LogicFilterLogicAnd (all must match) or Or (any match). Default And.
IsActiveboolWhether this configuration is currently applied.
CreatedDateDateTimeAuto-set on construction.
ModifiedDateDateTimeShould be updated on every change.
C# — building a configuration
var config = new FilterConfiguration("Q1 EU Customers")
{
    Logic = FilterLogic.And
};
config.AddCriteria(new FilterCriteria("Region",  FilterOperator.Equals,      "EU"));
config.AddCriteria(new FilterCriteria("Revenue", FilterOperator.GreaterThan, 50000));

// Serialize / deserialize (System.Text.Json compatible)
string json   = JsonSerializer.Serialize(config);
var    loaded = JsonSerializer.Deserialize<FilterConfiguration>(json);

FilterLogic.AndAll enabled criteria must match (default — most restrictive).

FilterEngine<T>

Generic, reflection-based engine that evaluates a FilterConfiguration against any IEnumerable<T>. Property info is cached on construction for performance.

Core Methods

MethodReturnsDescription
ApplyFilter(FilterConfiguration config) IEnumerable<T> Filters data source using all enabled criteria. Returns full source if config is null or has no enabled criteria.
ApplyQuickFilter(string text, string? prop) IEnumerable<T> Fast substring search. Pass null for prop to search all properties.
ApplyFilterGetIndices(FilterConfiguration config) List<int> Returns 0-based indices of matched items. Useful for grid row hiding without rebuilding collections.
C# — applying a filter in a consumer
// Works with any typed collection
var engine = new FilterEngine<Customer>(customers);

// Works with ExpandoObject (BeepGridPro uses this internally)
var engine = new FilterEngine<ExpandoObject>(gridRows);

// Option A: get filtered rows
var visible = engine.ApplyFilter(config).ToList();

// Option B: get matching indices (faster for large virtual grids)
var indices = engine.ApplyFilterGetIndices(config);
HideRowsNotIn(indices);

// Quick search across all properties
var results = engine.ApplyQuickFilter("Smith", propertyName: null);

Helper Components

FilterSuggestionProvider

Analyses the data source and provides smart value suggestions: recent values, most frequent values, and partial-text matches. Results are cached for 5 minutes.

C# — suggestions
var provider = new FilterSuggestionProvider();
provider.SetDataSource(myList);  // invalidates cache

var suggestions = provider.GetSuggestions(
    columnName: "Country", partialValue: "Ge", maxResults: 10);
// Returns: Recent (star), Frequent (fire), Partial-match
// Each item: Value, DisplayText, MatchCount, Relevance, Type

FilterValidationHelper

Validates operator/type compatibility and value requirements before criteria are committed.

C# — validation
var helper = new FilterValidationHelper();
var result = helper.Validate(criteria, DbFieldCategory.Numeric);
if (!result.IsValid)
    ShowError(result.ErrorMessage);
// Guards: null column, operator/type mismatch,
//         missing Value2 for Between, type incompatibility

FilterKeyboardHandler

Attached to the BeepFilter control; provides keyboard shortcuts for adding, clearing, and navigating criteria.

FilterAutocompletePopup

Popup created on demand when a value input is focused. Shows FilterSuggestion items. Dismissed on Escape or selection.

FilterIconProvider

Returns icon glyphs (Bootstrap Icons / strings) for operator symbols and filter actions, keeping painter code free of icon literals.

FilterPainterMetrics / IFilterPainterMetricsProvider

Provides computed layout metrics (heights, widths, paddings) derived from BeepStyling so all painters share a single source of truth for spacing.

Usage Patterns

Pattern 1: Standalone BeepFilter

C# — standalone, apply to any data source
var filter = new BeepFilter
{
    EntityStructure = myEntityStructure,
    FilterStyle     = FilterStyle.TagPills,
    DataSource      = myList,         // optional: enables suggestions
    Dock            = DockStyle.Top
};

filter.FilterApplied += (s, e) =>
{
    var engine  = new FilterEngine<Product>(allProducts);
    var results = engine.ApplyFilter(filter.ActiveFilter);
    listBox1.DataSource = results.ToList();
};

panel.Controls.Add(filter);

Pattern 2: Wired to BeepGridPro

C# — filter drives a grid
var beepFilter = new BeepFilter
{
    EntityStructure = grid.EntityStructure,
    FilterStyle     = FilterStyle.GroupedRows,
    Dock            = DockStyle.Top
};

beepFilter.FilterApplied += (s, e) =>
{
    // Grid's ActiveFilter setter calls FilterEngine<ExpandoObject> internally
    grid.ActiveFilter = beepFilter.ActiveFilter;
};

// Or use the grid's built-in integration:
grid.ShowAdvancedFilterDialog();   // opens AdvancedDialog style
grid.ApplyQuickFilter("Smith");    // internal QuickSearch

Pattern 3: Programmatic Filter (no UI)

C# — direct FilterEngine usage
var config = new FilterConfiguration();
config.AddCriteria(new FilterCriteria("Status", FilterOperator.Equals,            "Active"));
config.AddCriteria(new FilterCriteria("Score",  FilterOperator.GreaterThanOrEqual, 80));
config.Logic = FilterLogic.And;

var engine  = new FilterEngine<Employee>(employees);
var visible = engine.ApplyFilter(config).ToList();

// Get just the indices (faster for large virtual lists)
var indices = engine.ApplyFilterGetIndices(config);

Pattern 4: Save / Load Named Configurations

C# — persist filter presets
// Save
beepFilter.ConfigurationSaveRequested += (s, e) =>
{
    var config = beepFilter.ActiveFilter;
    config.Name = "EU High-Value Q1";
    File.WriteAllText("filters/eu-q1.json", JsonSerializer.Serialize(config));
};

// Load
beepFilter.ConfigurationLoadRequested += (s, e) =>
{
    var json   = File.ReadAllText("filters/eu-q1.json");
    var config = JsonSerializer.Deserialize<FilterConfiguration>(json)!;
    beepFilter.ActiveFilter = config;
    beepFilter.Invalidate();
};

Best Practices

  1. Never apply filters inside BeepFilter. Always subscribe to FilterApplied and use FilterEngine<T> in the consuming control.
  2. Disable criteria instead of removing them. Set IsEnabled = false to temporarily skip without losing the configuration.
  3. Use ApplyFilterGetIndices() for large grids. Returning indices avoids creating a new collection and maps directly to row-hiding in virtual grids.
  4. Set EntityStructure before FilterStyle. Painters read column definitions during CalculateLayout(); setting style first may paint an empty field list.
  5. Provide DataSource for smart suggestions. FilterSuggestionProvider can only generate recent/frequent hints when BeepFilter.DataSource is set.
  6. Call RecalculateLayout() + Invalidate() after programmatic changes. Internal layout recalculates only on user interaction and property changes, not on direct ActiveFilter.Criteria.Add() calls.
  7. Serialize FilterConfiguration with System.Text.Json. All properties are plain types — no custom converters needed for basic save/load.
  8. Avoid BeepQueryandFilter in new code. It does not integrate with the theming system, painter architecture, or FilterEngine<T> pipeline.
