Styling System

The complete painting architecture for Beep controls. A three-layer system: BeepThemesManager (color palette) → BeepStyling (rendering coordinator) → 170+ specialized painters. BaseControl and BeepiFormPro each own a slice of this pipeline.

Stable v1.0.164 TheTechIdea.Beep.Winform.Controls.Styling

System Architecture

Three independent layers cooperate to paint everything on screen:

LayerClass / NamespaceResponsibility
1 — Theme Layer BeepThemesManager
ThemeManagement/		 Holds the active IBeepTheme (color palette, typography, spacing tokens). Fires ThemeChanged / DpiChanged events. All controls subscribe and call ApplyTheme() in response.
2 — Style Coordinator BeepStyling
Styling/BeepStyling.cs		 Central static class. Maps FormStyle ↔ BeepControlStyle, routes every paint call to the correct specialist painter, and exposes helper shortcuts (GetRadius, GetBorderColor, …). Does not draw anything itself.
3 — Painter Subsystems 7 painter folders
170+ classes		 Each painter class owns one operation (background, border, text, shadow, …) for one design system. Painters call the 5 helper systems for style-specific values (colors, spacing, borders, shadows, typography).

How BaseControl fits in

BaseControl is the root for every Beep UI control. Its painting pipeline:

  1. OnPaint → DrawContent(g)
  2. EnsurePainter() — selects an IBaseControlPainter based on PainterKind
  3. Painter calls BeepStyling.PaintStyleBackground / PaintStyleBorder / PaintStyleText using the control's current BeepControlStyle
  4. Derived control overrides DrawContent to add specific content inside DrawingRect

How BeepiFormPro fits in

BeepiFormPro is the themed borderless form host. Its pipeline is parallel but separate from the control pipeline:

  1. FormStyle property changes → ApplyFormStyle()
  2. BeepStyling.GetControlStyle(FormStyle) → syncs BeepStyling.CurrentControlStyle so controls inside the form automatically pick up the same style
  3. PaintersFactory.GetPainter(FormStyle) → returns a cached IFormPainter
  4. IFormPainter.CalculateLayoutAndHitAreas() positions caption bar, buttons, title; IFormPainter.PaintBackground / PaintCaptionBar / PaintBorder render each layer
Key relationship: Setting BeepiFormPro.FormStyle calls BeepStyling.ApplyFormStyle() which sets BeepStyling.CurrentControlStyle. Every control inside that form reads BeepStyling.CurrentControlStyle during paint, so the form chrome and all controls always share the same design system.

Render Sequence

protected override void OnPaint(PaintEventArgs e)
{
    if (!IsReadyForDrawing) return;        // 1. Guard
    if (AllowBaseControlClear) e.Graphics.Clear(BackColor);

    DrawContent(e.Graphics);              // 2. Entry
}

protected virtual void DrawContent(Graphics g)
{
    EnsurePainter();                      // 3. Select painter
    _painter?.UpdateLayout(this);         // 4. Compute rects
    _painter?.Paint(g, this);             // 5. Render outer chrome
    _painter?.UpdateHitAreas(this);       // 6. Register hit zones
    // DrawingRect is now ready — derived control paints content here
}

Border Painter Style (independent)

Separate from PainterKind. The BorderPainterStyle property selects a BeepControlStyle-specific border painter that is applied after the main painter renders the background:

// In a custom control constructor
PainterKind         = BaseControlPainterKind.Classic;   // outer chrome
BorderPainterStyle  = BeepControlStyle.Fluent2;          // border look

UpdateBorderPainter() calls BorderPainterFactory.CreatePainter(style) and also syncs BorderRadius, BorderThickness, and BorderColor from the style defaults.

Theme Subscription Pattern

// BaseControl automatically subscribes in its constructor:
BeepThemesManager.ThemeChanged += (s, e) => {
    _currentTheme = e.NewTheme;
    ApplyTheme();
    Invalidate();
};

// ApplyTheme() copies theme tokens to control properties:
public virtual void ApplyTheme()
{
    ForeColor    = _currentTheme.ForeColor;
    BorderColor  = _currentTheme.BorderColor;
    BackColor    = IsChild ? Parent.BackColor : _currentTheme.BackColor;
    _textFont    = BeepThemesManager.ToFont(_currentTheme.BodyMedium);
    // ...
}

Creating a Custom Control

public class MyBeepLabel : BaseControl
{
    public MyBeepLabel()
    {
        PainterKind        = BaseControlPainterKind.Classic;
        BorderPainterStyle = BeepControlStyle.Minimal;
        IsChild            = true;  // transparent background
    }

    protected override void DrawContent(Graphics g)
    {
        base.DrawContent(g);   // paints chrome into DrawingRect

        // DrawingRect is the inner content area after padding/border
        using var sf = new StringFormat { Alignment = StringAlignment.Center,
                                          LineAlignment = StringAlignment.Center };
        g.DrawString(Text, _textFont, new SolidBrush(ForeColor), DrawingRect, sf);
    }
}

IFormPainter Contract

public interface IFormPainter
{
    void CalculateLayoutAndHitAreas(BeepiFormPro owner);
    void PaintBackground(Graphics g, BeepiFormPro owner);
    void PaintCaptionBar(Graphics g, BeepiFormPro owner);
    void PaintBorder(Graphics g, BeepiFormPro owner);
}

Each FormStyle has exactly one painter class (e.g., DraculaFormPainter, MacOSFormPainter). The painter owns everything: corner radius, caption height, window button positions, and visual effects for that style.

Layout Caching

To avoid recalculating hit areas on every paint, BeepiFormPro.Core.cs tracks a dirty flag. Layout only recalculates when:

  • Form size changes
  • FormStyle changes
  • Caption/button visibility changes
  • InvalidateLayout() is called explicitly

Usage Example

protected override void DrawContent(Graphics g)
{
    base.DrawContent(g);

    // Paint avatar image clipped to a circle
    float cx = DrawingRect.X + DrawingRect.Width  / 2f;
    float cy = DrawingRect.Y + DrawingRect.Height / 2f;
    float r  = Math.Min(DrawingRect.Width, DrawingRect.Height) / 2f - 2f;

    StyledImagePainter.PaintInCircle(g, cx, cy, r, ImagePath);

    // Or paint with a blue tint at 80% opacity:
    StyledImagePainter.PaintWithTint(g, _clipPath, ImagePath,
        Color.FromArgb(100, 0, 120, 215), opacity: 0.8f, cornerRadius: 8);
}

Best Practices

  • Never copy-paste style values into a control. Always ask StyleColors, StyleBorders, etc. so the control automatically stays correct when the design system evolves.
  • Let BeepStyling route painter calls; don't call individual painters directly from controls. The routing switch in BeepStyling.PaintStyleBackground() is the single maintenance point for adding new styles.
  • Always set BeepStyling.CurrentTheme whenever BeepThemesManager.CurrentTheme changes. BeepiFormPro.ApplyTheme() does this automatically; your custom root forms must do the same.
  • Use PainterKind = BaseControlPainterKind.None only when the control is doing fully custom rendering. Anything set to None bypasses the shadow, border, and background system entirely.
  • Keep painters stateless. All context (style, theme, focus state) must come in as method parameters. Painters must not cache mutable state between calls.
  • Use StyledImagePainter for every image draw operation. It handles icon loading, SVG fallback paths, DPI-aware sizing, and the shared cache. Do not call g.DrawImage() with an unconditional Image.FromFile() inside OnPaint.
  • Call InvalidateLayout() after property changes that affect size. For BeepiFormPro, this clears the cached border path and hit areas so the painter recalculates on the next paint cycle.
  • Implement IFormPainter for custom form skins. Register it in PaintersFactory.CreatePainter() under a new FormStyle value. That is the only change required system-wide.
