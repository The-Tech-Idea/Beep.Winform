Performance Guide

Optimize your Beep Controls applications for speed and responsiveness

Reduce Control Count

Each control on a form adds overhead. For complex layouts:

  • Use BeepSimpleGrid instead of many individual labels/textboxes for tabular data
  • Use BeepListBox or BeepTree for large lists instead of dynamically created controls
  • Use virtualization when displaying hundreds of items

Suspend Layout During Bulk Updates

// Before adding many controls
panel.SuspendLayout();

for (int i = 0; i < 100; i++)
{
    var label = new BeepLabel { Text = $"Item {i}" };
    panel.Controls.Add(label);
}

// Resume after all additions
panel.ResumeLayout(true);

Use Double Buffering

BaseControl enables double buffering by default. If you create custom controls, ensure it's enabled:

public class MyControl : BaseControl
{
    public MyControl()
    {
        SetStyle(ControlStyles.OptimizedDoubleBuffer |
                 ControlStyles.AllPaintingInWmPaint |
                 ControlStyles.UserPaint, true);
    }
}

Cache GDI+ Resources

Creating brushes and pens on every paint call is expensive. Cache them:

// Bad — creates new brush every paint
protected override void OnPaint(PaintEventArgs e)
{
    using var brush = new SolidBrush(CurrentTheme.BackColor);
    e.Graphics.FillRectangle(brush, ClientRectangle);
}

// Good — cached brush, recreated only on theme change
private SolidBrush _bgBrush;

protected override void OnThemeChanged()
{
    base.OnThemeChanged();
    _bgBrush?.Dispose();
    _bgBrush = new SolidBrush(CurrentTheme.BackColor);
}

protected override void OnPaint(PaintEventArgs e)
{
    e.Graphics.FillRectangle(_bgBrush, ClientRectangle);
}

Async Data Loading

Never load data on the UI thread. Use async patterns:

private async void LoadData()
{
    var wait = new BeepWait();
    wait.Show("Loading data...");

    var data = await Task.Run(() => FetchDataFromDatabase());

    grid.DataSource = data;
    wait.Close();
}

Minimize Theme Switching

Switching themes triggers a repaint cascade across all controls. Avoid switching themes rapidly (e.g., in a loop or timer). Switch themes only in response to deliberate user action.

Grid Performance Tips

  • Set VirtualMode = true for grids with >1000 rows
  • Avoid complex cell formatting in the paint handler
  • Use pagination via BeepDataNavigator instead of loading all rows
  • Disable auto-sizing columns for very wide data sets
