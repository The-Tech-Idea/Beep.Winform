BeepChartWidget

Chart and data-visualization widget with seven styles: bar, line, pie/donut, gauge, sparkline, heatmap, and combination charts — all rendered in pure GDI+.

Stable v1.0.164 Beep Widgets / Data / Charts

Overview

BeepChartWidget is a paint-based WinForms widget that inherits from BaseControl. It uses an IWidgetPainter pipeline to render rich visuals without child controls. The Style property selects one of 7 built-in render modes; changing it at runtime immediately swaps the painter and triggers a repaint.

Key Features

📊 Seven Chart Types

Bar, line, pie, gauge, sparkline, heatmap, combination — all via one Style enum.

🎨 Theme Palette

Colors defaults to the active theme's ChartDefaultSeriesColors list.

🖱️ Hit-test Events

DataPointClicked carries the clicked series index for drill-down navigation.

📐 Gridlines & Legend

ShowGrid and ShowLegend toggles with no extra configuration needed.

⚡ Pure GDI+

No third-party charting library — lightweight and always theme-consistent.

🔢 Axis Control

MinValue / MaxValue let you fix axis extents for meaningful comparisons.

Basic Usage

var chart = new BeepChartWidget
{
    Style      = ChartWidgetStyle.LineChart,
    Title      = "Weekly Sales",
    Values     = new List { 120, 135, 98, 160, 175, 210, 195 },
    Labels     = new List { "Mon","Tue","Wed","Thu","Fri","Sat","Sun" },
    ShowLegend = false,
    ShowGrid   = true,
    Size       = new Size(360, 220)
};
this.Controls.Add(chart);

Event Handling

chart.DataPointClicked += (s, e) =>
{
    int idx = (int)e.Data;  // index into Values/Labels
    MessageBox.Show($"Clicked: {chart.Labels[idx]} = {chart.Values[idx]}");
};

Best Practices

  • Set Style before adding the widget to the form to avoid double repaints.
  • Adjust Size to fit your layout — defaults are minimum comfortable sizes.
  • Read BeepEventDataArgs.Data in event handlers for context-specific click data.
  • Call Invalidate() after updating data properties to trigger a repaint.
  • Let theme colors flow from BeepStyling.CurrentTheme — avoid overriding color properties unless customization is explicitly desired.
