BeepControlWidget

Interactive input-control widget with ten styles: toggle switches, sliders, dropdowns, date pickers, search boxes, button groups, checkbox groups, range selectors, colour pickers, and number spinners.

Stable v1.0.164 Beep Widgets / Input Controls

Overview

BeepControlWidget is a paint-based WinForms widget that inherits from BaseControl. It uses an IWidgetPainter pipeline to render rich visuals without child controls. The Style property selects one of 10 built-in render modes; changing it at runtime immediately swaps the painter and triggers a repaint.

Key Features

🎛️ Ten Input Styles

From a simple toggle to a full range-selector — one control, ten behaviours.

✅ Strongly-Typed Values

ToggleValue, SliderValue, RangeValue, DateValue, NumberValue models carry all state.

🔔 ValueChanged Event

Universal ValueChanged event fires regardless of style; Data carries new value.

🎨 Theme Integration

Button, checkbox, and input colours all default to active theme tokens.

🚫 IsEnabled

Disabled state can be toggled at runtime; painters render the greyed-out style.

🖱️ OptionSelected

Dropdown and ButtonGroup fire OptionSelected with the chosen option string.

Basic Usage

// Toggle switch
var toggle = new BeepControlWidget
{
    Style      = ControlWidgetStyle.ToggleSwitch,
    Title      = "Dark Mode",
    ToggleValue = { IsOn = false },
    Size       = new Size(200, 60)
};

// Dropdown filter
var dropdown = new BeepControlWidget
{
    Style          = ControlWidgetStyle.DropdownFilter,
    Title          = "Filter by Status",
    Options        = new List { "All", "Active", "Inactive", "Pending" },
    SelectedOption = "All",
    Size           = new Size(220, 70)
};
this.Controls.Add(toggle);
this.Controls.Add(dropdown);

Event Handling

toggle.ValueChanged += (s, e) =>
{
    bool isOn = (bool)e.Data;
    ApplyDarkMode(isOn);
};

dropdown.OptionSelected += (s, e) =>
{
    string selected = (string)e.Data;
    FilterGrid(selected);
};

Best Practices

  • Set Style before adding the widget to the form to avoid double repaints.
  • Adjust Size to fit your layout — defaults are minimum comfortable sizes.
  • Read BeepEventDataArgs.Data in event handlers for context-specific click data.
  • Call Invalidate() after updating data properties to trigger a repaint.
  • Let theme colors flow from BeepStyling.CurrentTheme — avoid overriding color properties unless customization is explicitly desired.
