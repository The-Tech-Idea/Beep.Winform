BeepFormWidget

Form layout widget with ten styles: field groups, validation panels, form sections, input cards, multi-step form steps, fieldsets, inline/compact layouts, validated inputs, and form summaries.

Stable v1.0.164 Beep Widgets / Forms

Overview

BeepFormWidget is a paint-based WinForms widget that inherits from BaseControl. It uses an IWidgetPainter pipeline to render rich visuals without child controls. The Style property selects one of 10 built-in render modes; changing it at runtime immediately swaps the painter and triggers a repaint.

From compact inline to multi-step wizard — every form layout covered.

📋 FormSummary

Read-only summary style for review-before-submit screens.

🎨 Theme tokens

ValidColor, InvalidColor, and AccentColor default to active theme tokens.

⭐ Required Indicators

ShowRequired renders asterisks on required fields automatically.

Basic Usage

var form = new BeepFormWidget
{
    Style    = FormWidgetStyle.ValidationPanel,
    Title    = "User Registration",
    Fields   = new List
    {
        new FormField { Label = "First Name", Value = "", IsRequired = true  },
        new FormField { Label = "Last Name",  Value = "", IsRequired = true  },
        new FormField { Label = "Email",      Value = "", IsRequired = true,
                        IsValid = false, ErrorMessage = "Enter a valid email" },
        new FormField { Label = "Phone",      Value = "", IsRequired = false },
    },
    Size = new Size(380, 280)
};
this.Controls.Add(form);

Event Handling

form.FieldClicked += (s, e) =>
{
    var field = (FormField)e.Data;
    FocusNativeInput(field.Label);
};

form.SubmitClicked += (s, e) => ValidateAndSubmit();

Best Practices

  • Set Style before adding the widget to the form to avoid double repaints.
  • Adjust Size to fit your layout — defaults are minimum comfortable sizes.
  • Read BeepEventDataArgs.Data in event handlers for context-specific click data.
  • Call Invalidate() after updating data properties to trigger a repaint.
  • Let theme colors flow from BeepStyling.CurrentTheme — avoid overriding color properties unless customization is explicitly desired.
