BeepMapWidget

Location and mapping widget with ten styles: live tracking, route display, location cards, geographic heatmaps, address cards, map previews, location lists, travel cards, region maps, and place cards.

Stable v1.0.164 Beep Widgets / Maps / Location

Overview

BeepMapWidget is a paint-based WinForms widget that inherits from BaseControl. It uses an IWidgetPainter pipeline to render rich visuals without child controls. The Style property selects one of 10 built-in render modes; changing it at runtime immediately swaps the painter and triggers a repaint.

Key Features

🗺️ Ten Location Styles

From a simple address card to a geographic heatmap — one widget.

📍 Static Map Preview

MapPreview renders a styled placeholder map background with a pin marker.

🌡️ Geographic Heatmap

GeographicHeatmap colours a grid by value intensity.

🛣️ Route Display

RouteDisplay renders an origin→destination path with waypoints.

📋 Location List

LocationList shows multiple locations with distance and direction badges.

🏨 Place Cards

PlaceCard renders venue name, category, and star rating.

Basic Usage

var location = new BeepMapWidget
{
    Style    = MapWidgetStyle.LocationCard,
    Title    = "Our Office",
    Address  = "123 Tech Avenue",
    City     = "San Francisco",
    Country  = "USA",
    Latitude  = 37.7749,
    Longitude = -122.4194,
    Size     = new Size(300, 160)
};

var route = new BeepMapWidget
{
    Style     = MapWidgetStyle.TravelCard,
    Title     = "Delivery Route",
    Locations = new List
    {
        new LocationItem { Name = "Warehouse",    City = "Oakland"       },
        new LocationItem { Name = "Distribution", City = "San Francisco" },
        new LocationItem { Name = "Customer",     City = "Palo Alto"     },
    },
    Size = new Size(340, 200)
};

Event Handling

location.ActionClicked += (s, e) => OpenMapsApp(location.Latitude, location.Longitude);
route.LocationClicked  += (s, e) =>
{
    var loc = (LocationItem)e.Data;
    ShowLocationDetail(loc.Name);
};

Best Practices

  • Set Style before adding the widget to the form to avoid double repaints.
  • Adjust Size to fit your layout — defaults are minimum comfortable sizes.
  • Read BeepEventDataArgs.Data in event handlers for context-specific click data.
  • Call Invalidate() after updating data properties to trigger a repaint.
  • Let theme colors flow from BeepStyling.CurrentTheme — avoid overriding color properties unless customization is explicitly desired.
