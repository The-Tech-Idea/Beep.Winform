BeepNavigationWidget

Navigation widget with ten styles covering breadcrumbs, step indicators, tabs, pagination, menu bars, sidebar navigation, wizard steps, process flows, tree navigation, and quick actions.

Stable v1.0.164 Beep Widgets / Navigation

Overview

BeepNavigationWidget is a paint-based WinForms widget that inherits from BaseControl. It uses an IWidgetPainter pipeline to render rich visuals without child controls. The Style property selects one of 10 built-in render modes; changing it at runtime immediately swaps the painter and triggers a repaint.

Key Features

🧭 Ten Navigation Styles

Every navigation pattern from breadcrumb to tree — one widget.

✅ Step Progress

StepIndicator and WizardSteps track completed/active/pending visually.

🌳 Tree Support

TreeNavigation supports hierarchical items via NavigationItem.ParentId.

⚡ QuickActions

Row of icon+label buttons for toolbar-style quick access.

🖱️ NavigationChanged

Fires whenever the active item changes — no plumbing needed.

🔢 Pagination

Pagination style includes prev/next buttons and numbered page buttons.

Basic Usage

// Wizard step indicator
var wizard = new BeepNavigationWidget
{
    Style        = NavigationWidgetStyle.WizardSteps,
    CurrentIndex = 1,  // currently on step 2 (0-based)
    Items = new List
    {
        new NavigationItem { Label = "Account",  IsActive = false },
        new NavigationItem { Label = "Profile",  IsActive = true  },
        new NavigationItem { Label = "Payment",  IsActive = false },
        new NavigationItem { Label = "Confirm",  IsActive = false },
    },
    Size = new Size(480, 60)
};
this.Controls.Add(wizard);

Event Handling

wizard.NavigationChanged += (s, e) =>
{
    int step = (int)e.Data;
    ShowStepPanel(step);
};

Best Practices

  • Set Style before adding the widget to the form to avoid double repaints.
  • Adjust Size to fit your layout — defaults are minimum comfortable sizes.
  • Read BeepEventDataArgs.Data in event handlers for context-specific click data.
  • Call Invalidate() after updating data properties to trigger a repaint.
  • Let theme colors flow from BeepStyling.CurrentTheme — avoid overriding color properties unless customization is explicitly desired.
