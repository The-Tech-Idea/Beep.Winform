Overview

The BeepWebHeaderAppBar control provides a web-style application header bar with 16 distinct visual styles, each implemented as a painter class conforming to IWebHeaderStylePainter. Every painter extends WebHeaderStylePainterBase, which provides shared drawing primitives for logos, search boxes, tabs, and buttons.

The styles range from startup hero headers to SaaS professional, ecommerce, material design, multi-row compact, and mobile-first layouts. Each style controls header background, logo rendering, tab/breadcrumb rendering, search bar, action buttons, and responsive layout behavior.

IWebHeaderStylePainter Contract

public interface IWebHeaderStylePainter
{
    void PaintHeader(Graphics g, Rectangle bounds, WebHeaderAppBarConfig config);
    Rectangle GetTabBounds(int index, Rectangle headerBounds, WebHeaderAppBarConfig config);
    Rectangle GetButtonBounds(string buttonKey, Rectangle headerBounds, WebHeaderAppBarConfig config);
    string GetHitElement(Point point, Rectangle headerBounds, WebHeaderAppBarConfig config);
}
MethodDescription
PaintHeaderRenders the entire header: background, logo, tabs, search box, action buttons, notifications
GetTabBoundsReturns the bounding rectangle for a tab at the given index (used for click hit-testing)
GetButtonBoundsReturns the bounding rectangle for a named action button (e.g., "Search", "Cart", "Profile")
GetHitElementReturns a string identifying which UI element is at the given point: "Logo", "Tab:0", "Search", "Cart", etc.

Helper Classes (8)

The AppBar system is supported by 8 coordination and utility classes:

ClassResponsibility
BeepAppBarMasterHelperCentral orchestrator that coordinates layout, events, drawing, and state management across all sub-helpers
LayoutHelperComputes responsive layout: logo position, tab spacing, button alignment, search bar sizing, overflow handling
EventHelperRoutes mouse clicks, hover, and keyboard events to the appropriate header element (tab, button, search)
DrawingHelperLow-level GDI+ drawing utilities: gradient fills, rounded rectangles, text measurement, icon rendering
DragHelperHandles drag-drop reordering of tabs within the header bar
ComponentFactoryCreates and configures child controls (dropdowns, textboxes, buttons) that overlay the painted header
StateStorePersists per-header state: active tab index, search text, notification count, collapsed/expanded state
IBeepAppBarHostInterface that the parent form implements to receive header events (tab clicks, search submit, button clicks)

Examples

Header Style Switching

var appBar = new BeepWebHeaderAppBar();

appBar.LogoText = "MyApp";
appBar.Pages = new[] { "Dashboard", "Reports", "Settings", "Admin" };

appBar.HeaderStyle = HeaderStyle.SaaSProfessional;

appBar.HeaderStyle = HeaderStyle.EcommerceModern;
appBar.ShowSearchBox = true;
appBar.NotificationCount = 5;

appBar.HeaderStyle = HeaderStyle.MaterialDesign3;

appBar.HeaderStyle = HeaderStyle.StartupHero;

appBar.HeaderStyle = HeaderStyle.MultiRowCompact;
appBar.SearchPlaceholder = "Search everything...";

appBar.Host = this;
appBar.TabClicked += (s, e) =>
{
    var pageName = appBar.Pages[e.TabIndex];
    NavigateToPage(pageName);
};
