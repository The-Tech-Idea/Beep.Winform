Chart Viewport & Performance

Overview

Namespace: TheTechIdea.Beep.Winform.Controls.Charts
Assembly: TheTechIdea.Beep.Winform.Controls.dll

The chart control provides two static helper classes to manage viewport navigation and rendering performance. BeepChartViewportHelper (internal) handles interactive zoom/pan via mouse wheel drag and keyboard, auto-scale computation, and viewport limit enforcement. BeepChartPerformanceHelper (public) provides point culling, vertex decimation, grid density optimization, label recommendation, and rendering cost estimation for large datasets exceeding thousands of data points. Together, they enable smooth interaction with datasets of 100,000+ points on modest hardware.

The viewport is defined by four properties: ViewportXMin, ViewportXMax, ViewportYMin, ViewportYMax. These represent the visible data range and can diverge from the full data range when the user zooms or pans. Series painters receive the viewport range (not the full data range) through their xMin/xMax/yMin/yMax parameters, so only visible data is drawn.

Auto-Scale Logic

BeepChartViewportHelper provides two primary methods for viewport range management:

AutoScaleViewport

Scans all data points across all series to compute the full data range, then sets the viewport to that range plus 10% padding on each axis. This ensures no data point touches the edge of the plot area.

internal static void AutoScaleViewport(BeepChart chart)
{
    float dataXMin = float.MaxValue, dataXMax = float.MinValue;
    float dataYMin = float.MaxValue, dataYMax = float.MinValue;

    foreach (var series in chart.DataSeries)
    {
        foreach (var point in series.DataPoints)
        {
            float x = BeepChartDataHelper.ConvertXValue(chart, point);
            float y = BeepChartDataHelper.ConvertYValue(chart, point);
            dataXMin = Math.Min(dataXMin, x);
            dataXMax = Math.Max(dataXMax, x);
            dataYMin = Math.Min(dataYMin, y);
            dataYMax = Math.Max(dataYMax, y);
        }
    }

    float xPadding = (dataXMax - dataXMin) * 0.1f;
    float yPadding = (dataYMax - dataYMin) * 0.1f;

    // Handle degenerate ranges (single point or identical values)
    if (Math.Abs(dataXMax - dataXMin) < float.Epsilon)
        { xPadding = 1f; dataXMin -= 0.5f; dataXMax += 0.5f; }
    if (Math.Abs(dataYMax - dataYMin) < float.Epsilon)
        { yPadding = dataYMax * 0.1f; }

    chart.ViewportXMin = dataXMin - xPadding;
    chart.ViewportXMax = dataXMax + xPadding;
    chart.ViewportYMin = dataYMin - yPadding;
    chart.ViewportYMax = dataYMax + yPadding;
}

EnforceViewportLimits

Called after every user interaction (zoom, pan) to clamp values and prevent inverted ranges (XMin >= XMax). If the minimum exceeds the maximum due to over-zooming, the range is collapsed to a single-point span.

internal static void EnforceViewportLimits(BeepChart chart)
{
    if (chart.ViewportXMin > chart.ViewportXMax)
    {
        float center = (chart.ViewportXMin + chart.ViewportXMax) / 2f;
        chart.ViewportXMin = center - 0.5f;
        chart.ViewportXMax = center + 0.5f;
    }
    if (chart.ViewportYMin > chart.ViewportYMax)
    {
        float center = (chart.ViewportYMin + chart.ViewportYMax) / 2f;
        chart.ViewportYMin = center - 0.5f;
        chart.ViewportYMax = center + 0.5f;
    }
}

The 10% padding ensures data points never sit exactly on the axis lines, which is important for bar charts (bars would otherwise be clipped at the edge) and bubble charts (circles would be cut off). The padding is applied symmetrically on both sides of the range.

Real-Time Streaming

// Internal streaming buffer structure (per series):
class StreamingBuffer
{
    Queue<ChartDataPoint> pendingPoints = new();
    List<ChartDataPoint> activePoints = new(); // visible window

    void Append(ChartDataPoint point)
    {
        pendingPoints.Enqueue(point);
    }

    List<ChartDataPoint> Flush(int maxPoints)
    {
        while (pendingPoints.Count > 0)
        {
            activePoints.Add(pendingPoints.Dequeue());
            while (activePoints.Count > maxPoints)
                activePoints.RemoveAt(0);
        }
        return new List<ChartDataPoint>(activePoints);
    }
}

Point Culling

Point culling reduces the number of points sent to the painter by filtering out those that fall outside the current viewport. This is critical for large datasets where 99% of points may be off-screen.

PropertyDescription
EnablePointCullingbool, default false. When true, CullPointsOutsideViewport() is called before each paint cycle. Only points within the viewport range are passed to the series painter.
CullPointsOutsideViewportFilters a list to points where pointX >= xMin && pointX <= xMax. Critically, the first and last points of the series are always retained, even if they fall outside the viewport. This prevents line segments from abruptly starting/stopping at the viewport edge — the line painter can extend from the last off-screen point into the visible region.
public static List<ChartDataPoint> CullPointsOutsideViewport(
    List<ChartDataPoint> points,
    Func<ChartDataPoint, float> toXValue,
    float xMin, float xMax)
{
    if (!ShouldOptimizePerformance(points.Count, 1000))
        return new List<ChartDataPoint>(points); // pass-through for small sets

    var result = new List<ChartDataPoint> { points.First() };

    for (int i = 1; i < points.Count - 1; i++)
    {
        float x = toXValue(points[i]);
        if (x >= xMin && x <= xMax)
            result.Add(points[i]);
    }

    result.Add(points.Last());
    return result;
}

Vertex Decimation

Vertex decimation reduces the number of rendered vertices to a target count while preserving the visual shape of the data.

Property / MethodDescription
EnableVertexSimplificationbool, default false. When true, decimation is applied before rendering. The last data point is always preserved.
DecimatePoints(points, factor)Takes every Nth point from the list. factor=5 keeps every 5th point (80% reduction). Always includes the last point. Returns a new list.
EstimateDecimationFactor(count, target=200)Computes the factor needed to reduce a dataset to approximately the target point count. factor = Math.Max(1, count / target). For 10,000 points with target=200, returns factor=50.
SimplificationTolerancefloat property tied to the chart. Controls decimation aggressiveness. Higher = more points removed. Used in conjunction with EstimateDecimationFactor.
public static List<ChartDataPoint> DecimatePoints(
    List<ChartDataPoint> points, int factor)
{
    if (factor <= 1) return new List<ChartDataPoint>(points);

    var result = new List<ChartDataPoint>(points.Count / factor + 2);
    for (int i = 0; i < points.Count; i += factor)
        result.Add(points[i]);

    // Always include the last point to preserve the right boundary
    if (result.Last() != points.Last())
        result.Add(points.Last());

    return result;
}

public static int EstimateDecimationFactor(int count, int target = 200)
{
    if (count <= target) return 1;
    return Math.Max(1, count / target);
}

if (EnablePerformanceMode || EnablePointCulling || EnableVertexSimplification)
{
    bool shouldOpt = BeepChartPerformanceHelper.ShouldOptimizePerformance(
        GetTotalPointCount(), LargeDatasetThreshold);

    if (shouldOpt && EnablePointCulling)
    {
        optimizedPoints = BeepChartPerformanceHelper.CullPointsOutsideViewport(
            allPoints, p => ConvertXValue(p), ViewportXMin, ViewportXMax);

    if (shouldOpt && EnableVertexSimplification)

            optimizedPoints?.Count ?? allPoints.Count);

            optimizedPoints ?? allPoints, factor);
    }

    OnPerformanceOptimized(beforeCount, optimizedPoints?.Count ?? allPoints.Count);
    OnLabelOptimizationApplied(optimizedCount, factor);
}

Examples

Large Dataset Optimization

var chart = new BeepChart();
chart.ChartType = ChartType.Line;
chart.EnablePerformanceMode = true;
chart.LargeDatasetThreshold = 5000;  // optimize above 5K points
chart.EnablePointCulling = true;
chart.EnableVertexSimplification = true;
chart.SimplificationTolerance = 0.5f;

// Generate 100,000 points
var points = new List<ChartDataPoint>();
var rng = new Random(42);
for (int i = 0; i < 100000; i++)
{
    float x = i * 0.1f;
    float y = (float)(Math.Sin(x * 0.1) * 100 + rng.NextDouble() * 10);
    points.Add(new ChartDataPoint { X = x.ToString("F1"), Y = y.ToString("F2") });
}

chart.DataSeries.Add(new ChartDataSeries
{
    SeriesName = "Signal",
    DataPoints = points
});

chart.AutoScaleViewport();

// Monitor optimization
chart.PerformanceOptimized += (s, e) =>
    Console.WriteLine($"Optimized: {e.BeforePointCount} -> {e.AfterPointCount} points");

// Get rendering cost estimate
float cost = BeepChartPerformanceHelper.EstimateRenderingCost(
    chart.GetTotalPointCount(), chart.Width);
if (cost > 0.5f)
    chart.EnablePointCulling = true;

Real-Time Streaming Setup

chart.ChartType = ChartType.Line;
chart.EnableRealTimeStreaming = true;
chart.StreamRenderThrottleMs = 50;   // 20 FPS max
chart.MaxStreamPointsPerSeries = 500; // rolling 500-point window
chart.ChartTitle = "Live Sensor Data";

chart.DataSeries.Add(new ChartDataSeries
{
    SeriesName = "Temperature",
    DataPoints = new List<ChartDataPoint>()
});

chart.AutoScaleViewport();

// Streaming data source (e.g., Timer or background thread)
var timer = new System.Timers.Timer(100); // 10 Hz data source
timer.Elapsed += (s, e) =>
{
    float value = ReadSensor();
    chart.BeginInvoke(new Action(() =>
    {
        chart.AppendDataPoint(0, new ChartDataPoint
        {
            X = DateTime.Now.ToString("HH:mm:ss.fff"),
            Y = value.ToString("F1")
        }, flush: false); // batch every 50ms via throttle
    }));
};
timer.Start();

chart.StreamingDataAppended += (s, e) =>
    Console.WriteLine($"{e.NewPointCount} new, {e.TotalBufferedPoints} buffered");

chart.IncrementalRenderCompleted += (s, e) =>
    Debug.WriteLine($"Batch render: {e.RenderTimeMs}ms, {e.PointsRendered}pts");
