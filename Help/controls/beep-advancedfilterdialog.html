Overview

BeepAdvancedFilterDialog is a modal dialog built on BeepiFormPro that provides a grid-based interface for creating complex, multi-row filter conditions. Each row represents one FilterCriteria — the user selects a column, an operator (from 17 supported operators), and one or two values. Multiple rows can be combined with AND or OR logic to form a complete FilterConfiguration.

The dialog is designed to be invoked from a parent grid context (typically BeepGridPro). It receives the grid's column metadata, renders an interactive BeepGridPro for filter editing, and returns a fully populated FilterConfiguration via the Result property when the user clicks Apply. Saved configurations can be persisted to JSON and reloaded later.

This control lives in the TheTechIdea.Beep.Winform.Controls.GridX.Filtering namespace. It shares the filtering model layer with BeepFilter, BeepQuickFilterBar, and BeepDateRangePicker.

Key Features

Grid-Based Filter Editor

Uses a BeepGridPro to display filter rows — each row is a FilterRowData instance bound via BindingList<T>.

17 Filter Operators

All operators from the FilterOperator enum: Equals, Contains, GreaterThan, Between, IsNull, Regex, In, and more.

AND / OR Logic

A top-level combo box lets users choose FilterLogic.And (all criteria must match) or FilterLogic.Or (any criterion matches).

Pre-load Existing Configurations

Accepts an optional existingConfig parameter to pre-populate the grid with a previously saved filter set.

JSON Persistence

Save and Load buttons serialize/deserialize the FilterConfiguration to/from .json files on disk.

Full Theme Integration

Inherits BeepiFormPro theming. Deep-propagates theme to all child controls including the embedded BeepGridPro.

Between / NotBetween Support

Two value columns — the second column (Value 2) is used when the operator is Between or NotBetween.

Dynamic Add / Remove Rows

Buttons to add new filter rows or remove selected rows. Clear All resets to a single empty row.

FilterRowData Model

FilterRowData is an internal INotifyPropertyChanged class that represents one row in the filter grid. It bridges the UI row to the FilterCriteria domain model.

PropertyTypeDefaultDescription
ColumnNamestring""The name of the column to filter on. Bound to a BeepComboBox with all visible grid columns.
Operatorstring"Contains"The string name of the FilterOperator. Bound to a BeepComboBox showing all 17 operators with symbols.
Valuestring""The primary filter value. Bound to a BeepColumnType.Text cell editor.
Value2string""The secondary filter value used with Between/NotBetween operators. Bound to a text editor column labeled "Value 2 (Between)".

All four properties raise PropertyChanged on set, allowing the grid to reflect live edits.

The constructor flow is: set theme → call InitializeDialog() → call LoadColumns() to extract visible columns → call ConfigureFilterGrid() to bind data and configure grid columns → optionally load an existing configuration → call ApplyTheme().

User Interface Layout

The dialog is composed of three docked panels:

PanelDockHeightContent
Logic Panel
_logicPanel		 Top 40px A BeepLabel ("Combine filters using:") and a BeepComboBox with "AND" and "OR" items. Controls how multiple criteria are combined.
Filter Grid
_filterGrid		 Fill Auto A BeepGridPro backed by BindingList<FilterRowData>. Four columns: Column (combo), Operator (combo with symbols), Value (text), Value 2 / Between (text). Read-only mode is off; navigator and filters are hidden.
Button Panel
_buttonPanel		 Bottom 50px Seven BeepButton controls: + Add, - Remove, Clear All, Save, Load, Apply, and Cancel.

The dialog window itself is 900×500px, styled as FormStyle.Modern, centered on parent, with caption bar visible and min/max buttons hidden (maximize is enabled).

Filter Configuration Flow

The dialog interacts with the following types from the TheTechIdea.Beep.Winform.Controls.Filtering namespace:

TypeRole in Dialog
FilterConfigurationThe output container. Holds Name, Logic, IsActive, and a List<FilterCriteria>. Serialized to JSON on save.
FilterCriteriaA single filter condition: ColumnName, Operator (enum), Value, optional Value2, CaseSensitive, IsEnabled.
FilterOperatorEnum with 17 values: Equals, NotEquals, Contains, NotContains, StartsWith, EndsWith, GreaterThan, GreaterThanOrEqual, LessThan, LessThanOrEqual, Between, NotBetween, IsNull, IsNotNull, Regex, In, NotIn.
FilterLogicEnum: And, Or.

Theming

The dialog inherits theming from BeepiFormPro. ApplyTheme() resolves the active theme from CurrentTheme or BeepThemesManager.CurrentTheme or BeepThemesManager.GetDefaultTheme(). It then applies colors:

  • BackColor / ForeColor — mapped from theme.BackColor / theme.ForeColor with SystemColors fallback.
  • Panel Backgrounds — both _logicPanel and _buttonPanel use theme.PanelBackColor with fallback to theme.BackColor.
  • Embedded Grid — _filterGrid.Theme is set to the dialog's Theme, then _filterGrid.ApplyTheme() is called.
  • Child Controls — PropagateThemeToChildren(this) recurses the control tree and sets Theme on every BaseControl or IBeepUIComponent.

Examples

Example 1: Basic Invocation

Open Advanced Filter from a Grid
// In the parent form or grid toolbar handler:
var dialog = new BeepAdvancedFilterDialog(
    grid.ColumnConfigs,
    existingConfig: null,
    themeName: "Dark"
);

if (dialog.ShowDialog() == DialogResult.OK
    && dialog.Result != null)
{
    grid.ApplyFilterConfiguration(dialog.Result);
}
dialog.Dispose();

Example 2: Pre-loading an Existing Filter

Edit an Existing Filter
// _currentFilter is a previously created FilterConfiguration
var dialog = new BeepAdvancedFilterDialog(
    grid.ColumnConfigs,
    existingConfig: _currentFilter
);

if (dialog.ShowDialog() == DialogResult.OK
    && dialog.Result != null)
{
    _currentFilter = dialog.Result;
    RefreshGrid();
}
dialog.Dispose();

Example 3: Build OR-based Date Range Filter

OR Logic with Multiple Criteria
// The user would:
// 1. Set "Combine filters using:" to "OR"
// 2. Row 1: Column=OrderDate, Operator=Between, Value=2023-01-01, Value2=2023-06-30
// 3. Click [+ Add] for row 2
// 4. Row 2: Column=Status, Operator=Equals, Value=Pending
// 5. Click [Apply]

// Result:
// FilterConfiguration with Logic=Or and two FilterCriteria:
//   (OrderDate Between) and (Status Equals)
// Effectively: OrderDate in H1 OR Status is Pending

Example 4: Preserving Filter State

Serialize / Deserialize Filter
// Save current filter to JSON (user clicks Save button)
// Internally:
var json = JsonSerializer.Serialize(config, new JsonSerializerOptions
{
    WriteIndented = true
});
File.WriteAllText(dialog.FileName, json);
BeepDialogManager.Instance.NotifyExportSuccess(fileName, criteriaCount);

// Load from JSON (user clicks Load button)
// Internally:
var json = File.ReadAllText(dialog.FileName);
var config = JsonSerializer.Deserialize<FilterConfiguration>(json);
LoadConfiguration(config);
BeepDialogManager.Instance.NotifyImportSuccess(fileName, criteriaCount);

Architecture

BeepAdvancedFilterDialog.cs Contains two classes:
FilterRowData — INotifyPropertyChanged model for grid rows
BeepAdvancedFilterDialog — the dialog form with grid editor, logic selector, toolbar, and serialization
FilterConfiguration (Filtering) Domain model: collection of FilterCriteria with logical operator, name, timestamps, and serialization support.
FilterCriteria (Filtering) Single filtered condition: column, operator, value(s), case-sensitivity, enabled state.
FilterOperator (Filtering) Enum of 17 comparison operators with GetDisplayName() and GetSymbol() extension methods.
BeepiFormPro (Forms/ModernForm) Base class providing modern form chrome, caption bar, theming, and resizing.
BeepGridPro (GridX) Embedded data grid used as the filter row editor surface.
