BeepAppTree

A control that displays App hierarchical data in a tree format with dynamic branch management, drag-drop, and context menus

Overview

BeepAppTree extends BeepTree and implements ITree to provide an application-level hierarchical tree view. It supports dynamic branch creation from configuration-driven assembly classes, drag-and-drop reordering, right-click context menus with dynamic method invocation, checkbox toggling, runtime icon and text updates, and recursive branch search/retrieval. The control integrates with the Beep service infrastructure (IBeepService, IDMEEditor, IAppManager) for configuration-driven UI generation.

Key Features

  • Configuration-driven branches — Creates branch nodes dynamically from DMEEditor.ConfigEditor.BranchesClasses using reflection-based activation
  • Genre and Root branches — Supports Genre-type grouping branches and Root-type entity branches with automatic hierarchy
  • Drag-and-drop — Built-in BeepTreeNodeDragandDropHandler for reordering tree nodes
  • Context menus — Dynamic menu methods created via DynamicMenuManager for each branch, with click and double-click invocation
  • Runtime icon/text updates — ChangeBranchIcon() and ChangeBranchText() with 3 overloads each (by ID, name, or branch)
  • Checkbox support — TurnonOffCheckBox() toggles visibility of checkboxes across all nodes
  • Recursive branch search — GetBranch() with 4 overloads (by text, ID, text+parent, text+branchtype) plus GUID and entity ID lookups
  • Tree refresh — RefreshTree() with 4 overloads to rebuild the entire tree or update individual branches
  • Node removal — RemoveNode() with 4 overloads (by ID, branch, name, GUID) with recursive cleanup from Nodes and Branches
  • Filtering — PreShowItem event allows canceling branch visibility before rendering

Theming

BeepAppTree inherits theming from BeepTree, which extends BaseControl. In design mode, it defaults to TreeStyle.AntDesign. The base BeepTree provides full theming support including node colors, selection highlighting, expand/collapse icons, checkbox styling, and scrollbar theming. All colors are resolved through the Beep theme system.

Examples

// Create and initialize BeepAppTree
var appTree = new BeepAppTree();
appTree.init(beepService, appManager);

// The tree is now fully configured:
// - Treebranchhandler set up
// - Drag-and-drop handler attached
// - Node click and menu selection events wired
// - ExtensionsHelpers created

// Create the root tree from configuration
var result = appTree.CreateRootTree();

// Wait for completion (CreateRootTree creates all Genre and Root branches)
if (result.Flag == Errors.Ok)
{
    Console.WriteLine($"Created {appTree.Nodes.Count} root nodes, " +
                      $"{appTree.Branches.Count} branches");
}

// Change a branch icon at runtime
appTree.ChangeBranchIcon("Customers", "customer-active.svg");

// Change branch text
appTree.ChangeBranchText(42, "Updated Branch Name");

// Toggle checkboxes
appTree.TurnonOffCheckBox(true);

// Find a branch by various criteria
var branch1 = appTree.GetBranch("Customers");
var branch2 = appTree.GetBranch(42);
var branch3 = appTree.GetBranchByGuidID("abc-123-def");

// Handle events
appTree.PreShowItem += (s, e) =>
{
    // Optionally cancel showing a branch
    if (e.ObjectType.Contains("Hidden"))
        e.Cancel = true;
};

appTree.RefreshBranch += (s, branch) =>
{
    Console.WriteLine($"Refresh requested for branch: {branch.Name}");
};

// Refresh specific branches
appTree.RefreshTree(branch1);       // by branch reference
appTree.RefreshTree(42);            // by ID
appTree.RefreshTree("Customers");   // by name

// Remove a branch
appTree.RemoveNode("OldBranch");
appTree.RemoveNodeByGuidID("guid-to-remove");

Architecture

BeepAppTree extends BeepTree and implements ITree. It integrates deeply with the Beep configuration system: IDMEEditor.ConfigEditor.BranchesClasses provides assembly class definitions that are activated via reflection (ObjectActivator<IBranch>) to create branch instances. Genre branches act as grouping containers, while Root branches represent entity-level nodes.

The control uses DynamicMenuManager to create context menu methods and global menus for each branch. DynamicFunctionCallingManager handles dynamic method invocation with applicability checks (IsMethodApplicabletoNode()). Branch retrieval uses recursive search through FindBranchRecursive() which traverses the full ChildBranchs hierarchy.

The PreShowItem event allows consumer code to conditionally hide branches through a cancel mechanism, enabling dynamic visibility filtering based on permissions, user roles, or runtime state. The partial class structure supports designer-sample nodes in design mode and full production behavior at runtime.
