Overview

BeepBlockNavigationBar is a record-navigation and command toolbar bound to a single BeepBlock instance. It renders ten SVG-icon buttons for record navigation (first, previous, next, last), CRUD operations (new, delete), query management (enter/execute query), and persistence (commit/rollback). A position label on the right displays the current record index and total count.

Namespace: TheTechIdea.Beep.Winform.Controls.Integrated.Blocks
Assembly: TheTechIdea.Beep.Winform.Controls.Integrated.dll
Inheritance: BaseControl → BeepBlockNavigationBar (sealed)
Designer: none (no custom designer attribute)

All buttons are created once in the constructor and their visibility and enablement are controlled by the bound block via IsNavigatorCommandVisible() and IsNavigatorCommandEnabled().

Key Features

  • Ten command buttons — First, Previous, Next, Last, New, Delete, Enter Query, Execute Query, Save (Commit), and Rollback
  • All SVG icons — Each button uses an embedded SVG image path (e.g. angle-double-small-left.svg, floppy-disk.svg)
  • Position indicator — Right-aligned label showing current / total (e.g. 5 / 42) via GetNavigatorPositionText()
  • Block-driven visibility — Each button's visibility and enabled state is resolved from the bound BeepBlock per command key
  • Frameless button style — All buttons are frameless, 24x24px with 18x18 max image size, appearing as a clean icon strip
  • Theme-aware — Theme is inherited from the bound BeepBlock automatically

Theming

BeepBlockNavigationBar disables borders, shadows, and rounding (ShowAllBorders = false, ShowShadow = false, IsRounded = false). ApplyTheme() sets the Theme property on the _positionLabel and all buttons in the _buttons dictionary. Chrome backgrounds fall back to ParentBackColor or SystemColors.Control when the inherited back color is fully transparent.

Examples

// Create and bind a navigation bar to a BeepBlock
var navBar = new BeepBlockNavigationBar
{
    Dock = DockStyle.Bottom,
    Block = myBeepBlock
};
this.Controls.Add(navBar);

// The nav bar automatically syncs button visibility/enablement
// from the block's ViewStateChanged event
myBeepBlock.ViewStateChanged += (s, e) =>
{
    // navBar updates automatically;
    // manual refresh is not needed
};

Architecture

The control uses a two-column TableLayoutPanel: the left column (100% width) contains a FlowLayoutPanel with all command buttons; the right column (120px fixed) contains the BeepLabel position indicator. Buttons are created once in CreateButtons() and stored in a Dictionary<string, BeepButton> keyed by command name. SyncFromBlock() is called on every ViewStateChanged event and iterates through button keys, querying the block's IsNavigatorCommandVisible(key) and IsNavigatorCommandEnabled(key) to set each button's visible and enabled state. When no block is bound, all buttons disable and the position label shows 0 / 0.
