Overview

BeepBlock is a BaseControl-derived block-level UI surface that implements IBeepBlockView. Each BeepBlock is bound to one logical FormsManager block and provides a complete CRUD form with field definitions, record binding, validation, presenter delegates, and query mode support. It integrates with the BeepForms host for coordinated data operations.

Namespace: TheTechIdea.Beep.Winform.Controls.Integrated.Blocks
Assembly: TheTechIdea.Beep.Winform.Controls.Integrated.dll
Inheritance: BaseControl → BeepBlock
Implements: IBeepBlockView
Designer: BeepBlockDesigner

Defined as a partial class, the control manages a complete UI lifecycle: binding to a FormsHost, syncing from the manager, resolving entity metadata, reconciling designer-generated field bindings, and refreshing presentation layouts.

Key Features

  • Full CRUD surface — Provides a Block-level UI surface for viewing, creating, editing, and deleting records
  • Manager-bound — Connects to one logical FormsManager block via ManagerBlockName for data operations
  • Definition-driven layout — BeepBlockDefinition controls field layout, presentation, captions, and editor options
  • Entity metadata — Exposes BeepBlockEntityDefinition with connection, entity, and field structure
  • Presenter registry — BeepBlockPresenterRegistry manages custom field presenters with default registrations
  • Record binding — Syncs current record, record count, dirty state, and query mode from the manager
  • Validation-aware — Subscribes to manager validation events and resets validation state on unbind
  • Trigger state — Manages trigger/rule state derived from the manager and block definition
  • Grid integration — Syncs grid state from the FormsManager for multi-record views
  • Design-time support — BeepBlockDesigner with custom editors for Definition, Entity, and BlockName

Theming

BeepBlock inherits theming from BaseControl. It initializes with double-buffering, user paint, resize-redraw, and transparent background support via SetStyle() in InitializeComponent(). The control supports auto-scrolling for overflow content. Presentation is refreshed through RefreshPresentation() which is called on definition changes, binding/unbinding, and manager sync.

Examples

// Create a BeepBlock for customer data
var customerBlock = new BeepBlock
{
    BlockName = "CustomerDetails",
    Definition = new BeepBlockDefinition
    {
        BlockName = "CustomerDetails",
        Caption = "Customer Details",
        ManagerBlockName = "CustomerDetails",
        Entity = new BeepBlockEntityDefinition
        {
            ConnectionName = "MainDB",
            EntityName = "Customers",
            IsMasterBlock = true,
            Fields = new List<BeepBlockFieldDefinition>
            {
                new BeepBlockFieldDefinition
                {
                    FieldName = "CustomerName",
                    Caption = "Customer Name",
                    EditorType = FieldEditorType.TextBox,
                    IsRequired = true
                },
                new BeepBlockFieldDefinition
                {
                    FieldName = "Email",
                    Caption = "Email Address",
                    EditorType = FieldEditorType.TextBox
                },
                new BeepBlockFieldDefinition
                {
                    FieldName = "Status",
                    Caption = "Status",
                    EditorType = FieldEditorType.ComboBox
                }
            }
        }
    }
};

// Bind to a BeepForms host
customerBlock.Bind(beepForms);

// Handle view state changes
customerBlock.ViewStateChanged += (s, e) =>
{
    var vs = customerBlock.ViewState;
    Console.WriteLine($"IsDirty: {vs.IsDirty}, Mode: {vs.Mode}, " +
                      $"Record: {vs.CurrentRecordIndex + 1}/{vs.RecordCount}");
};

// Sync from manager
customerBlock.SyncFromManager();

// Create entity snapshot for introspection
var snapshot = customerBlock.CreateEntitySnapshotFromManager();

// Customize the presenter registry
customerBlock.PresenterRegistry.RegisterDefaults();

Architecture

BeepBlock is a partial class implementing IBeepBlockView. It extends BaseControl for theming and rendering. The control uses a BeepBlockViewState for tracking bound state, dirty flags, query mode, record index, and record count. A BeepBlockPresenterRegistry manages field-level custom presenters.

The Bind()/Unbind() methods manage the full lifecycle: attaching to a BeepForms host, subscribing to manager events for validation and triggers, syncing record binding from the IUnitOfWork, and reconciling designer-generated field bindings with the effective definition. The SyncFromManager() method refreshes runtime definition from block info, validates the current unit of work, updates the view state, and refreshes the presentation layout.

The partial class pattern separates the public IBeepBlockView implementation from internal layout, record binding, trigger management, grid synchronization, validation, and presentation logic across multiple partial files.
