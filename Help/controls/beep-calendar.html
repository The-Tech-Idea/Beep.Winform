Overview

Namespace: TheTechIdea.Beep.Winform.Controls.Calendar
Assembly: TheTechIdea.Beep.Winform.Controls.dll
Inheritance: BaseControl

BeepCalendar is the most decomposed control in the Beep library, split across 90+ partial files organized into 8 subsystems. It provides a complete calendar experience with 7 view modes (Month, Week, WorkWeek, Day, Timeline, Agenda, List), full event lifecycle management (add, update, delete, undo/redo), conflict detection, mouse/touch interaction with drag-to-create and drag-to-move, and a theme-aware painting pipeline. An optional sidebar mini-calendar provides quick date navigation.

The control uses a centralized Paint() dispatcher that routes to view-mode-specific renderers, a style-based painter system, and a legacy renderer fallback. During design time in Visual Studio, it renders a simplified preview.

Key Features

  • 7 View Modes — Month (7-column grid, up to 5 events per cell with overflow), Week/WorkWeek (7-day grid with hourly time slots), Day (single day with 24 hourly slots), Timeline (horizontal time spans), Agenda/List (chronological event list). Configurable density: Compact, Standard, Comfortable.
  • Event CRUD — Add, update, delete events with a three-phase mutation pipeline (EventMutating → execution → EventMutated + history recording). Cancelable pre-mutation.
  • Undo/Redo — Two-stack history system (_undoStack + _redoStack). Each mutation records full before/after state. Convenience API: CanUndo, CanRedo, Undo(), Redo().
  • Conflict Detection — Three policy modes: None, Warn, Strict. Checks scheduling overlap before commit. Events fire ConflictDetected with conflicting event details.
  • Drag-to-Create — Click-drag on date cells or empty surfaces to create new events. Visual ghost preview during drag. Ctrl+drag copies existing events.
  • Time Snapping — Configurable snap interval (default 15 min). Events snap to nearest N-minute boundary during drag operations.
  • CalendarEventEditor — QuickEdit (560x420) and DialogEdit (720x620) modes. Fields: Title, Start/End DateTimePickers, AllDay CheckBox, Location, Status, Organizer, Description.
  • Categories & Resources — Color-coded categories with hierarchy. Room/equipment resource scheduling.
  • Sidebar — Compact mini-calendar with month navigation, dot indicators for days with events, today highlight.
  • Theme-Aware — 9-file LayoutTheme subsystem maps IBeepTheme tokens to calendar colors (cells, headers, today, selected, events, weekends).

Interaction System

16 partial files handle all pointer/touch/mouse events with view-mode-specific hit testing:

  • Drag Modes — ResolveDragMode() returns CalendarInteractionMode: MoveEvent (event block in Day/Week/WorkWeek/Timeline), ResizeStart/End (event block edges), SelectEvent (Month/Agenda/List), RangeSelect (Day/Week date cells), CreateEvent (empty surface).
  • Hit Testing — 8 files: HitTesting.cs dispatches to view-specific resolvers (MonthView, WeekView, DayView, TimelineView, TimedView, Agenda). Returns CalendarInteractionHitTestResult with TargetKind (EventBlock/DateCell/EmptySurface), RequestedMode, Location, and Date.
  • Timing Snap — InteractionSnapIntervalMinutes (default 15). Events snap to nearest interval. Timing.cs, Snap.cs, Snap.Location.cs handle the snapping logic.
  • Ctrl+Drag Copy — IsCopyModifierDown() checks for Ctrl. When active, dragging an existing event creates a copy at the drop location.
  • Commitment — Commit.cs dispatches to NewEvent.cs (create from proposal), Copy.cs (Ctrl+drag copy), or ExistingEvent.cs (mutation). All route through EventMutating → CRUD → EventMutated → History.

Event Management

Event Model

CalendarEvent: Id, Title, Start, End, IsAllDay, Location, Description, Status, Category (references EventCategory), Organizer, CreatedDate, ModifiedDate.

Mutation Pipeline

  1. Pre-mutation — RaiseMutating fires EventMutating. Cancelable.
  2. Execution — Data change occurs (add/update/delete).
  3. Post-mutation — RaiseMutated fires EventMutated. History record created. View button states updated.

CalendarEventEditor

Implements ICalendarEventEditor. Two modes: QuickEdit (560x420 compact) and DialogEdit (720x620 full form). TryEdit(request, out editedEvent) opens a BeepDialogForm with CalendarEventEditorContent (UserControl) containing Title, Start/End DateTimePickers, AllDay CheckBox, Location, Status ComboBox, and optional Advanced panel (Organizer, Description).

Conflict Detection

CalendarConflictPolicyMode: None (disabled), Warn (fire ConflictDetected but allow), Strict (fire and block). Conflict resolution checks scheduling overlap. GetConflicts(event) returns conflicting events.

Paint Dispatcher

private void Paint(Graphics g, Rectangle bounds)
{
    if (IsDesignModeSafe)
        DrawDesignTimePreview(g, bounds);       // VS designer
    else if (_usePainterSystem && _stylePainter != null)
        DrawWithPainter(g, contentRect);        // New system
    else
        DrawWithLegacyRenderer(g, contentRect);  // Fallback
}

cal.ViewMode = CalendarViewMode.Month;
cal.CurrentDate = DateTime.Today;
cal.DensityMode = CalendarDensityMode.Standard;

Event Management with Validation

cal.EventMutating += (s, e) =>
{
    if (e.Kind == CalendarEventMutationKind.Delete
        && e.Original.Category == "Locked")
        e.Cancel = true;
};

cal.EventMutated += (s, e) =>
    Console.WriteLine($"{e.Kind}: {e.Result.Title}");

// Conflict detection
cal.ConflictPolicyMode = CalendarConflictPolicyMode.Strict;
cal.ConflictDetected += (s, e) =>
    MessageBox.Show($"Cannot schedule: {e.Description}");

Meeting Scheduler Example

public class MeetingScheduler
{
    private BeepCalendar cal;
    private List<DateTime> bookedSlots = new();

    private void Calendar_InteractionCompleted(object s, CalendarInteractionEventArgs e)
    {
        var selectedTime = e.Date;
        if (IsTimeSlotAvailable(selectedTime))
            ScheduleMeeting(selectedTime);
        else
            MessageBox.Show("This slot is already booked");
    }

    private bool IsTimeSlotAvailable(DateTime dt)
        => !bookedSlots.Any(s => Math.Abs((s - dt).TotalMinutes) < 60);

    private void ScheduleMeeting(DateTime dt)
    {
        bookedSlots.Add(dt);
        cal.Events.Add(new CalendarEvent
        {
            Title = "New Meeting",
            Start = dt,
            End = dt.AddHours(1)
        });
    }
}
