A themable, painter-driven combo box with 11 visual variants, multi-select, inline editing, search filtering, loading states, and full accessibility support.

Overview

BeepComboBox is a modern, fully themable combo box control built on the BaseControl framework. It uses a painter methodology where each visual variant (ComboBoxType) delegates rendering to a dedicated painter class, enabling consistent styling across 11 design-system variants while sharing core logic for selection, popup management, and keyboard navigation.

The control is implemented across 6 partial class files and supported by 11 painter classes, 5 helper engines, and the shared BeepContextMenu for dropdown popups. It inherits border rendering, DPI scaling, theme application, and hit-testing from BaseControl.

Key Features

11 Visual Variants

OutlineDefault, OutlineSearchable, FilledSoft, RoundedPill, SegmentedTrigger, MultiChipCompact, MultiChipSearch, DenseList, MinimalBorderless, CommandMenu, VisualDisplay — each with dedicated painter and layout tokens.

Painter Methodology

Stateless frame-by-frame rendering: ComboBoxStateFactory snapshots live state into an immutable ComboBoxRenderState, ComboBoxLayoutEngine computes rectangles, and the active painter renders without reaching back into the control.

Multi-Select with Chips

Two dedicated multi-select variants (MultiChipCompact, MultiChipSearch) render selected items as chip tokens with close (×) buttons, overflow count badges (+N), and animated chip add/remove transitions.

Inline Text Editor

When editable or searchable, a BeepTextBox child control overlays the text area for live typing. Supports auto-complete with prefix/fuzzy/full matching modes, free-text tokenization via delimiter characters, and debounced suggestions.

Searchable Popup

Types that support search (OutlineSearchable, MultiChipSearch, CommandMenu) show a search box in the dropdown. The ComboBoxSearchEngine provides prefix, substring, and fuzzy-subsequence matching against both primary text and sub-text.

Loading States

Two distinct loading mechanisms: an animated spinner that replaces the chevron while IsLoading = true, and a shimmer skeleton sweep when ShowSkeleton = true (for async data population).

Validation States

Four validation states (None, Error, Warning, Success) drive border colour, background tint, and trailing status icons (SVG: AlertCircle, AlertTriangle, Check).

RTL & Accessibility

Full right-to-left layout mirroring via IsRtl. Custom AccessibleObject exposes combo box value, role, and expanded/collapsed state to screen readers. Keyboard navigation covers Up/Down/Home/End/PageUp/PageDown/Enter/Escape/Space/Backspace.

ComboBoxType Variants

Each variant maps to a dedicated IComboBoxPainter implementation and a set of ComboBoxVisualTokens (button width, inner padding, corner radius, text inset, row density). Popup Behavior

The dropdown popup is implemented using BeepContextMenu, the same menu infrastructure used by the application menu bar. Each time the dropdown opens:

  1. A fresh BeepContextMenu instance is created with DestroyOnClose = true.
  2. The menu is configured with the combo box's current state: MultiSelect, ShowCheckBox, ShowSearchBox, Theme.
  3. All ListItems are copied to MenuItems, with checkbox state synced for multi-select.
  4. The menu is positioned at PointToScreen(Point(0, Height)) with width matching the control.
  5. ItemClicked and MenuClosed events are wired for selection and state cleanup.

Popup Row Model System

The popup content is normalized through ComboBoxPopupModelBuilder, which converts raw SimpleItem collections into immutable ComboBoxPopupRowModel instances. Each row knows its RowKind:

Dropdown ListBoxType Mapping

Each ComboBoxType maps to a ListBoxType for popup appearance. When DropdownListBoxType is null: OutlineDefault → Outlined, FilledSoft → Filled, RoundedPill → Rounded, CommandMenu → CommandList, VisualDisplay → AvatarList, etc.

Inline Editor & Search

Inline Editor Lifecycle

The inline editor is a BeepTextBox child control (frameless, no shadow, ControlStyle.None) created lazily on first use. It overlays _textAreaRect when the user clicks the text area in an editable or searchable combo box.

  • Creation: Theme and font applied once at creation — never on every show.
  • Show: ShowInlineEditor() positions the editor over _textAreaRect, pre-fills with current display text, suppresses live-search during programmatic text assignment.
  • Live Search: InlineEditor_TextChanged syncs _inputText and opens the dropdown if not already open.
  • Free-Text Tokenization: When AllowFreeText + AllowMultipleSelection are enabled, typing , or ; creates a chip from the typed token and clears input.
  • Key Handling: Enter commits, Escape discards, Down commits and shows dropdown.
  • Commit: HideInlineEditor(true) matches typed text via FindItemByText. If found, selects the match; otherwise keeps free text.

Auto-Complete Engine

  • Prefix: StartsWith(StringComparison.OrdinalIgnoreCase)
  • Fuzzy: All typed characters must appear in order in the item text (character walk).
  • Full: Contains(StringComparison.OrdinalIgnoreCase)

Configurable debounce via AutoCompleteDelay. Results capped at MaxSuggestions.

Search Engine (Popup)

ComboBoxSearchEngine.FilterRows provides scored search:

  • Prefix: baseWeight + 300 - lengthDiff
  • Substring: baseWeight + 200 - startIndex
  • Fuzzy: baseWeight + 100 - spread

Group headers are kept only when at least one child item matches. Best match tracked for keyboard focus.

Multi-Select Mode

Enabling

Set AllowMultipleSelection = true or use ComboBoxType.MultiChipCompact / MultiChipSearch. The popup switches to checkbox rows; selected items render as chip tokens.

Chip Layout Engine

ComboBoxChipLayoutEngine computes chip positions left-to-right, wrapping to a second row if needed (up to 2 rows). Each chip has body, text, and close button (16px ×) rectangles. Exceeding space produces a +N overflow badge.

Chip Rendering

ComboBoxChipPainter draws rounded-rect chips using theme tokens (SelectedBackColor, SelectedForeColor, SelectedBorderColor). Disabled chips use muted colours. Contrast ratio checked (2.5:1 minimum). Overflow badge shows +N text centered. Close buttons render SVG X at 75% opacity.

Hit-test rects for chip bodies and close buttons are registered in ChipCloseRects and ChipBodyRects dictionaries, keyed by item identity. Chip body click opens popup and can scroll to item.

Chip Animation

When SelectedItems is set, the setter diffs old/new lists and starts animations for added/removed chips. A 16 ms (~60 FPS) timer drives linear progress 0→1 (add) or 1→0 (remove) over ChipAnimationDuration.

Popup Selection Flow

  • Immediate: Checkbox toggle updates SelectedItems via deferred BeginInvoke. Popup stays open.
  • Apply/Cancel: Pending list; Apply commits, Cancel restores snapshot.
  • Primary Action: Single CTA button with {count} placeholder.
  • Select All: Virtual _selectall row toggles all/none.

Validation & Skeleton Loading

Validation States

ValidationState controls:

  • Border: Resolved from ErrorBorderColor, WarningBorderColor, SuccessBorderColor.
  • Background tint: 20-alpha overlay over the text area.
  • Status icon: Trailing SVG: AlertCircle (error), AlertTriangle (warning), Check (success).
  • Text colour: Error state uses ComboBoxErrorForeColor.

Loading Spinner

When IsLoading = true: translucent haze covers the text area (90-alpha of ComboBoxBackColor). Rotating CircleArrowRight SVG animates in the dropdown button area at 20 FPS (15°/tick, 50 ms). All mouse interaction disabled.

Skeleton Shimmer

When ShowSkeleton = true: all normal content replaced by shimmer. Base gray fill with an animated highlight band sweeping left-to-right using a triangular-blend LinearGradientBrush. Sweep advances at ~42 FPS (24 ms interval, SkeletonOffset 0→1). Clipped to text area.

Architecture

The BeepComboBox follows a layered architecture with strict separation of concerns:

Partial Class Breakdown

FileLinesContents

BeepComboBox.Core.cs743Private fields, constructor, layout management, DPI scaling (OnDpiScaleChanged), loading/skeleton timers, chip animation helpers, Dispose.

Render Pipeline (per frame)

  1. DrawContent(g) calls base.DrawContent(g) (border + background via BaseControl).
  2. ComboBoxStateFactory.Build(this) snapshots all live state into an immutable ComboBoxRenderState.
  3. ComboBoxLayoutEngine.Compute(DrawingRect, renderState, this) calculates sub-rectangles (text area, dropdown button, clear button, image, chips).
  4. If IsRtl, all rectangles are mirrored horizontally.
  5. The painter is lazily created via ComboBoxTypeRegistry.CreatePainter.
  6. _comboBoxPainter.Paint(g, this, state, layout) renders chips → text → decorations → clear button → validation icon → dropdown button → spinner/skeleton.
  7. RegisterHitAreas() registers interactive zones for mouse hover/cursor changes.
