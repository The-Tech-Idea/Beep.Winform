Overview

Namespace: TheTechIdea.Beep.Winform.Controls.DialogsManagers.Forms

Assembly: TheTechIdea.Beep.Winform.Controls

Inheritance: BeepiFormPro

Source: 1156 lines, single file (DialogsManagers\Forms\BeepDialogForm.cs)

The BeepDialogForm is a pure code-behind modal dialog form with no designer file. It follows the same architecture pattern as BeepCommandPaletteDialog, constructing all panels and controls programmatically in the constructor. It provides a header (icon + title), a scrollable body (message, optional input/combo, validation, details, confirmation), and a button footer panel with 1–3 action buttons that support 12 button-set configurations.

The dialog integrates with the BeepDialogManager infrastructure, supports 18 semantic DialogPreset values for color/icon/button adaptation, and provides advanced features: input validation with shake animation, typed confirmation for destructive actions, auto-close countdown, keyboard navigation (Tab/Shift+Tab cycling, Enter/Escape), and callback-driven validation.

Key Features

  • Pure code-behind — no .Designer.cs file; all layout is programmatic with Dock stacking
  • Three-panel layout: Header (BeepPanel with BeepImage + BeepLabel), Body (BeepPanel with scroll), Button footer (BeepPanel)
  • 6 DialogType values: Information, Warning, Error, Question, GetInputString, GetInputFromList, None
  • 12 BeepDialogButtons sets: Ok, Cancel, YesNo, OkCancel, AbortRetryIgnore, SaveDontSaveCancel, SaveAllDontSaveCancel, Close, Help, TryAgainContinue, OK, Continue
  • 18 DialogPreset semantic values: Information, Success, Warning, Danger, Question, DestructiveConfirm, UnsavedChanges, BlockingError, SessionTimeout, InlineValidation, SuccessWithUndo, MultiStepProgress, Announcement
  • Inline text input with password masking and multiline support
  • ComboBox-based list selection (GetInputFromList)
  • Input validation via Func<string, string?> callback with red border + error label + shake animation
  • Typed confirmation for destructive actions (e.g., type "DELETE" to confirm)
  • Expandable details section (Show/Hide toggle)
  • Auto-close countdown timer with annotated button labels (e.g., "OK (5)")
  • Keyboard navigation: Tab/Shift+Tab cycling between inputs and buttons; Enter for default action, Escape for cancel
  • Custom content hosting via CustomContent property (any Control, DockStyle.Fill)
  • Custom button labels via CustomButtonLabels dictionary
  • Preset-driven icon selection and semantic theming (accent, warning, danger, success colors)
  • Reduced motion flag to disable shake animation
  • Auto-focus on shown: input box → combo box → default button

Events

BeepDialogForm does not expose custom public events. Communication is via the standard DialogResult property and the following state properties read after ShowDialog() returns:

  • ReturnValue (string) — text payload (input text, selected combo item text, or clicked button label)
  • ReturnItem (SimpleItem?) — selected combo box item for GetInputFromList mode
  • DialogResult (WinForms) — standard OK/Cancel/Yes/No/Abort/Retry/Ignore mapping

Internal events that drive button behavior:

  • _leftButton.Click → LeftButton_Click: sets ReturnValue, maps DialogResult per button set, closes
  • _middleButton.Click → MiddleButton_Click: sets ReturnValue, maps DialogResult per button set, closes
  • _rightButton.Click → RightButton_Click: sets ReturnValue to null, maps DialogResult per button set, closes
  • _detailsToggleButton.Click → DetailsToggleButton_Click: toggles _detailsExpanded and updates visibility
  • _inputBox.TextChanged → InputBox_TextChanged: runs InputValidator callback
  • _confirmationBox.TextChanged → ConfirmationBox_TextChanged: checks typed keyword match

Theming

BeepDialogForm applies theming via ApplyTheme(), which propagates the current BeepTheme to all child controls (panels, labels, text boxes, combo boxes, buttons). Key theme tokens used:

Theme TokenUsage
ErrorColorValidation error label text and input border color
TextBoxBorderColorInput box border when no error (fallback: ButtonBorderColor)
ButtonBackColor / ButtonForeColorCancel/secondary button styling
AccentColorDefault primary button color (when no preset)
Preset-specific button colorsResolved via DialogStyleAdapter.GetPresetPrimaryColor() — maps DialogPreset to appropriate theme color (SuccessColor, ErrorColor, WarningColor, DialogInformationButtonBackColor, etc.)
Preset-specific header tintResolved via DialogStyleAdapter.GetPresetHeaderTint() — subtle semantic background tint

The primary button also receives computed contrast foreground (ColorUtils.GetContrastColor) and hover back color (ColorUtils.ShiftLuminance(primaryColor, 0.12f)).

Examples

Simple Information Dialog

using (var dialog = new BeepDialogForm())
{
    dialog.Title = "Operation Complete";
    dialog.Message = "Your files have been processed successfully.";
    dialog.DialogType = DialogType.Information;
    dialog.PresetIntent = DialogPreset.Success;
    dialog.ShowDialog(this);

    if (dialog.DialogResult == DialogResult.OK)
        Console.WriteLine("User acknowledged.");
}

Text Input with Validation

using (var dialog = new BeepDialogForm())
{
    dialog.Title = "Enter Name";
    dialog.Message = "Please enter your full name:";
    dialog.DialogType = DialogType.GetInputString;
    dialog.DialogButtons = BeepDialogButtons.OkCancel;
    dialog.InputValidator = input =>
        string.IsNullOrWhiteSpace(input) ? "Name is required" : null;
    dialog.ShowDialog(this);

    if (dialog.DialogResult == DialogResult.OK)
        Console.WriteLine($"Name: {dialog.ReturnValue}");
}

Destructive Confirmation

using (var dialog = new BeepDialogForm())
{
    dialog.Title = "Delete Project?";
    dialog.Message = "This action is permanent and cannot be undone.";
    dialog.PresetIntent = DialogPreset.DestructiveConfirm;
    dialog.RequireTypedConfirmation = true;
    dialog.ConfirmationKeyword = "DELETE";
    dialog.DisablePrimaryUntilAcknowledged = true;
    dialog.DialogType = DialogType.Error;
    dialog.CustomButtonLabels = new Dictionary<BeepDialogButtons, string>
    {
        [BeepDialogButtons.Ok] = "Delete",
        [BeepDialogButtons.Cancel] = "Keep"
    };
    dialog.ShowDialog(this);

    if (dialog.DialogResult == DialogResult.OK)
        Console.WriteLine("Project deleted.");
}

Combo Box Selection

using (var dialog = new BeepDialogForm())
{
    dialog.Title = "Select Category";
    dialog.Message = "Choose a category:";
    dialog.DialogType = DialogType.GetInputFromList;
    dialog.DialogButtons = BeepDialogButtons.OkCancel;
    dialog.Items = new List<SimpleItem>
    {
        new SimpleItem { Text = "Electronics" },
        new SimpleItem { Text = "Clothing" },
        new SimpleItem { Text = "Books" }
    };
    dialog.ShowDialog(this);

    if (dialog.DialogResult == DialogResult.OK)
        Console.WriteLine($"Category: {dialog.ReturnValue}, Item: {dialog.ReturnItem?.Text}");
}

Auto-Close Timer

using (var dialog = new BeepDialogForm())
{
    dialog.Title = "Session Timeout";
    dialog.Message = "Your session will expire soon.";
    dialog.PresetIntent = DialogPreset.SessionTimeout;
    dialog.DialogType = DialogType.Warning;
    dialog.CustomButtonLabels = new Dictionary<BeepDialogButtons, string>
    {
        [BeepDialogButtons.Ok] = "Sign In",
        [BeepDialogButtons.Cancel] = "Not Now"
    };
    dialog.StartAutoClose(30000); // 30-second countdown
    dialog.ShowDialog(this);
}

Architecture

FileResponsibility
DialogsManagers\Forms\BeepDialogForm.csSingle-file implementation (1156 lines). Contains: constructor (builds header/body/button panels with all child controls), DialogType and DialogButton configuration, button click handlers with DialogResult mapping, keyboard handling (ProcessCmdKey for Tab/Escape/Enter), Focus cycling (CycleFocus), input validation (InputBox_TextChanged), typed confirmation (ConfirmationBox_TextChanged), details expand/collapse, auto-close timer, theming (ApplyTheme), body content layout (ReflowBodyContent and MeasureWrappedTextHeight), and primary action enable/disable logic
DialogsManagers\Helpers\DialogStyleAdapter.csResolves preset-aware primary button colors and header tints from BeepTheme tokens
DialogsManagers\Helpers\DialogMotionEngine.csProvides shake animation for validation errors (cycles, amplitude, duration)
DialogsManagers\Models\DialogPreset.csDialogPreset enum and DialogPresetStyling utility class
Forms\BeepDialogModal.csDefines the DialogType enum (Information, Warning, Error, Question, GetInputString, GetInputFromList, None)

The dialog uses a Dock stacking layout: Body (Fill) is added first, then Button panel (Bottom), then Header (Top) — this ensures correct Dock order. The body panel uses AutoScroll with AutoScrollMinSize dynamically set by ReflowBodyContent() which measures wrapped text heights and positions all body controls vertically with 8px spacing.

Button handling maps the BeepDialogButtons set to 1–3 visible buttons using a logical-to-physical mapping (MapLogicalButtonToControl). Button positioning is right-aligned for 2–3 buttons, centered for single buttons.
