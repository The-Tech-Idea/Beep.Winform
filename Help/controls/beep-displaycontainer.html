BeepDisplayContainer

A container control for hosting addins in tabbed or single-panel layouts with theme integration and event-driven lifecycle management

Overview

BeepDisplayContainer is a container control that implements IDisplayContainer for hosting IDM_Addin components. It inherits from BaseControl and resides in TheTechIdea.Beep.Winform.Controls. The control supports two layout modes: SinglePanel (one addin at a time in a plain Panel) and TabbedPanel (multiple addins in BeepTabs pages). It manages addin lifecycle (initialize, resume, dispose), provides popup support for forms and controls, and handles keyboard shortcuts. Container rendering is intentionally skipped (PainterKind.None, empty OnPaint/OnPaintBackground) since child controls paint themselves, and WS_EX_COMPOSITED is enabled for smooth DPI-aware compositing.

Key Features

  • Dual container modes: SinglePanel (one addin) and TabbedPanel (multiple tabs via BeepTabs)
  • Five tab visual styles: Classic, Underline, Capsule, Minimal, Segmented (plus Button, Card)
  • Auto-mapping of BeepControlStyle to appropriate TabStyle via predefined rules
  • Addin lifecycle management: initialize, resume, dispose
  • Popup support for Form and Control addins (rendered in BeepiFormPro)
  • Addin deduplication using AddinScopeCreateType.Single attribute check
  • Event system for addin add/remove/move/change operations
  • Key combination forwarding via KeyPressed event
  • DPI-aware composited rendering with WS_EX_COMPOSITED
  • Theme propagation to TabContainerPanel and ContainerPanel backgrounds

Theming

ApplyTheme() sets ContainerPanel.BackColor and BackColor from _currentTheme.BackgroundColor. It forwards Theme and TabStyle to TabContainerPanel. The ControlStyle setter triggers UpdateTabStyleFromControlStyle() which maps Beep control styles to tab styles: FigmaCard/TailwindCard/DiscordStyle/StripeDashboard/NeoBrutalist → Card; Minimal/NotionMinimal/VercelClean → Minimal; GlassAcrylic/Windows11Mica/Fluent2/Material3 → Underline; PillRail/ChakraUI → Capsule; iOS15/MacOSBigSur → Segmented; Bootstrap/AntDesign/Neumorphism → Button. Theme re-application checks if ControlStyle changed externally via _lastControlStyle tracking.

Examples

// Tabbed container with multiple addins
var container = new BeepDisplayContainer
{
    ContainerType = ContainerTypeEnum.TabbedPanel,
    TabStyle = TabStyle.Underline,
    ControlStyle = BeepControlStyle.Material3,
    Size = new Size(800, 600),
    Dock = DockStyle.Fill
};

container.AddinAdded += (s, e) =>
{
    Console.WriteLine($"Addin added: {e.TitleText}");
};

container.AddinRemoved += (s, e) =>
{
    Console.WriteLine($"Addin removed: {e.TitleText}");
};

// Add addins
container.AddControl("Dashboard", myDashboardAddin, ContainerTypeEnum.TabbedPanel);
container.AddControl("Settings", mySettingsAddin, ContainerTypeEnum.TabbedPanel);
container.AddControl("Reports", myReportsAddin, ContainerTypeEnum.TabbedPanel);

// Show a specific addin
container.ShowControl("Dashboard", myDashboardAddin);

// Show addin as popup
await container.ShowPopup(myQuickViewAddin);

// Apply a tab style preset
container.SetTabStylePreset(TabStyle.Segmented);

// Single panel mode
container.ContainerType = ContainerTypeEnum.SinglePanel;
container.AddControl("MainView", myMainAddin, ContainerTypeEnum.SinglePanel);

this.Controls.Add(container);

Architecture

BeepDisplayContainer implements IDisplayContainer and manages two internal hosting surfaces: ContainerPanel (Panel for SinglePanel mode) and TabContainerPanel (BeepTabs for TabbedPanel mode). Only one surface is visible at a time, controlled by UpdateContainerLayout() which also migrates controls between modes. An internal Dictionary<string, (IDM_Addin, BeepTabPage?)> tracks all hosted addins by title. AddControl() checks for AddinAttribute.ScopeCreateType == Single to prevent duplicate single-scope addins, calls control.Initialize(), and wraps controls in BeepTabPage for tabbed mode. ShowPopup() handles both Form-based and Control-based addins, wrapping the latter in a BeepiFormPro dialog. Mouse events are forwarded to TabContainerPanel for proper tab interaction. The control uses PainterKind.None, empty OnPaint/OnPaintBackground, and WS_EX_COMPOSITED for optimal child rendering performance.
