Overview

Namespace: TheTechIdea.Beep.Winform.Controls.Docking

Assembly: TheTechIdea.Beep.Winform.Controls

Inheritance: Panel

The BeepDockspace is a Krypton-style docking cell that owns a single tab header bar and hosts DockPanel instances as tabbed pages within its client area. It represents one edge of the docking layout (Left, Right, Top, or Bottom) and is typically created and managed by a BeepDockingManager. Each dockspace manages its own tab strip, caption buttons (Close, Pin, DropDown), and active page visibility.

The control is not toolbox-visible ([ToolboxItem(false)]) and is designed to be instantiated programmatically by the docking manager infrastructure. It implements a Krypton-inspired page selection API with SelectPage, PageForKey, and CellVisiblePages methods. All header rendering is delegated to the DockingPainterFactory caption renderers based on the current BeepControlStyle.

Key Features

  • Krypton-style tab header bar with caption button support (Close, Pin/AutoHide, DropDown)
  • Tab strip rendering via CaptionLayoutManager with configurable tab widths (min 1px, max 160px)
  • Active page tracking via ActivePanelKey with page bring-to-front logic
  • Designer-aware activation: pages are selected via ISelectionService at design time
  • DockPosition synchronization: child DockPanel instances inherit the dockspace's position
  • Drop-down context menu for active pages: Floating, Auto Hide, Close
  • Owner-drawn header via caption painter factory (style-dependent rendering)
  • Double-buffered painting with OptimizedDoubleBuffer and AllPaintingInWmPaint
  • Automatic panel layout: all dock panels share the same bounds (below the header) with z-order stacking
  • Manager-attachment: panels auto-adopt the dockspace's manager when added
  • Designtime detection supporting both Site.DesignMode and process-name heuristics

Events

BeepDockspace does not expose custom public events. Interaction is routed through the following inherited events and internal callbacks:

  • MouseDown/MouseMove — inherited from Panel, used for tab hit-testing and cursor changes
  • Paint — inherited from Panel, used for header rendering
  • ControlAdded/ControlRemoved — inherited from Panel, used for DockPanel lifecycle management
  • Layout — inherited from Panel, used for panel layout synchronization

Caption button actions are dispatched to the BeepDockingManager: Close (_manager.ClosePanel), AutoHide (_manager.MakeAutoHiddenRequest), Float (_manager.FloatPanel).

Theming

The dockspace uses DockingThemeColors which provides:

Theme TokenUsage
PanelBackColorDockspace background color
PanelForeColorDockspace foreground/text color

The header rendering is delegated to DockingPainterFactory.GetRenderers(ControlStyle).Caption.Paint(), which uses a DockingPainterContext with the theme colors, control style, and header bounds. The caption layout manager (CaptionLayoutManager) handles the geometry of tabs and buttons, while the caption painter renders the actual pixels.

Control style defaults to Material3 and can be changed by the owning BeepDockingManager. The theme is propagated to all child DockPanel instances via their ApplyDockingTheme method.

Examples

Creating a Dockspace with a Docking Manager

var manager = new BeepDockingManager();
manager.Dock = DockStyle.Fill;
this.Controls.Add(manager);

var dockspace = new BeepDockspace
{
    Dock = DockStyle.Left,
    Width = 250
};
dockspace.Manager = manager;

var panel = new DockPanel
{
    Key = "toolbox",
    Title = "Toolbox",
    CanClose = true,
    CanAutoHide = true,
    CanFloat = true
};
panel.Controls.Add(new Label { Text = "Tool content" });
dockspace.Controls.Add(panel);
this.Controls.Add(dockspace);

Navigating Panels Programmatically

var dockspace = manager.Dockspaces.FirstOrDefault();
dockspace.SelectPage("properties");  // Krypton-style

DockPanel panel = dockspace.PageForKey("toolbox");
if (panel != null)
    dockspace.ActivatePanel(panel);

DockPanel[] allPanels = dockspace.CellVisiblePages("toolbox");
foreach (var p in allPanels)
    Console.WriteLine(p.Title);

Handling Tab Selection

dockspace.MouseDown += (s, e) =>
{
    var hitPanel = dockspace.HitTestTabAt(e.Location);
    if (hitPanel != null)
    {
        Console.WriteLine($"Tab hit: {hitPanel.Title}");
        dockspace.SelectTabAt(e.Location);
    }
};

Architecture

FileResponsibility
Docking\BeepDockspace.csSingle-file implementation (514 lines). Contains all logic: constructor, properties, theme application, tab/page navigation (ActivatePanel, SelectPage, PageForKey, CellVisiblePages), header drawing (RecomputeCaptionLayout, DrawHeader), context menu (ShowActivePageMenu), hit testing (HitTestTab, SelectTabAt, HandleHeaderMouseDown), panel layout (LayoutPanels), panel lifecycle (OnControlAdded/Removed), mouse handlers, and designer process detection
Docking\Helpers\CaptionLayoutManager.csShared layout engine that computes tab and caption button geometry; used by RecomputeCaptionLayout
Docking\Models\CaptionTabModel.csData model for a tab entry: Key, Title, IconPath, IsDirty, IsActive, Tag
Docking\Models\DockPanel.csThe page control hosted inside the dockspace; has Key, Title, CanClose, CanAutoHide, CanFloat, TabBounds
Docking\Painters\DockingPainterFactory.csFactory that provides style-specific caption renderers
Docking\Models\DockingThemeColors.csTheme color data for the docking system

The dockspace follows the caption layout manager pattern: tab geometries are computed by CaptionLayoutManager.Compute() and stored in a dictionary keyed by CaptionTabModel. Tab bounds are mirrored back to each DockPanel.TabBounds for compatibility with docking manager operations. The active panel is resolved by ActivePanelKey lookup, falling back to frontmost child by z-order.

Caption buttons (Close, Pin, DropDown) are built dynamically for the active page based on its CanClose, CanAutoHide, and presence. The drop-down context menu offers Floating, Auto Hide, and Close actions, dispatching to the owning BeepDockingManager.
