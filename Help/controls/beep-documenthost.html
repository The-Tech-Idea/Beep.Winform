Overview

BeepDocumentHost is a UserControl-based multi-document interface container that provides Visual-Studio-style tabbed document management. Drop it from the toolbox onto a form and it handles the full document lifecycle — open, close, activate, split, float, auto-hide, and persist — using an internal tab strip and nested BeepDocumentPanel surfaces.

Namespace: TheTechIdea.Beep.Winform.Controls.DocumentHost
Assembly: TheTechIdea.Beep.Winform.Controls.dll
Inheritance: System.Windows.Forms.UserControl → BeepDocumentHost
Designer: BeepDocumentHostDesigner (intelligent smart-tag with wizard, context menus, layout assistant)

Organized across multiple partial files spanning the core host, layout tree, features, and an extensive design-time designer with 8 partial files for verbs, context menus, drag-drop, panel siting, setup, tab selection, layout presets, and design-time document CRUD.

Key Features

  • Tabbed document MDI — Open multiple documents as tabbed panels, each hosting any WinForms control
  • Six tab styles — Chrome, VS Code, Underline, Pill, Office, and Flat via DocumentTabStyle
  • Flexible tab positioning — Top, Bottom, Left, Right, or Hidden via TabStripPosition
  • Configurable close behaviour — Close button always visible, on hover only, or hidden entirely (TabCloseMode)
  • Split groups — Side-by-side horizontal or vertical split views with independent tab strips per group
  • Nested splits — Recursive split layouts up to MaxSplitDepth nesting levels
  • Floating documents — Drag a tab out to a free-floating window; dock it back with one call
  • Auto-hide strips — Collapse documents to slim edge strips with configurable hover-delay peek behaviour
  • Keyboard shortcuts — Built-in Ctrl+Tab, Ctrl+W, Ctrl+1–9, Ctrl+Shift+T via CommandRegistry
  • Quick-switch popup — Ctrl+Tab document picker with MRU ordering
  • Layout persistence — Full layout tree serialization to/from JSON via SaveLayout()
  • Session auto-save — Auto-save and restore the tab arrangement to a SessionFile
  • Design-time wizard — First-drop quick setup picks tab style, seeds sample documents, selects template
  • Per-document tab colours — Accent bar, full background, or bottom border via TabColorMode
  • Tab preview tooltips — Rich thumbnail previews on tab hover
  • Cross-host drag — Drag tabs between separate BeepDocumentHost instances
  • Transactional docking — Split/dock mutations wrapped in designer transactions for full undo support
  • Theme integration — Propagates ThemeName to tab strip and all document panels

Document States

Each document managed by BeepDocumentHost can be in one of four primary states, mirroring Visual Studio’s MDI model:

  • Docked — The document occupies a region inside the host with its tab visible in the group’s tab strip. Splitters separate side-by-side groups.
  • Split — The layout is divided into multiple BeepDocumentGroup regions, each with its own independent tab strip and active document.
  • Floating — The document is detached from the host in a separate tool window. Drag the title bar to reposition; right-click for a dock-back option.
  • Auto-Hidden — The document collapses to a slim edge strip. Hover (after AutoHideHoverDelay ms) to peek the panel as a flyout; click to pin it open.

State transitions are managed by the host. The AllowFloat, AllowSplit, AllowPin, and AllowAutoHide policy properties let the application restrict which states are permitted.

// Query the current dock state of a document
var state = documentHost.GetDocumentDockState("editor1");
switch (state)
{
    case DocumentDockState.Docked:
        // Document is docked in a group
        break;
    case DocumentDockState.Floating:
        // Document is in its own floating window
        break;
    case DocumentDockState.AutoHide:
        // Document is collapsed to an edge strip
        break;
}

Split Groups & Nested Layouts

BeepDocumentHost supports recursive split layouts via a layout tree composed of GroupLayoutNode and SplitLayoutNode instances. Each group has its own tab strip, independent tab positioning (via SetGroupTabPosition), and active document.

// Split the current document horizontally (side-by-side)
documentHost.SplitDocumentHorizontal("editor1");

// Split the current document vertically (top/bottom)
documentHost.SplitDocumentVertical("editor2");

// Create a nested split — another split within an existing group
documentHost.SplitDocumentHorizontal("editor3");

// Inspect the current layout tree
var root = documentHost.LayoutRoot;
// root may be a GroupLayoutNode (single group)
// or a SplitLayoutNode (horizontal/vertical split with two children)

// Merge everything back to one group
documentHost.MergeAllGroups();

Configure split behaviour through MaxGroups (1–8), MaxSplitDepth (1–5), SplitHorizontal, and SplitRatio properties. The LayoutRoot property provides read-only access to the current tree structure, while GroupTabPositions returns a per-group position summary string.

Theming

BeepDocumentHost supports the Beep theming system through ThemeName. Setting a theme name propagates to the tab strip, document panels, and all nested child controls:

// Apply a Beep theme by name
documentHost.ThemeName = "DarkTheme";
// Theme propagates to tab strip painters and all BeepDocumentPanel children.

// Change appearance of the host itself
documentHost.ControlStyle = DocumentHostStyle.Raised;

// Per-document tab colouring via TabColorMode
documentHost.TabColorMode = TabColorMode.AccentBar;
// Each document can set its own TabColor/AccentColor via DocumentDescriptor.

The TabStyle property controls the visual shape of tabs independently of the theme colour system. The six built-in styles are Chrome (rounded trapezoid), VSCode (flat with active highlight), Underline (bottom accent line), Pill (capsule shape), Office (ribbon-style), and Flat (square edges). Toggle TabPreviewEnabled to overlay thumbnail previews on tab hover tooltips.

Examples

Basic Setup

// BeepDocumentHost is a UserControl — drop it on a form from the toolbox
// or create it in code:
var documentHost = new BeepDocumentHost
{
    Dock = DockStyle.Fill,
    TabStyle = DocumentTabStyle.Chrome,
    TabPosition = TabStripPosition.Top,
    CloseMode = TabCloseMode.OnHover,
    ShowAddButton = true,
    KeyboardShortcutsEnabled = true
};
this.Controls.Add(documentHost);

// Open documents by adding panels with hosted content
documentHost.AddDocument("welcome", "Welcome", new Label
{
    Text = "Welcome to BeepDocumentHost!",
    Dock = DockStyle.Fill,
    TextAlign = ContentAlignment.MiddleCenter
});

documentHost.AddDocument("editor1", "Program.cs", new RichTextBox
{
    Dock = DockStyle.Fill
});

documentHost.AddDocument("output", "Output", new TextBox
{
    Dock = DockStyle.Fill,
    Multiline = true,
    ReadOnly = true
});

Float, Split & Auto-Hide

// Float a document into its own window
documentHost.FloatDocument("output");

// Create a side-by-side split view
documentHost.SplitDocumentHorizontal("editor1");
// Now two side-by-side document groups exist, each with its own tab strip

// Auto-hide a document to the right edge
documentHost.AutoHideDocument("outline", AutoHideSide.Right);

// Restore the auto-hidden document
documentHost.RestoreAutoHideDocument("outline");

// Dock a floated document back
documentHost.DockBackDocument("output");

// Pin a frequently-used document
documentHost.PinDocument("welcome", pinned: true);

Layout Persistence

// Save the entire layout (groups, splits, tab order, active document)
string json = documentHost.SaveLayout();
System.IO.File.WriteAllText("session.json", json);

// Later: restore the exact layout
string saved = System.IO.File.ReadAllText("session.json");
documentHost.LoadLayout(saved);

// Or enable automatic session persistence
documentHost.AutoSaveLayout = true;
documentHost.SessionFile = "workspace_layout.json";
// The host automatically saves on close and restores on next load.

Events & Keyboard Shortcuts

// React to document lifecycle events
documentHost.ActiveDocumentChanged += (s, e) =>
{
    Console.WriteLine($"Active document is now: {documentHost.ActiveDocumentId}");
};

documentHost.DocumentClosed += (s, e) =>
{
    Console.WriteLine($"Document closed: {e.DocumentId}");
};

documentHost.DocumentFloated += (s, e) =>
{
    Console.WriteLine($"Document floated: {e.DocumentId}");
};

// Customize keyboard shortcuts via the CommandRegistry
var registry = documentHost.CommandRegistry;
// Bind custom actions to key combinations
// registry.RegisterCommand("MyCommand", Keys.Control | Keys.Shift | Keys.D, ...);

Cross-Host Tab Drag

// Enable dragging tabs between two separate BeepDocumentHost instances
leftHost.AllowDragBetweenHosts = true;
rightHost.AllowDragBetweenHosts = true;

// When a user drags a tab from leftHost to rightHost, the document
// is moved between hosts automatically. Both hosts must share the
// same process (they cannot be in separate applications).

The host is a standalone control — it does not require BeepDocumentManager to function. When a BeepDocumentManager is bound to a BeepDocumentHost, the manager acts as a non-visual component orchestrator that coordinates the host alongside docking panels, providing a unified design-time API that surfaces identical smart-tag actions from both the manager and the host.

Designer: On first drop from the toolbox, BeepDocumentHostDesigner launches a quick-setup wizard (tab style, sample documents, template). A single designer transaction wraps the entire initialization so Ctrl+Z undoes the first-drop setup. The smart-tag exposes all properties grouped by category: Tabs, Split View, Nested Splits, Interaction, Preview, History, Policies, Animation, Session, Cross-Host Drag, Appearance, and Quick Actions.
