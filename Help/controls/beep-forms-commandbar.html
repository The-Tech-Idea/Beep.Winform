Overview

BeepFormsCommandBar is a button strip that provides top-level form commands: a block selector (popup listing all registered blocks) and a sync button. It binds to a BeepForms host and reacts to state changes, enabling or disabling buttons dynamically.

Key Features

  • Block selector popup — Dropdown button listing all blocks from the bound BeepForms host; selecting a block calls SwitchToBlockAsync
  • Sync command — Sync button calls SyncFromManager() to refresh the form state from FormsManager
  • Configurable buttons — The CommandButtons flags enum controls which buttons are visible (BlockSelector, Sync, or All)
  • Flow direction control — CommandFlowDirection sets the layout direction of the button row
  • Auto-bind to host — Automatically resolves a nearby BeepForms in the control hierarchy

No public events are defined. The control communicates through the bound BeepForms host's methods (SwitchToBlockAsync, SyncFromManager).

Theming

BeepFormsCommandBar sets UseThemeColors = true and passes the current Theme to each created BeepButton. The FlowLayoutPanel uses a transparent background inheriting from the parent. Button states (enabled/disabled) are updated dynamically based on the bound host's state.

Examples

// Command bar with only the block selector
var commandBar = new BeepFormsCommandBar
{
    Dock = DockStyle.Top,
    CommandButtons = BeepFormsCommandBarButtons.BlockSelector,
    FormsHost = myBeepForms,
    AutoBindFormsHost = false
};

// Command bar with sync button flowing right-to-left
var commandBar2 = new BeepFormsCommandBar
{
    CommandButtons = BeepFormsCommandBarButtons.Sync,
    CommandFlowDirection = FlowDirection.RightToLeft
};
this.Controls.Add(commandBar2);

Architecture

The control wraps a FlowLayoutPanel that hosts BeepButton instances created dynamically by RefreshCommandStrip(). Buttons are created or destroyed based on the CommandButtons flags value. The block selector button uses PopupMode = true with a BindingList<SimpleItem> sourced from _formsHost.Blocks. Button widths are measured via TextRenderer.MeasureText. The internal flow panel is hidden when no buttons are configured.
