Overview

BeepFormsPersistenceShelf is a compact button bar providing commit and rollback actions for a BeepForms host. Both buttons are dirty-state-aware: they are enabled only when the bound host's view state indicates pending changes (IsDirty).

Key Features

  • Commit — Calls CommitFormAsync() on the bound BeepForms host; enabled only when IsDirty
  • Rollback — Calls RollbackFormAsync() on the bound BeepForms host; enabled only when IsDirty
  • Configurable buttons — PersistenceButtons flags enum controls which buttons are shown
  • Flow direction control — PersistenceFlowDirection sets the layout direction
  • Auto-bind to host — Automatically discovers a nearby BeepForms in the parent hierarchy

Theming

BeepFormsPersistenceShelf uses UseThemeColors = true and passes the current Theme to each created BeepButton. Button widths are auto-measured via TextRenderer.MeasureText with a minimum of 90px (Commit) and 96px (Rollback). Shadows are disabled on all buttons to keep the shelf compact.

Examples

// Full persistence shelf with auto-bind
var persistenceShelf = new BeepFormsPersistenceShelf
{
    Dock = DockStyle.Top,
    PersistenceButtons = BeepFormsPersistenceShelfButtons.All
};
this.Controls.Add(persistenceShelf);

// Commit-only shelf, explicit host
var commitOnly = new BeepFormsPersistenceShelf
{
    PersistenceButtons = BeepFormsPersistenceShelfButtons.Commit,
    FormsHost = myBeepForms,
    AutoBindFormsHost = false
};

Architecture

The control wraps a FlowLayoutPanel that is rebuilt by RefreshPersistenceShelf() whenever the PersistenceButtons flags change. The panel is hidden when no buttons are configured. State updates are driven by UpdatePersistenceShelfState(), which checks _formsHost.ViewState.IsDirty to enable or disable both buttons. Button click handlers invoke async methods with ConfigureAwait(true) to return to the UI context.
