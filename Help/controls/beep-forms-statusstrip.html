Overview

BeepFormsStatusStrip is a multi-line status display that renders six information lines from a BeepForms host: status (with bootstrap state prefix), current message, coordination summary, workflow history, savepoint state, and alert state. Each line can be independently shown or hidden, and colors encode message severity.

Namespace: TheTechIdea.Beep.Winform.Controls.Integrated.Forms
Assembly: TheTechIdea.Beep.Winform.Controls.Integrated.dll
Inheritance: BaseControl → BeepFormsStatusStrip
Designer: BeepFormsStatusStripDesigner

Uses a six-row TableLayoutPanel whose row heights collapse to zero when lines are hidden or empty. The overall height dynamically adjusts based on the number of visible lines. Bootstrap state (Running, PartialSuccess, Failed) is prepended to the status line.

Key Features

  • Six status lines — Status, Message, Coordination, Workflow (multiline with history), Savepoint, and Alert
  • Independent visibility — Each line controlled by a Show*Line boolean property
  • Severity color coding — Green for success, orange for warning, red for error, black/default for informational
  • Bootstrap state prefix — Automatic prefix on the status line: [Loading…], [Partial], or [Load Failed]
  • Workflow history — Displays recent workflow entries in a rolling surface, with configurable visible count
  • Dynamic height — Control height recalculated from visibleLines * 18 + 16 with a minimum of 36px

Theming

BeepFormsStatusStrip uses UseThemeColors = true and delegates theming to child BeepLabel controls created by CreateLineLabel(). Row heights are calculated at 18px per line. The workflow line supports multiline with word wrap and auto-height via GetPreferredSize. Message severity colors map as: Success → ForestGreen, Warning → DarkOrange, Error → Firebrick, default → Black.

Examples

// Full status strip showing all 6 lines
var statusStrip = new BeepFormsStatusStrip
{
    Dock = DockStyle.Bottom,
    WorkflowHistoryVisibleCount = 5,
    FormsHost = myBeepForms
};
this.Controls.Add(statusStrip);

// Minimal status strip: only status and workflow
var minimal = new BeepFormsStatusStrip
{
    ShowStatusLine = true,
    ShowMessageLine = false,
    ShowCoordinationLine = false,
    ShowWorkflowLine = true,
    ShowSavepointLine = false,
    ShowAlertLine = false,
    WorkflowHistoryVisibleCount = 3
};

Architecture

The control contains a six-row TableLayoutPanel with one BeepLabel per row. UpdateFromViewState() reads the bound host's BeepFormsViewState and populates each label from properties like StatusText, CurrentMessage, CoordinationText, WorkflowHistory, SavepointText, and AlertText. The workflow line uses BuildWorkflowHistoryText() which formats entries as HH:mm SEVERITY message (where SEVERITY is ERR/WARN/OK/INFO/NOTE). Workflow severity is resolved as the worst severity among visible history entries. The status line prepends a bootstrap prefix based on BootstrapState.
