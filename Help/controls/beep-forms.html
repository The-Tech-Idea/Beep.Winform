Overview

BeepForms is a non-visual Control-derived coordinator that implements IBeepFormsHost. It manages a collection of IBeepBlockView instances (typically BeepBlock controls) and orchestrates them over a FormsManager (IUnitofWorksManager). It supports definition-driven form configuration, automatic block creation from definitions, active block tracking, and async bootstrap operations.

Defined as a partial class spanning multiple files. It manages the full block lifecycle from registration through sync, including master-detail shell context updates and unit-of-work event subscriptions.

Key Features

  • Block host coordinator — Registers, unregisters, and manages a collection of IBeepBlockView instances
  • Definition-driven configuration — BeepFormsDefinition drives automatic block materialization and form layout
  • Auto-create blocks — When AutoCreateBlocksFromDefinition is true, hosted BeepBlock controls are created automatically from the definition
  • Async bootstrap — InitializeAsync() bootstraps all blocks via IUnitofWorksManager.SetupBlockAsync()
  • Active block tracking — Maintains ActiveBlockName and raises ActiveBlockChanged events
  • FormsManager integration — Attaches/detaches from IUnitofWorksManager with full event subscription management
  • View state — Exposes BeepFormsViewState including BootstrapState (Running/Succeeded/PartialSuccess)
  • Design-time support — BeepFormsHostDesigner with custom BeepFormsDefinitionEditor for the Definition property

Theming

BeepForms is a non-visual coordinator and does not participate directly in theming. It uses double-buffered painting with a transparent background. The hosted BeepBlock controls handle their own theming through BaseControl.

Examples

// Create and configure BeepForms coordinator
var beepForms = new BeepForms
{
    FormName = "CustomerForm",
    AutoCreateBlocksFromDefinition = true,
    Definition = new BeepFormsDefinition
    {
        FormName = "CustomerForm",
        Blocks = new List<BeepBlockDefinition>
        {
            new BeepBlockDefinition
            {
                BlockName = "CustomerList",
                Caption = "Customers",
                Entity = new BeepBlockEntityDefinition
                {
                    ConnectionName = "MainDB",
                    EntityName = "Customers",
                    IsMasterBlock = true
                }
            },
            new BeepBlockDefinition
            {
                BlockName = "CustomerOrders",
                Caption = "Orders",
                Entity = new BeepBlockEntityDefinition
                {
                    ConnectionName = "MainDB",
                    EntityName = "Orders"
                }
            }
        }
    },
    Dock = DockStyle.Fill
};

// Set the FormsManager
beepForms.FormsManager = myUnitOfWorkManager;

// Bootstrap all blocks asynchronously
bool success = await beepForms.InitializeAsync();

// Handle events
beepForms.ActiveBlockChanged += (s, e) =>
    Console.WriteLine($"Active block: {beepForms.ActiveBlockName}");

beepForms.BootstrapCompleted += (s, e) =>
    Console.WriteLine($"Bootstrap result: {e.State}");

// Add to form
this.Controls.Add(beepForms);

Architecture

BeepForms is a partial class implementing IBeepFormsHost. It uses a BeepFormsManagerAdapter to bridge the IUnitofWorksManager with the block views. The BeepFormsViewState tracks active block name, bootstrap state, and navigation context. The BeepFormsCommandRouter handles cross-block command routing. A BeepFormsMessageService provides inter-block notification services.

When Definition is set, RebuildBlocksFromDefinition() creates BeepBlock controls via IDesignerHost.CreateComponent() at design time, or programmatically at runtime. The partial class pattern separates the public API facade from the internal layout, management, and design-time logic across multiple partial files.
