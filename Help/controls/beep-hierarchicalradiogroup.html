BeepHierarchicalRadioGroup

A hierarchical radio group control with tree-like expand/collapse support, multiple render styles, and bidirectional keyboard navigation

Overview

BeepHierarchicalRadioGroup extends BaseControl to provide a radio list with parent-child tree nesting. Items are defined via SimpleItem objects whose Children property creates the hierarchy. Expand/collapse toggles, indented tree lines, and DPI-aware rendering are built in.

Namespace: TheTechIdea.Beep.Winform.Controls.RadioGroup  |  Assembly: TheTechIdea.Beep.Winform.Controls.dll
Inheritance: Component → Control → BaseControl → BeepHierarchicalRadioGroup

Key Features

  • Hierarchical data model — Uses SimpleItem.Children for arbitrary-depth nesting via a flattened visible-item list
  • Expand/collapse — Toggle parent nodes with mouse clicks on expander buttons, or with Right/Left keyboard arrows
  • Tree lines — Dashed vertical and horizontal lines visually connect child items to their ancestors
  • Multiple render styles — Flat, Material, Card, Chip, and Circular via pluggable IRadioGroupRenderer providers
  • Single and multiple selection — Controlled by AllowMultipleSelection; state managed by RadioGroupStateHelper
  • Keyboard navigation — Arrow keys move focus; Right expands, Left collapses or moves to parent
  • Auto-size — Automatically calculates the control size based on flattened item layout
  • Image support — MaxImageSize property; renderers implementing IImageAwareRenderer draw item icons
  • Style/color profiles — RadioGroupStyleConfig and RadioGroupColorConfig for fine-grained appearance control
  • Accessibility — Full AccessibleObject metadata including selected count, expanded count, and focused item

Theming

All renderers automatically respect the active Beep theme. Colors, fonts, and border styles are sourced from _currentTheme tokens. When UseThemeColors is false, the ColorProfile properties override theme defaults.

// Apply a dark theme, all hierarchies update automatically
BeepThemesManager.ApplyTheme("MaterialDark");

Examples

Two-Level Hierarchy

var group = new BeepHierarchicalRadioGroup();

var europe = new SimpleItem { Text = "Europe" };
europe.Children.Add(new SimpleItem { Text = "France" });
europe.Children.Add(new SimpleItem { Text = "Germany" });

var asia = new SimpleItem { Text = "Asia" };
asia.Children.Add(new SimpleItem { Text = "Japan" });
asia.Children.Add(new SimpleItem { Text = "India" });

group.RootItems.Add(europe);
group.RootItems.Add(asia);

Chip-Style Selection with Events

var group = new BeepHierarchicalRadioGroup
{
    RenderStyle = RadioGroupRenderStyle.Chip,
    ItemSpacing = 8
};
group.ItemClicked += (s, e) =>
    Console.WriteLine($"Clicked: {e.Item.Text}");
group.SelectionChanged += (s, e) =>
    Console.WriteLine($"Selection changed: {e.SelectedItems?.Count} item(s)");

Programmatic Expand/Collapse and Search

// Collapse all, then expand only the "Europe" branch
group.CollapseAll();
var node = group.FindItem("europe-guid");
if (node != null) node.IsExpanded = true;
group.UpdateHierarchy(); // manual invalidation after direct property change

Architecture

ComponentResponsibility
RadioGroupLayoutHelperCalculates item rectangles, indentation offsets, and total size for auto-sizing
RadioGroupHitTestHelperMouse hit detection, hover/press/focus index tracking, expander click routing
RadioGroupStateHelperSelection state management, single/multi toggle, selected value lookups
IRadioGroupRenderer implementationsVisual rendering per style (Flat, Material, Card, Chip, Circular)
IImageAwareRendererOptional interface for renderers that support image painting from ImagePath
RadioGroupStyleConfigStyle defaults: item height, spacing, padding, control style
RadioGroupColorConfigColor overrides for background, text, and sub-items when not using theme colors

The control maintains three parallel lists: _rootItems (user data), _flattenedItems (visible-for-rendering), and _itemStates (hover/select/focus per index). UpdateHierarchy() flattens the tree respecting IsExpanded flags, then recalculates layout, states, and hit-test data in a single pass.
