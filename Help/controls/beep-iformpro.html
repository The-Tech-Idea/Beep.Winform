Overview

BeepiFormPro is the Beep framework's flagship application shell form, replacing System.Windows.Forms.Form. It renders a fully custom chrome (title bar, borders, caption buttons) via a painter pipeline and supports 26+ visual styles.

Namespace: TheTechIdea.Beep.Winform.Controls.Forms.ModernForm
Assembly: TheTechIdea.Beep.Winform.Controls.dll
Inheritance: Component → Control → ScrollableControl → ContainerControl → Form → BeepiFormPro
Implements: IFormStyle

The class is organized across 11 partial files: Core, Drawing, Theme, Win32, Backdrop, Painters, Managers, Events, ExternalDrawing, Models, and the main entry file.

Key Features

  • Custom window chrome — Fully owner-drawn caption bar, borders, and caption buttons (close, min, max, theme, style, profile, search)
  • Painter pipeline — 26+ form painters covering Material, Fluent, Office, Dark, and more styles via FormStyle enum
  • DPI-aware — AutoScaleMode.Dpi with proper scaling across 100-200% displays
  • Hit-test region system — Custom caption and border hit areas via BeepiFormProHitAreaManager
  • Caption customization — Toggleable search box, theme/style/profile buttons, custom action button
  • Backdrop effects — Acrylic and Mica backdrop support on Windows 11
  • Accessibility — High contrast mode, screen reader support, caption keyboard navigation
  • Animation system — Smooth resize, hover, focus, and window-state transitions
  • Design-time safe — Detects DesignMode and switches to simplified painting for VS designer
  • Auto-theme cascade — Child BaseControl descendants automatically receive the form’s FormStyle

Caption Bar Customization

The caption bar is fully owner-drawn and can host toggleable widgets:

  • Theme button — cycles through available themes
  • Style button — cycles through FormStyle options
  • Profile button — for user-menu integration
  • Search box — inline text input
  • Custom action button — application-specific action
  • Custom regions — registered via AddRegion() for toolbar-like overlays
// Configure caption bar widgets
myForm.ShowCaptionBar = true;
myForm.ShowThemeButton = true;
myForm.ShowStyleButton = true;
myForm.ShowSearchBox = true;

// Add a custom interactive region (e.g., a toolbar icon)
var region = new FormRegion
{
    Id = "custom-tool",
    IsInteractive = true,
    Dock = RegionDock.Caption,
    Bounds = new Rectangle(10, 4, 32, 32)
};
myForm.AddRegion(region);

Backdrops

On supported Windows 10 (build 1903+) and Windows 11, BeepiFormPro can enable system backdrop effects:

  • Acrylic — Semi-transparent blur effect (EnableAcrylicForGlass = true)
  • Mica — Windows 11’s desktop-tinted material (EnableMicaBackdrop = true)

These require FormBorderStyle.None and the custom rendering pipeline to function correctly. The backdrop is applied automatically during OnHandleCreated.

Accessibility

  • High contrast — Set HighContrastMode = true to render using system high-contrast colors
  • Screen reader — Set ScreenReaderSupport = true to expose caption regions and buttons via UIA properties
  • Keyboard navigation — Caption buttons are focusable and operable via keyboard when the caption bar has focus

Theming

The FormStyle property drives the active painter. Built-in styles include:

// Set a specific form style
myForm.FormStyle = FormStyle.Material3;      // Material Design 3
myForm.FormStyle = FormStyle.FluentDark;      // Fluent dark
myForm.FormStyle = FormStyle.Office365;       // Office 365
myForm.FormStyle = FormStyle.ModernDark;      // Modern dark
myForm.FormStyle = FormStyle.VSCode;          // VS Code-inspired

// Apply a named theme from the theme manager
myForm.Theme = "HighContrastTheme";           // Switch to a registered theme

When UseThemeColors is true, the active theme’s tokens drive background, border, text, button, and shadow colors. When false, a built-in fallback palette is used.

Child controls that inherit from BaseControl automatically receive the form’s style during ControlAdded, ensuring visual consistency across the form.

Examples

Basic Usage

// Create a themed form with custom chrome
var form = new BeepiFormPro
{
    Text = "My Application",
    FormStyle = FormStyle.Material3,
    Theme = "DefaultTheme",
    ShowSearchBox = true,
    ShowThemeButton = true,
    Size = new Size(1024, 768)
};

form.SearchBoxTextChanged += (s, query) =>
{
    Console.WriteLine($"Search: {query}");
};

Application.Run(form);

Design-Time Setup

// Partial class for designer.cs integration
partial class MainForm
{
    private void InitializeComponent()
    {
        this.beepiFormPro1 = new BeepiFormPro();
        this.SuspendLayout();

        // beepiFormPro1
        this.beepiFormPro1.FormStyle = FormStyle.FluentLight;
        this.beepiFormPro1.ShowSearchBox = true;
        this.beepiFormPro1.ShowThemeButton = true;
        // ...

        this.ResumeLayout(false);
    }
}

Child Control Auto-Styling

form.FormStyle = FormStyle.VSCode;

// When you add any BaseControl descendant, it auto-receives the form's style:
var textBox = new BeepTextBox { Text = "Hello" };
var button = new BeepButton { Text = "Submit" };
form.Controls.Add(textBox);
form.Controls.Add(button);
// Both controls now render in VSCode style automatically

Mica Backdrop (Windows 11)

var form = new BeepiFormPro
{
    Text = "Mica Demo",
    FormStyle = FormStyle.Material3,
    EnableMicaBackdrop = true,        // Windows 11 only
    BackColor = Color.Transparent      // Required for Mica
};

Designer: BeepiFormProDesigner provides smart-tag actions for toggling caption widgets and selecting the form style at design time. In design mode the form uses a simplified paint path (no double buffering) to ensure reliable designer rendering.

DPI scaling: The form uses AutoScaleMode.Dpi with an explicit AutoScaleDimensions of 96×96. All painter metrics are generated at the current DPI scale via FormPainterMetrics.DefaultForCached().
