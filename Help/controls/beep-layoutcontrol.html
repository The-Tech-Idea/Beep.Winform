Overview

BeepLayoutControl is a container control that renders pre-built layout templates with Beep styling and theme support. It inherits from BaseControl and resides in TheTechIdea.Beep.Winform.Controls.Layouts. A TemplateType enum determines which layout to build: Invoice, Product, Profile, Report, VerticalStack, HorizontalStack, Grid, SplitContainer, or Dock. The control uses specialized LayoutHelper classes (e.g., InvoiceLayoutHelper, GridLayoutHelper) to construct the internal layout controls dynamically based on LayoutOptions.

Key Features

  • 9 pre-built layout templates: Invoice, Product, Profile, Report, VerticalStack, HorizontalStack, Grid, SplitContainer, Dock
  • Configurable Grid layout with Row and Column counts
  • LayoutOptions object for theming and styling configuration (auto-created from control settings)
  • Event notifications on layout build and template change
  • Builds layout automatically in non-design mode via OnCreateControl
  • Rounded border rendering with theme-aware color via BeepStyling
  • Exception-safe layout building with design-mode debug logging

Theming

The control paints a background using BeepStyling.PaintStyleBackground with the current ControlStyle on each OnPaint call. A subtle border is drawn using a theme color from BeepStyling.GetThemeColor("Border"). The LayoutOptions.FromControl(this) factory method automatically captures theme and style settings from the control to pass to layout helper classes. The ControlStyle property is inherited from BaseControl and not explicitly redefined on this control.

Examples

// Grid layout with custom dimensions
var gridLayout = new BeepLayoutControl
{
    Template = BeepLayoutControl.TemplateType.Grid,
    GridRows = 4,
    GridColumns = 3,
    Size = new Size(800, 600),
    ControlStyle = BeepControlStyle.Material3,
    Dock = DockStyle.Fill
};

gridLayout.LayoutBuilt += (s, e) =>
{
    Console.WriteLine("Grid layout built successfully");
};

gridLayout.LayoutChanged += (s, e) =>
{
    Console.WriteLine("Template type changed");
};

this.Controls.Add(gridLayout);

// Invoice layout
var invoiceLayout = new BeepLayoutControl
{
    Template = BeepLayoutControl.TemplateType.Invoice,
    Size = new Size(700, 900)
};
invoiceLayout.BuildLayout();
this.Controls.Add(invoiceLayout);

// Split container layout
var splitLayout = new BeepLayoutControl
{
    Template = BeepLayoutControl.TemplateType.SplitContainer,
    Size = new Size(600, 400)
};
this.Controls.Add(splitLayout);

Architecture

BeepLayoutControl uses a strategy pattern where BuildLayout() delegates to dedicated static helper classes (InvoiceLayoutHelper, ProductLayoutHelper, ProfileLayoutHelper, ReportLayoutHelper, VerticalStackLayoutHelper, HorizontalStackLayoutHelper, GridLayoutHelper, SplitContainerLayoutHelper, DockLayoutHelper) based on the TemplateType enum. Each helper's Build() method receives the control instance and LayoutOptions, constructing the internal control tree. On template change, existing controls are cleared via Controls.Clear() before rebuilding. GridRows and GridColumns only trigger rebuild when Template == TemplateType.Grid, optimizing for other templates. Layout-building is wrapped in SuspendLayout/ResumeLayout for performance and is exception-safe with design-mode debug output.
