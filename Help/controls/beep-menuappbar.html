Overview

BeepMenuAppBar extends BeepMenuBar to provide an application-level menu bar with dynamic menu creation driven by the Beep configuration system. It integrates with the IBeepService infrastructure and uses DynamicMenuManager to create combined menu items from configuration data. The control supports configurable menu names for switching between different menu configurations at runtime.

Namespace: TheTechIdea.Beep.Winform.Controls.MenuBar
Assembly: TheTechIdea.Beep.Winform.Controls.Integrated.dll
Inheritance: BeepMenuBar → BeepMenuAppBar
Display Name: "Beep Menu App Bar"

This control builds on top of BeepMenuBar's rendering and layout capabilities, adding configuration-driven menu population via DynamicMenuManager.CreateCombinedMenuItems().

Key Features

  • Dynamic menu creation — CreateMenuItems() populates the menu bar from combined menu sources via DynamicMenuManager
  • Named menu support — CreateMenuItems(string menuname) changes the current menu context and rebuilds items
  • Beep service integration — Constructor accepts IBeepService for configuration and DME editor access
  • Error handling — Returns IErrorsInfo with Errors.Ok or Errors.Failed and exception details

Events

BeepMenuAppBar does not define any new events directly. It inherits all events from BeepMenuBar, including menu item click events, selection events, and base control events.

Theming

BeepMenuAppBar inherits its theming entirely from BeepMenuBar, which extends BaseControl. The base BeepMenuBar provides full Material3 theming with support for menu item colors, hover effects, active states, separator styling, and font management through the Beep theme system.

Examples

// Create a BeepMenuAppBar with Beep service integration
var menuBar = new BeepMenuAppBar(beepService)
{
    Dock = DockStyle.Top
};

// Create menu items using the default "Beep" configuration
var result = menuBar.CreateMenuItems();

if (result.Flag == Errors.Ok)
{
    Console.WriteLine("Menu items created successfully");
}
else
{
    Console.WriteLine($"Failed to create menu items: {result.Message}");
}

// Switch to a different menu configuration
menuBar.CreateMenuItems("AdminMenu");

// Create without service (uses parameterless constructor)
var standaloneMenu = new BeepMenuAppBar();
standaloneMenu.beepServices = beepService;
standaloneMenu.CreateMenuItems("Reports");

// Handle inherited events from BeepMenuBar
// (menu item clicks, selection changes, etc.)
menuBar.ItemClicked += (s, e) =>
{
    Console.WriteLine($"Menu item clicked: {e.Item?.Text}");
};

// Add to form
this.Controls.Add(menuBar);

Architecture

BeepMenuAppBar is a thin extension of BeepMenuBar that adds Beep-configuration-driven menu population. It holds a reference to IBeepService (via both constructor injection and the beepServices property) and uses the service's DMEEditor indirectly through DynamicMenuManager.CreateCombinedMenuItems().

The control stores the current menu name in a private currentMenuName field (defaulting to "Beep"). The CreateMenuItems() method delegates to DynamicMenuManager which combines menu items from multiple configuration sources (Beep configuration editor, assembly classes, and global menu definitions). After adding items to the inherited MenuItems collection, the control calls Invalidate() to trigger a repaint. All errors are captured and returned via IErrorsInfo with the ErrorsInfo implementation.

The partial class declaration supports splitting additional functionality (designer integration, layout management) across separate files while keeping the primary API clean and focused.
