Grouped notification control that displays multiple similar notifications as a stack with expand/collapse and count badge

Overview

BeepNotificationGroup is a BaseControl-derived grouped notification control that aggregates similar notifications into a compact, expandable stack. It displays a colored header with icon, title, subtitle preview, expand/collapse chevron, and a count badge. Expanding reveals individual notification items with title, message, and relative timestamp.

Namespace: TheTechIdea.Beep.Winform.Controls.Notifications
Assembly: TheTechIdea.Beep.Winform.Controls.dll
Inheritance: BaseControl → BeepNotificationGroup
Toolbox: Beep Controls tab

The control uses Material3 styling with rounded corners, drop shadow, and DPI-aware scaling. It supports keyboard navigation (Enter/Space to toggle) and accessibility via AccessibleRole.Grouping.

Key Features

  • Grouped stacking — Aggregates notifications by key, type, or source into a single collapsible card
  • Expand / collapse — Click the header or chevron to reveal or hide individual notification items
  • Count badge — Circular badge shows notification count (99+ for overflow)
  • Color-coded by type — Header background adapts to NotificationType (Info, Warning, Error, Success)
  • Message preview — Shows the most recent notification message in the collapsed header
  • Relative timestamps — Each expanded item shows "now", "5m", "3h", "2d" time-ago labels
  • Hover effects — Individual items and expand button highlight on mouse hover
  • Keyboard accessible — Enter/Space to toggle expand, AccessibleRole.Grouping
  • DPI-aware — All dimensions scaled via DpiScalingHelper
  • Material3 themed — Inherits BaseControl theming with rounded corners, shadows, and border

Theming

BeepNotificationGroup inherits theme support from BaseControl via ApplyTheme(). It uses the _currentTheme to set the _textFont from BeepFontManager.ToFont(_currentTheme.BodyMedium). The notification type colors are resolved through NotificationThemeHelpers.GetColorsForType() which returns icon and border colors for each NotificationType. Contrast colors for the badge text and borders are calculated using NotificationThemeHelpers.GetContrastColor() and NotificationThemeHelpers.ShiftLuminance().

Examples

// Create and configure a notification group
var group = new BeepNotificationGroup
{
    GroupKey = "build-errors",
    GroupTitle = "Build Errors",
    GroupType = NotificationType.Error,
    Location = new Point(10, 10),
    Size = new Size(350, 60)
};

// Add notifications
group.AddNotification(new NotificationData
{
    Id = "err-1",
    Title = "CS1002: Semicolon expected",
    Message = "Line 42 in Program.cs",
    Type = NotificationType.Error,
    Source = "Build",
    Timestamp = DateTime.Now.AddMinutes(-5)
});

group.AddNotification(new NotificationData
{
    Id = "err-2",
    Title = "CS0246: Type not found",
    Message = "Line 108 in Services.cs",
    Type = NotificationType.Error,
    Source = "Build",
    Timestamp = DateTime.Now.AddMinutes(-2)
});

// Handle notification click events
group.NotificationClicked += (s, e) =>
{
    var notification = e.Notification;
    Console.WriteLine($"Notification clicked: {notification.Title}");
};

// Handle popup open/close
group.PopupOpened += (s, e) => Console.WriteLine("Group expanded");
group.PopupClosed += (s, e) => Console.WriteLine("Group collapsed");

// Add to form
this.Controls.Add(group);

Architecture

BeepNotificationGroup extends BaseControl and overrides DrawContent(Graphics g) for custom painting. The layout is calculated in RecalculateLayout() which determines _headerRect, _badgeRect, _expandButtonRect, and _contentRect regions. Notifications are stored in a private List<NotificationData> and exposed as IReadOnlyList. The control supports Material3 styling with BeepControlStyle.Material3, rounded borders, drop shadows, and DPI-aware scaling through DpiScalingHelper. Size constraints use scaled minimum (280x60) and maximum (420x600) values. The expand/collapse behavior fires PopupOpened and PopupClosed events and recalculates the control height via UpdateSize().
