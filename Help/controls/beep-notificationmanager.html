A singleton notification/toast manager with 6 animation styles, 13 layout variants, 16 visual themes, auto-grouping, scheduling, DND mode, history tracking, sound support, and action buttons.

Overview

BeepNotificationManager is a thread-safe singleton that manages the full lifecycle of toast-style notifications. It controls creation, display, animation, positioning, grouping, queuing, scheduling, and dismissal. Inspired by commercial notification managers (DevExpress, Telerik), it provides a comprehensive API for showing rich toast messages with action buttons, progress bars, icons, and custom sounds.

Notifications are rendered as borderless BeepiFormPro windows positioned relative to a target screen with smart multi-monitor detection. A pluggable painter system routes each notification's NotificationLayout and NotificationVisualStyle to one of 16 painter implementations, each offering a distinct design-system aesthetic (Material3, iOS15, Fluent2, etc.).

Key Features

Singleton Manager

Access via BeepNotificationManager.Instance. Manages all notification lifecycle: show, animate, queue, group, schedule, dismiss, and cleanup.

6 Animation Styles

Slide, Fade, SlideAndFade, Bounce, Scale, and None. Each uses a cubic ease-out easing function at 60 FPS. Bounce uses Robert Penner's bounce-out formula.

13 Layout Variants

Standard, Compact, Prominent, Banner, Toast, Elevated, Snackbar, InlineAlert, Callout, Chip, Timeline, MediaRich, ActionSheet, StatusBar, Overlay, FullWidth.

16 Visual Painter Styles

Material3, iOS15, AntDesign, Fluent2, MaterialYou, Windows11Mica, MacOSBigSur, ChakraUI, TailwindCard, NotionMinimal, Minimal, VercelClean, StripeDashboard, DarkGlow, DiscordStyle, GradientModern.

7 Screen Positions

TopLeft, TopCenter, TopRight, BottomLeft, BottomCenter, BottomRight, Center. Each stack notifications with configurable spacing and margin.

Smart Positioning

Auto-detects active monitor via cursor position or anchor form. Clamps notifications within screen working-area bounds to prevent off-screen placement.

Notification Grouping

Auto-groups similar notifications (same GroupKey or Source) when count exceeds threshold. Groups display as an expandable card with count badge.

Do Not Disturb Mode

When enabled, only notifications meeting the configured minimum priority are shown; others are queued. Configurable minimum priority threshold.

Scheduling & Recurring

Schedule notifications for future display via ScheduleNotification or create recurring notifications with ScheduleRecurringNotification. Runs on a 1-second tick timer.

Templates

Register reusable NotificationData templates by name. Show via ShowFromTemplate with optional customization lambda.

Rich Action Buttons

Each notification can carry NotificationAction[] with support for primary/secondary styling, custom colors, and per-action click handlers.

Sound System

Optional type-based system sounds (Info=Asterisk, Warning=Exclamation, Error=Hand) or custom WAV file playback via BeepNotificationSound.

History Panel

Dismissed notifications are tracked in BeepNotificationHistory, a scrollable panel with search, type filtering, read/unread status, and time-ago display.

Bulk Operations

DismissAll, DismissAllByType, DismissAllByPriority, DismissAllExceptPriority, DismissAllRead, DismissAllUnread, MarkAllRead, MarkAllUnread, PinAll, UnpinAll, Clear.

Show Methods & Convenience APIs

All Show* methods ultimately route through Show(NotificationData), which handles:

  1. DND Check: If DND is active and priority is below threshold, the notification is queued.
  2. Capacity Check: If active count exceeds MaxVisibleNotifications, the notification is queued.
  3. Grouping Check: If grouping is enabled and a matching group key exists, it's added to the group. If similarity count reaches the threshold, a new group is created.
  4. Notification Creation: A new BeepNotification form is instantiated with the data.
  5. Sound Playback: If PlaySound is true, the appropriate sound is played via BeepNotificationSound.
  6. Positioning: Target location is calculated based on position, screen, existing notifications, and groups.
  7. Animation: The notification is animated in via BeepNotificationAnimator.

Management Methods

The manager provides a full bulk-operation API. Each dismiss method removes the notification from active state, adds it to history (if tracking is on), disposes its animator, repositions remaining notifications, and dequeues the next pending notification if any exist.

Pin/Unpin and MarkRead/MarkUnread operations call Invalidate() on each affected notification to trigger a visual refresh. The RepositionNotificationsAnimated method uses SuspendLayout/ResumeLayout across all active notifications to eliminate flicker during repositioning.

Notification Templates

The template system allows pre-defining notification configurations for reuse. Templates are stored in a Dictionary<string, NotificationData>. When showing from a template, the data is deep-cloned via CloneNotificationData, which copies all primitive/string properties and shallow-copies arrays. An optional customize lambda receives the clone before display.

var errorTemplate = new NotificationData
{
    Type = NotificationType.Error,
    Priority = NotificationPriority.Critical,
    ShowCloseButton = true,
    PlaySound = true,
    Duration = 0 // persistent
};
BeepNotificationManager.Instance.RegisterTemplate("app-error", errorTemplate);

// Later, show with customization
BeepNotificationManager.Instance.ShowFromTemplate("app-error", data =>
{
    data.Title = "Database Error";
    data.Message = "Connection to database 'Orders' failed.";
});

Notification Scheduling

Scheduling uses an internal List<ScheduledNotification> and a 1-second Timer. The timer starts when the first notification is scheduled and stops when all scheduled notifications are processed. Recurring notifications automatically re-schedule themselves by adding their interval to ShowTime.

Each ScheduledNotification contains the NotificationData, the ShowTime (DateTime), an optional Interval (TimeSpan), and an IsRecurring flag.

var manager = BeepNotificationManager.Instance;

// Schedule one-time notification
manager.ScheduleNotification(
    new NotificationData { Title = "Meeting", Message = "Starts in 5 minutes", Type = NotificationType.Warning },
    DateTime.Now.AddMinutes(5)
);

// Schedule recurring notification
manager.ScheduleRecurringNotification(
    new NotificationData { Title = "Backup", Message = "Completed", Type = NotificationType.Success },
    TimeSpan.FromHours(24)
);

// Cancel all
manager.CancelAllScheduledNotifications();

Supporting Controls

BeepNotification (partial class: 4 files)

A borderless BeepiFormPro window that renders the notification visually. Configured with FormBorderStyle.None, ShowInTaskbar = false, TopMost = true, and ShowCaptionBar = false. Features:

  • Core (BeepNotification.Core.cs): Properties (NotificationData, Title, Message, NotificationType, LayoutStyle, Duration, ShowCloseButton, ShowProgressBar), constructor, form configuration, painter initialization, DPI scaling.
  • Events (BeepNotification.Events.cs): Mouse click routing for close button, action buttons, and general click zones. Keyboard navigation (Esc to dismiss, Enter/Space for primary action, 1-3 for action indices, Ctrl+M to mark read, Ctrl+P to toggle pin).
  • Methods (BeepNotification.Methods.cs): Show (starts auto-dismiss timer), Dismiss, Pause (on hover), Resume, timer ticks, mouse enter/leave for pause-on-hover.
  • Drawing (BeepNotification.Drawing.cs): UpdateNotificationVisuals (colors), RecalculateLayout (layout helpers), OnPaint (delegates to painter for background, icon, close button, title, message, actions, progress bar).

BeepNotificationHistory

A BaseControl that displays dismissed notifications in a scrollable list with search filtering, type combobox filter, read/unread status combobox filter, "Clear" and "Mark All Read" buttons. Items are drawn with type-colored backgrounds, bold titles, message previews, and time-ago timestamps. Limited to MaxHistorySize (default 100).

BeepNotificationGroup

A BaseControl that aggregates similar notifications into an expandable stacked card. Displays a header with group type-colored background, icon, title, most-recent message preview, a chevron toggle button, and a count badge. When expanded, individual notification items are rendered with hover effects and separators. Has keyboard support (Enter/Space to toggle). Designer support via BeepNotificationGroupDesigner with action lists and verb commands.

BeepNotificationAnimator

An internal class handling 6 animation types at 60 FPS with a 300ms duration. Uses cubic ease-out easing. Bounce animation implements Robert Penner's bounce-out formula. Supports AnimateShow, AnimateHide, and Stop. Manages a Timer with 1000 / 60 interval. Applies Lerp (linear interpolation) for position and opacity.

BeepNotificationSound

Internal static class for sound management. Plays SystemSounds.Asterisk for Info, SystemSounds.Exclamation for Success/Warning, SystemSounds.Hand for Error, and SystemSounds.Beep for System. Custom WAV files are played asynchronously on a background task via SoundPlayer.

Theming & Painters

The notification system uses a two-axis painter factory:

  1. Layout (geometric template): Determined by NotificationLayout — icon size, text placement, action button positioning, padding, spacing.
  2. Visual Style (rendering engine): Determined by NotificationVisualStyle — colors, shapes, shadows, border treatment, font choices.

The NotificationPainterFactory.CreatePainter(layout, visualStyle, owner, theme) method returns the appropriate INotificationPainter implementation. Layout helpers (NotificationStyleHelpers, NotificationLayoutHelpers) provide recommended padding, spacing, and icon sizes per layout. Color helpers (NotificationThemeHelpers) provide type-based color palettes, contrast color calculation, and luminance shifting.

Theme fonts are pushed via BeepFontManager.ToFont and piped through _painter.RefreshFonts(theme) which sets TitleFont, MessageFont, ButtonFont, and CaptionFont on the painter base class.

Code Examples

Example 1: Basic Usage

var manager = BeepNotificationManager.Instance;

// Simple info notification
manager.Show("Welcome", "You have successfully logged in.");

// Success
manager.ShowSuccess("File Saved", "Report has been saved to Downloads.");

// Error (persistent, requires user dismissal)
manager.ShowError("Connection Failed", "Unable to reach server at 192.168.1.100.");

// Warning
manager.ShowWarning("Low Disk Space", "Drive C: has less than 1 GB remaining.");

Example 2: Rich Notification with Actions

var manager = BeepNotificationManager.Instance;

manager.Show(new NotificationData
{
    Title = "New Message",
    Message = "John Smith sent you a file: report-q4.xlsx",
    Type = NotificationType.Info,
    Priority = NotificationPriority.High,
    Layout = NotificationLayout.MediaRich,
    VisualStyle = NotificationVisualStyle.Windows11Mica,
    PlaySound = true,
    ShowProgressBar = true,
    Duration = 10000,
    Actions = new NotificationAction[]
    {
        new NotificationAction
        {
            Id = "open",
            Text = "Open",
            IsPrimary = true,
            OnClick = (data) => Console.WriteLine($"Opening: {data.Message}")
        },
        new NotificationAction
        {
            Id = "dismiss",
            Text = "Dismiss",
            OnClick = (data) => { /* dismissed */ }
        }
    }
});

Example 3: Configure Manager & DND

var manager = BeepNotificationManager.Instance;

manager.DefaultPosition = NotificationPosition.TopRight;
manager.MaxVisibleNotifications = 3;
manager.NotificationSpacing = 8;
manager.ScreenMargin = 16;
manager.DefaultAnimation = NotificationAnimation.Bounce;
manager.SoundEnabled = true;
manager.TrackHistory = true;
manager.EnableGrouping = true;
manager.GroupingThreshold = 3;

// Anchor to a specific form
manager.AnchorForm = this; // 'this' = your MainForm
manager.SmartPositioning = true;

// Enable Do Not Disturb (only Critical passes through)
manager.DoNotDisturbMode = true;
manager.MinimumPriorityInDND = NotificationPriority.Critical;

// Disable DND
manager.DoNotDisturbMode = false;

Example 4: History Panel & Cleanup

var manager = BeepNotificationManager.Instance;

// Show history panel on your form
var historyPanel = manager.HistoryPanel;
historyPanel.Dock = DockStyle.Fill;
this.Controls.Add(historyPanel);

// Query stats
int unread = manager.UnreadCount;
int pinned = manager.PinnedCount;
int active = manager.ActiveCount;
int queued = manager.QueuedCount;

Console.WriteLine($"{unread} unread, {pinned} pinned, {active} active, {queued} queued");

// Bulk operations
manager.MarkAllRead();
manager.PinAll();
manager.DismissAllRead();
manager.Clear(); // DismissAll + ClearQueue

// Cleanup on application exit
manager.Dispose();

Example 5: Full Customization with All Layouts

var manager = BeepNotificationManager.Instance;

// Material Snackbar
manager.Show(new NotificationData
{
    Title = "Item Deleted",
    Message = "File 'notes.txt' was moved to trash.",

    Title = "Payment Received",

The notification pipeline flows as: BeepNotificationManager.Show(data) → ShowNotificationInternal → creates BeepNotification form + wires events → plays sound via BeepNotificationSound → calculates position → creates BeepNotificationAnimator → animates show → BeepNotification.Show() starts timers. On dismiss: NotificationDismissed event → DismissNotificationInternal → animates hide → CleanupNotification adds to history, removes from active list, disposes animator, repositions remaining, dequeues next.
