BeepRadioGroup is a comprehensive, owner-draw radio button group control that supports single-selection (radio) and multi-selection (checkbox) modes with 11 distinct visual render styles. It uses a pluggable renderer architecture (IRadioGroupRenderer), a layout helper for vertical/horizontal/grid/flow arrangements, a hit-test helper for precise mouse/keyboard interaction, and a state helper for selection tracking. All rendering is done via GDI+ owner-draw — no child Windows controls are created. The control provides full WCAG 2.2 accessibility support via AccessibleObject overrides, per-item disabled states, validation (IsRequired, HasError, ErrorMessage), and data-binding through SetValue/GetValue.

A companion control, BeepHierarchicalRadioGroup (1155 lines), extends the concept with tree-structured items, expand/collapse buttons, hierarchy lines, and keyboard navigation (Right to expand, Left to collapse or focus parent).

Key Features

  • 11 render styles: Material, Card, Chip, Circular, Flat, Checkbox, Button, Toggle, Segmented, Pill, Tile
  • Single and multiple selection modes: Switch between radio-button (exclusive) and checkbox (multi-select) behavior via AllowMultipleSelection
  • 4 orientation layouts: Vertical, Horizontal, Grid (with configurable ColumnCount), Flow
  • 4 alignment modes: Left, Center, Right, Stretch
  • Pluggable renderers: Each render style implements IRadioGroupRenderer with RenderItem, MeasureItem, GetContentArea, GetSelectorArea, RenderGroupDecorations, Cleanup
  • Per-item states: Each item tracks IsSelected, IsHovered, IsFocused, IsPressed, IsEnabled, IsError, AnimationProgress, Index
  • Per-item disabled support: Disable/enable individual items by text value using DisableItem, EnableItem, SetItemsEnabled, EnableAllItems, IsItemDisabled
  • Image support: Items render images from ImagePath with configurable MaxImageSize (default 24x24)
  • Validation: IsRequired property, HasError state, ErrorMessage, Validate() method returning bool
  • Keyboard navigation: Arrow keys in all orientations, Tab/Shift+Tab for focus movement, Space/Enter for selection
  • WCAG 2.2 accessibility: Custom AccessibleObject (Grouping role) with item-level BeepRadioGroupItemAccessibleObject supporting RadioButton/CheckButton roles, Bounds, HitTest, DoDefaultAction, State, Value
  • Style/color profiles: RadioGroupStyleConfig and RadioGroupColorConfig for runtime customization with expandable property editor support
  • SubText support: Items can display secondary descriptive text below the main label
  • Data-binding: SetValue accepts string, List<string>, SimpleItem, List<SimpleItem>; GetValue returns selected item(s)
  • DPI-aware rendering: Full DPI scaling via OnDpiScaleChanged override with renderer re-initialization
  • Focus-aware rendering: Renderers can implement IFocusAwareRenderer for WCAG-compliant focus rings (min 2px, 3:1 contrast)
  • Design-time support: Placeholder rendering in Visual Studio designer showing style, item count, and control info
  • IBeepUIComponent integration: Fires ValueChanged via InvokeOnValueChanged on selection change with LinkedProperty support

Theming

BeepRadioGroup applies theming through the ApplyTheme() override:

  1. Calls base.ApplyTheme() for base theme tokens (background, foreground, border)
  2. Uses RadioGroupFontHelpers.ApplyFontTheme(this, Style, _currentTheme) to resolve fonts from theme tokens based on the active Style (Material3, Flat, etc.)
  3. _isApplyingThemeFont guard prevents the TextFont setter from disabling UseThemeFont during theme application
  4. Propagates _currentTheme to the active renderer via _currentRenderer.UpdateTheme(_currentTheme)
  5. Updates all registered renderers in the _renderers dictionary by calling UpdateTheme on each
  6. Re-assigns the ItemMeasurer delegate to use the current renderer's MeasureItem
  7. Calls RequestVisualRefresh(resetLayout: true) for batched repaint

Font resolution chain: RadioGroupFontHelpers.ApplyFontTheme reads theme font tokens appropriate for the render style and BeepControlStyle. When UseThemeFont is true, the helper overrides the TextFont property from theme tokens. Setting TextFont manually disables UseThemeFont (unless _isApplyingThemeFont guard is active).

Color resolution: When UseThemeColors is true (default), the theme's SideMenuBackColor is used for the background during rendering. When false, colors from ColorProfile are applied (GroupBackgroundColor → BackColor, TextColor → ForeColor). Each renderer independently resolves its own colors from the theme based on the active ControlStyle.

DPI scaling: On OnDpiScaleChanged, all renderers receive UpdateTheme with the current theme, layout is recalculated, hit-test data is rebuilt, and a full visual refresh is requested.

Examples

Example 1: Basic Material-Style Radio Group

var radioGroup = new BeepRadioGroup
{
    Location = new Point(20, 20),
    Size = new Size(300, 200),
    RadioGroupStyle = RadioGroupRenderStyle.Material,
    Orientation = RadioGroupOrientation.Vertical,
    ItemSpacing = 8
};

radioGroup.AddItem("Option A");
radioGroup.AddItem("Option B", "icons/star.svg");
radioGroup.AddItem("Option C", null, "Recommended choice");

radioGroup.SelectionChanged += (s, e) =>
{
    Console.WriteLine($"Selected: {radioGroup.SelectedValue}");
};

this.Controls.Add(radioGroup);

Example 2: Multiple Selection with Checkbox Renderer

var multiGroup = new BeepRadioGroup
{
    AllowMultipleSelection = true,
    RadioGroupStyle = RadioGroupRenderStyle.Checkbox,
    Orientation = RadioGroupOrientation.Vertical,
    Size = new Size(300, 250)
};

multiGroup.AddItem("Feature A");
multiGroup.AddItem("Feature B");
multiGroup.AddItem("Feature C");
multiGroup.AddItem("Feature D");

// Pre-select multiple items
multiGroup.SetValue(new List<string> { "Feature A", "Feature C" });

// Get all selected values
List<string> selected = multiGroup.SelectedValues; // ["Feature A", "Feature C"]

// Toggle individual item
multiGroup.DeselectItem("Feature A");

// Select all
multiGroup.SelectAll();

Example 3: Segmented Control (Horizontal Button Group)

var segmented = new BeepRadioGroup
{
    RadioGroupStyle = RadioGroupRenderStyle.Segmented,
    Orientation = RadioGroupOrientation.Horizontal,
    ItemSpacing = 0, // Segmented controls have no gaps
    Size = new Size(400, 44)
};

segmented.AddItem("Daily");
segmented.AddItem("Weekly");
segmented.AddItem("Monthly");
segmented.AddItem("Yearly");

segmented.SelectedValue = "Daily";
segmented.ItemClicked += (s, e) =>
    Console.WriteLine($"View: {e.Item.Text}");

Example 4: Per-Item Disabling and Validation

var validatedGroup = new BeepRadioGroup
{
    IsRequired = true,
    RadioGroupStyle = RadioGroupRenderStyle.Card,
    Size = new Size(350, 220)
};

validatedGroup.AddItem("Credit Card", "icons/credit-card.svg");
validatedGroup.AddItem("PayPal", "icons/paypal.svg");
validatedGroup.AddItem("Bank Transfer", "icons/bank.svg", "2-3 business days");
validatedGroup.AddItem("Crypto", "icons/crypto.svg");

validatedGroup.DisableItem("Crypto"); // Unavailable option

if (!validatedGroup.Validate())
{
    validatedGroup.HasError = true;
    validatedGroup.ErrorMessage = "Payment method is required";
}

validatedGroup.Reset();
validatedGroup.EnableAllItems();

Example 5: Grid Layout with Toggle Renderer

var toggleGrid = new BeepRadioGroup
{
    AllowMultipleSelection = true,
    RadioGroupStyle = RadioGroupRenderStyle.Toggle,
    Orientation = RadioGroupOrientation.Grid,
    ColumnCount = 3,
    ItemSpacing = 10,
    Size = new Size(500, 180),
    MaxImageSize = new Size(32, 32)
};

toggleGrid.AddItem("Wi-Fi", "icons/wifi.svg");
toggleGrid.AddItem("Bluetooth", "icons/bluetooth.svg");
toggleGrid.AddItem("NFC", "icons/nfc.svg");
toggleGrid.AddItem("GPS", "icons/gps.svg");
toggleGrid.AddItem("Data", "icons/data.svg");
toggleGrid.AddItem("Hotspot", "icons/hotspot.svg");

toggleGrid.SetItemsEnabled(
    new[] { "Wi-Fi", "Bluetooth", "Data" }, true);
toggleGrid.SetItemsEnabled(
    new[] { "NFC", "GPS", "Hotspot" }, false);

Example 6: Custom Renderer Registration

var group = new BeepRadioGroup();
group.RegisterRenderer(RadioGroupRenderStyle.Custom, new MyCustomRadioRenderer());

Example 7: Data-Binding Integration

// Single selection binding
radioGroup.BoundProperty = "PaymentMethod";
radioGroup.SetValue("CreditCard");

// Get value from IBeepUIComponent

// Multiple selection binding

BeepHierarchicalRadioGroup

A companion control (BeepHierarchicalRadioGroup) in the same namespace extends the radio group concept with a tree-like hierarchical structure. Items use the SimpleItem.Children property (a BindingList<SimpleItem>) to define parent-child relationships. The hierarchy is flattened for rendering with DPI-aware indentation.

Key properties: RootItems (hierarchical item collection, CollectionEditor), ShowExpanderButtons (toggle +/- buttons, default true), IndentSize (pixels per level, default 20, scaled by DPI), RenderStyle (subset of 5 renderers: Material, Card, Chip, Circular, Flat). Supports single/multi selection with the same AllowMultipleSelection flag.

Key features: Recursive hierarchy flattening (respects IsExpanded on each item), dotted-line drawing for parent-child relationship visualization, expand/collapse with clickable +/- ellipses, keyboard navigation (Right arrow to expand, Left arrow to collapse or focus parent), ExpandAll() / CollapseAll() bulk operations, FindItem(string guidId) recursive search, AddRootItem(SimpleItem) and AddChildItem(SimpleItem parent, SimpleItem child) for programmatic tree construction.

Events: All standard radio group events (SelectionChanged, ItemSelectionChanged, ItemClicked, ItemHoverEnter, ItemHoverLeave) plus ItemExpandedChanged (with ItemExpandedEventArgs carrying the item and its new IsExpanded state).

Data-binding: Supports SetValue/GetValue with string, List<string>, or SimpleItem. Has StyleProfile and ColorProfile matching the flat version. DPI-aware with S()/SF() scaling helpers. Accessibility reports selected count, expanded group count, and focused item.

Renderer interface methods:

  • Initialize(BaseControl, IBeepTheme) — Setup on activation
  • RenderItem(Graphics, SimpleItem, Rectangle, RadioItemState) — Draw a single item
  • MeasureItem(SimpleItem, Graphics) — Calculate minimum item size for layout
  • GetContentArea(Rectangle) — Content bounds within item rect (excluding margins/borders)
  • GetSelectorArea(Rectangle) — Radio button/checkbox clickable area
  • RenderGroupDecorations(Graphics, Rectangle, ...) — Group-level visual effects
  • UpdateTheme(IBeepTheme) — Theme change notification
  • Cleanup() — Release GDI+ resources (fonts, pens, brushes)

Optional interfaces: IImageAwareRenderer (MaxImageSize property for image rendering), IFocusAwareRenderer (DrawFocusRing for WCAG 2.2 compliant 2px focus outlines with 3:1 contrast).

Renderers that don't support multiple selection (SupportsMultipleSelection = false: Circular, Button, Segmented) auto-switch AllowMultipleSelection to false when selected. The control handles this automatically in the RadioGroupStyle setter.
