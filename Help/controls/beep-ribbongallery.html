BeepRibbonGallery

Interactive tile-based gallery control with pinning, recent tracking, category headers, keyboard navigation, and large preview popup support

Overview

BeepRibbonGallery is a UserControl that renders a collection of SimpleItem items as interactive tile buttons within a scrolling FlowLayoutPanel. It serves as the visual component behind ribbon gallery commands—collections of related options such as document styles, color palettes, shapes, or chart types. The gallery supports compact/small and expanded/large display modes, pinning of frequently-used items, tracking of recently selected items, category-based grouping, full keyboard navigation, and an optional large preview popup dialog.

Galleries are automatically created by BeepRibbonControl during command tree building when a command node's text, display field, or name contains the word "gallery" and it has at least 2 child items. They are hosted inside a ToolStripControlHost within the ribbon group. The gallery can also be used standalone outside of a ribbon context.

The control is sealed, meaning it cannot be subclassed. Extension is achieved through configuration properties and the ItemSelected/StateChanged events.

Key Features

  • Tile-Based Layout — Items render as themed Button tiles in a wrapping FlowLayoutPanel with tile sizes computed by RibbonGalleryRenderer.GetTileSize() per density and compact mode.
  • Compact and Large Modes — Compact mode shows 4 tiles with smaller dimensions; expanded mode shows 6 tiles and optional item images. Toggle via the Compact property.
  • Automatic Overflow — When items exceed the visible tile count, a "..." button appears. Clicking it opens a ContextMenuStrip popup with pinned, recent, and category-organized sections.
  • Pinning System — Items can be pinned/unpinned via right-click context menu or via the SetPinned() method. Pinned items sort first in the tile order and display with a "*" prefix. State persisted in a HashSet<string> for O(1) lookups.
  • Recent Tracking — Automatically records the most recently selected items (up to MaxRecentItems, default 8). Recent items sort after pinned items but before remaining items in the item order.
  • Category Headers — Groups items in the "More" popup by GroupName or Category property when EnableCategoryHeaders is true. Falls back to "Items" category for items without explicit grouping.
  • Large Preview Dialog — Opens a resizable modal dialog (720x460 default) containing a full gallery with all items visible, complete pinning support, selection synchronization, and dedicated focus handling.
  • Full Keyboard Navigation — Arrow keys (Up/Down/Left/Right) navigate selection, Enter activates, Escape closes popups/dialogs, Tab moves focus with configurable TabNavigatesOutOfGallery behavior.
  • Selection Persistence — Selected item key is tracked in _selectedKey and styled with the SelectionBack color via RibbonGalleryRenderer.StyleButton().
  • Accessibility-First Design — Each tile has computed AccessibleName, AccessibleDescription (category, tooltip, shortcut, pinned/selected state), and AccessibleRole.PushButton. The gallery itself declares AccessibleRole.List.
  • State Change Propagation — Fires StateChanged whenever pinned or recent keys are modified, enabling the parent ribbon to persist gallery state in its _galleryPinnedKeys and _galleryRecentKeys dictionaries.

Theming

Calling ApplyTheme(RibbonTheme theme, RibbonDensity density) configures the gallery's visual appearance comprehensively:

Color Assignment

  • Background: Both BackColor and the internal _tilesPanel.BackColor are set to theme.GroupBack, ensuring the gallery blends seamlessly with its parent ribbon group.
  • Renderer Colors: The RibbonGalleryRenderer receives the full theme for per-tile button styling: HoverBack for mouse hover, PressedBack for pressed state, SelectionBack for selected items, Text for foreground, GroupBorder for flat button borders, and DisabledBack/DisabledText for disabled tiles.

Tile Sizing

Tile dimensions are computed by _renderer.GetTileSize(_compact), which varies by both density and compact mode. The renderer determines the pixel dimensions for each tile button based on these settings.

Image Rendering

When not in Compact mode, gallery tiles display images next to text. Images are created at density-appropriate sizes: 14px (Compact), 16px (Comfortable), or 20px (Touch). Each image is rendered to a transparent Bitmap using StyledImagePainter.PaintWithTint() with theme.IconColor as the recolor tint. All generated images are tracked in _images and disposed when tiles are cleared or the gallery is disposed.

Button Styling

Each tile button's visual style is applied by _renderer.StyleButton(button, selected) which sets BackColor, ForeColor, FlatStyle = Flat, FlatAppearance.BorderColor, FlatAppearance.MouseDownBackColor, and FlatAppearance.MouseOverBackColor based on the item's selection state and the theme colors.

Item Ordering

The GetOrderedItems() method determines the display order of gallery tiles using a three-tier sorting strategy:

  1. Pinned Items: Items whose keys are in the _pinnedKeys set appear first, in their natural order from the item collection.
  2. Recent Items: Items that are NOT pinned but appear in the _recentKeys list appear next, sorted by their recency index (0 = most recent). Items with no recent index (-1 from GetRecentIndex()) are excluded from this tier.
  3. Remaining Items: All items not already placed in the pinned or recent tiers appear last, in their original order from the item collection.

A HashSet<string> (skipKeys) accumulates keys already placed in pinned and recent tiers to avoid duplicates in the remaining items set. The final concatenation is pinned.Concat(recent).Concat(remaining).

The tile button text is augmented with a "*" prefix for pinned items via GetTileButtonText(): return _pinnedKeys.Contains(key) ? $"* {text}" : text;

Keyboard Navigation

The gallery implements a complete keyboard navigation system via the HandleKeyboardInput() method, which is wired to both the gallery's own KeyDown event and each tile button's KeyDown event:

KeyBehaviorImplementation
Right / DownMove selection forward by 1 (wraps to first from last)MoveKeyboardSelection(1)
Left / UpMove selection backward by 1 (wraps to last from first)MoveKeyboardSelection(-1)
EnterActivate the currently selected itemActivateKeyboardSelection() → SelectItem(item, true). Fires ItemSelected, registers recent, updates styles.
EscapeClose "More" popup if open; close large preview dialog if in modal gallery modeChecks _moreMenu.Visible, then checks if parent form is modal and contains "Gallery" in title.
TabWhen TabNavigatesOutOfGallery: call MoveFocusOutOfGallery(backward)Uses SelectNextControl() on parent and form to find the next valid control in tab order. Both forward and backward (Shift+Tab) supported.

The MoveKeyboardSelection() method operates on the ordered item list, decrements/increments the _keyboardSelectionIndex with wrapping, updates _selectedKey, calls UpdateTileStyles() to visually highlight the selected tile, and calls FocusSelectedTileControl() to move input focus to the selected tile button or the tile panel.

The IsInputKey() override ensures that Arrow keys, Enter, Escape, and Tab are all processed by the gallery rather than being consumed by the parent form's default navigation.

More Popup & Large Preview

"More" Popup

When the item count exceeds the visible tile count (4 in compact, 6 in expanded), a "..." button is appended to the tile panel. Clicking it calls ShowMoreMenu(), which:

  1. Clears and rebuilds the _moreMenu ContextMenuStrip.
  2. Computes pinned, recent, and others subsets of the remaining items.
  3. Adds sections via AddMoreMenuSection(): "Pinned" (if any), "Recent" (if any), then category-grouped sections (if EnableCategoryHeaders) or a single "Items" section.
  4. Each section header is a disabled ToolStripMenuItem using the group typography font.
  5. Each item is a clickable ToolStripMenuItem with right-click context menu support for pin/unpin and preview.
  6. If EnableLargePreviewPopup is true, appends a separator and "Open Gallery Preview..." item.
  7. Auto-focuses the first enabled item and shows the popup at the anchor button location.

Large Preview Dialog

The ShowLargePreviewDialog() method creates a modal form (720x460, sizable tool window) containing a child BeepRibbonGallery configured in expanded mode with all items visible:

var preview = new BeepRibbonGallery
{
    Dock = DockStyle.Fill,
    Compact = false,
    ShowAllItemsInPopup = true,
    EnableCategoryHeaders = _enableCategoryHeaders,
    EnableLargePreviewPopup = false,
    MaxRecentItems = _maxRecentItems
};
preview.SetItems(_items);
preview.SetPinnedKeys(_pinnedKeys);
preview.SetRecentKeys(_recentKeys);
preview.ApplyTheme(_theme, _density);
preview.SetSelected(GetSelectedItem());

// Selection in the preview syncs back to the parent gallery
preview.ItemSelected += (_, e) => {
    _selectedKey = GetItemKey(e.Item);
    SetPinnedKeys(preview.GetPinnedKeys());
    SetRecentKeys(preview.GetRecentKeys());
    UpdateTileStyles();
    ItemSelected?.Invoke(this, new RibbonGalleryItemSelectedEventArgs(e.Item));
    dialog.Close();
};

// State changes during preview pin/unpin sync back immediately
preview.StateChanged += (_, __) => {
    SetPinnedKeys(preview.GetPinnedKeys());
    SetRecentKeys(preview.GetRecentKeys());
};

State Management

The gallery maintains several pieces of state that can be externally persisted and restored:

Pin/Recent State Trimming

The TrimStateToCurrentItems() method ensures pinned and recent keys are always a subset of the current items. When SetItems(), SetPinnedKeys(), or SetRecentKeys() is called, this method removes any pinned or recent keys that no longer correspond to an item in the collection.

Selection Key Tracking

The _selectedKey field stores the key of the currently selected item. It is updated by SelectItem(), SetSelected(), and MoveKeyboardSelection(). The _buttonMap dictionary maps tile buttons to their SimpleItem data, enabling O(1) lookup for selection styling.

Image Resource Management

All tile images are tracked in _images and disposed in ClearTiles() and Dispose(). The DisposeImages() method iterates and disposes each Bitmap, then clears the list. This prevents GDI+ resource leaks during ribbon rebuilds.

Event Handler Cleanup

Before clearing tiles, ClearTiles() unsubscribes all Click, MouseUp, KeyDown, and PreviewKeyDown handlers from each tracked button to prevent event handler leaks.

Example 4: Programmatic Pin/Recent Management

// Read current state
var pinned = gallery.GetPinnedKeys();
var recent = gallery.GetRecentKeys();

// Programmatically pin an item
gallery.SetPinnedKeys(pinned.Append("style_h1").Distinct());

// Programmatically mark as recent (insert at position 0)
var updatedRecent = new List<string> { "style_h1" };
updatedRecent.AddRange(recent.Where(k => k != "style_h1"));
gallery.SetRecentKeys(updatedRecent);

// Persist to file (example with JSON)
var state = new { Pinned = gallery.GetPinnedKeys(), Recent = gallery.GetRecentKeys() };
File.WriteAllText("gallery-state.json",
    JsonSerializer.Serialize(state));

// Restore from file
var saved = JsonSerializer.Deserialize<SavedState>(
    File.ReadAllText("gallery-state.json"));
gallery.SetPinnedKeys(saved?.Pinned);
gallery.SetRecentKeys(saved?.Recent);

Example 5: Accessibility and Focus Management

// Configure for keyboard-driven workflows
gallery.TabNavigatesOutOfGallery = false; // Keep focus inside gallery
gallery.ChildTileTabStops = true;          // Let user Tab between tiles
gallery.TabStop = true;                    // Gallery is in the form's tab order

// Manually focus the gallery programmatically
void ShowStylePicker()
{
    gallery.Focus();
    // The gallery's Enter handler auto-focuses the selected tile
}

// Prevent the gallery from consuming Tab if in a dialog context
gallery.PreviewKeyDown += (s, e) =>
{
    if (e.KeyCode == Keys.Tab)
    {
        e.IsInputKey = true; // Allow Tab to be processed by gallery
    }
};

Gallery/RibbonGalleryRenderer.csVisual rendering engine for gallery tiles. Provides GetTileSize(bool compact) returning tile dimensions based on density and compact mode, and StyleButton(Button button, bool selected) which applies theme colors (background, foreground, border, hover, pressed, selected, disabled) to tile buttons. Exposes a Theme property for font access via BeepThemesManager.ToFont(). Used by the gallery's CreateTileButton() and UpdateTileStyles() methods.

The gallery's internal rendering surface is a FlowLayoutPanel (_tilesPanel) with DockStyle.Fill, WrapContents = true, and AutoScroll = true for overflow scrolling. Each tile is a fully styled Button with FlatStyle.Flat, custom colors via renderer, and accessibility attributes set on creation. The gallery is typically hosted inside a ToolStripControlHost within a BeepRibbonGroup in the ribbon. When used standalone, it can be placed directly on any Form or Panel.
