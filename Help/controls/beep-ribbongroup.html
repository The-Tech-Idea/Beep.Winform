BeepRibbonGroup

Density-aware command group container for ribbon tabs with adaptive sizing, button factories, and theming

Overview

NamespaceTheTechIdea.Beep.Winform.Controls
AssemblyTheTechIdea.Beep.Winform.Controls.dll
InheritanceSystem.Windows.Forms.ToolStrip
ImplementsStandard ToolStrip interfaces; custom rendered via BeepRibbonToolStripRenderer

BeepRibbonGroup is a ToolStrip-derived control that acts as a command container within a ribbon tab. Each ribbon tab page typically hosts multiple groups laid out horizontally, with each group displaying a title and a collection of commands rendered as large (image-above-text) or small (image-before-text) buttons.

Groups are created automatically by BeepRibbonControl.AddGroup() during the ribbon build process. They are rendered with the BeepRibbonToolStripRenderer (a nested private class inside BeepRibbonControl) which delegates to BeepRibbonRenderer for custom painting of backgrounds, interactive item states, and separators.

The group extends ToolStrip rather than UserControl, which means it natively supports ToolStripItem children (buttons, separators, dropdowns, control hosts) and can be hosted within the parent TabPage via standard Controls.Add(). The control sets Dock = DockStyle.Top and GripStyle = Hidden in its constructor, making it stack vertically within its tab page container.

Key Features

  • Density-Aware Sizing — Automatically adjusts height and large button widths when Density changes. Uses a switch expression across Comfortable (48px), Compact (40px), and Touch (56px) density values.
  • Large and Small Button Factories — AddLargeButton() creates fixed-width buttons with ImageScaling.None and TextImageRelation.ImageAboveText for iconic command buttons. AddSmallButton() creates auto-sized buttons with ImageScaling.SizeToFit and TextImageRelation.ImageBeforeText for toolbar-style compact layouts.
  • Theme-Driven Dynamic Spacing — ApplyMetrics() recalculates item margins, separator spacing, and padding from the theme's ItemSpacing and GroupSpacing tokens, scaled by the current density.
  • Touch Target Compliance — Configurable MinTouchTargetWidth (default 44px, minimum 32px) ensures accessibility-friendly minimum hit areas for all interactive items.
  • Popup State Tracking — Exposes IsPopupOpen boolean and raises PopupOpened / PopupClosed events for parent ribbon awareness of child popup lifecycle.
  • Seamless Custom Rendering — Connected to the BeepRibbonRenderer rendering engine via BeepRibbonToolStripRenderer for consistent background surfaces, button hover/press/selected states, separator lines, and text coloring.
  • Stretched Layout — The Stretch = true property causes the group to fill available horizontal space within its tab page, ensuring groups occupy equal widths.

Density Profiles

The group height and large button width adapt to the RibbonDensity enum value via computed properties:

DensityGroup HeightLarge Button WidthVertical PaddingUse Case
Comfortable48px72px2pxStandard desktop use with mouse input. Balanced spacing for readability and click accuracy.
Compact40px64px1pxSpace-constrained UIs, dense data applications, or secondary monitors. Minimal padding maximizes content area.
Touch56px84px4pxTablet, kiosk, or touch-screen deployments. Generous padding ensures comfortable finger targeting.

When Density changes, ApplyDensity() iterates all existing items. For each ToolStripButton where TextImageRelation == ImageAboveText (large button), it updates the width to the new density value. All buttons have their height set to the new group height. Separator margins are also recalculated via the vertical padding value.

Theming

Calling ApplyTheme(RibbonTheme theme) configures the group's appearance from the RibbonTheme color and typography tokens:

  • BackColor ← theme.GroupBack — The background color for the group's tool strip surface.
  • ForeColor ← theme.Text — The default text color for all items in the group.
  • Font ← BeepThemesManager.ToFont(theme.CommandTypography) — The font resolved from the CommandTypography typography style token (Segoe UI, 8.5pt by default).

The group's Renderer is typically set to a BeepRibbonToolStripRenderer instance (done automatically by BeepRibbonControl.AddGroup()). This ensures custom painting for:

  • ToolStrip background — Filled with theme.GroupBack via BeepRibbonRenderer.DrawToolStripSurface().
  • Interactive button states — Hover (theme.HoverBack), pressed (theme.PressedBack), and selected/checked (theme.SelectionBack) backgrounds.
  • Separator lines — Vertical or horizontal lines in theme.Separator color.
  • Text rendering — Enabled text in theme.Text, disabled text in theme.DisabledText.

Metrics & Spacing

The ApplyMetrics() method is the central spacing calculator, called after theme changes, density changes, and button additions. It computes:

  • Padding: Uses ItemSpacing (horizontal) from the theme and a density-dependent vertical padding value (1/2/4px for Compact/Comfortable/Touch).
  • Separator margins: Horizontal margin = GroupSpacing / 2 (minimum 2). Creates visual separation between command clusters.
  • Item margins: Horizontal margin = ItemSpacing / 2 (minimum 1). Provides consistent spacing between adjacent buttons.
private void ApplyMetrics()
{
    int itemSpacing = Math.Max(1, _theme?.ItemSpacing ?? 4);
    int groupSpacing = Math.Max(itemSpacing + 1, _theme?.GroupSpacing ?? 8);
    int verticalPadding = _density switch
    {
        RibbonDensity.Compact => 1,
        RibbonDensity.Touch => 4,
        _ => 2
    };
    Padding = new Padding(itemSpacing, verticalPadding, itemSpacing, verticalPadding);
    foreach (ToolStripItem item in Items)
    {
        if (item is ToolStripSeparator separator)
        {
            int h = Math.Max(2, groupSpacing / 2);
            separator.Margin = new Padding(h, Math.Max(2, verticalPadding + 2),
                                          h, Math.Max(2, verticalPadding + 2));
        }
        else
        {
            int h = Math.Max(1, itemSpacing / 2);
            item.Margin = new Padding(h, 1, h, 1);
        }
    }
}

This ensures consistent spacing regardless of which density or theme is active, and automatically updates all existing items when configuration changes.

Button Lifecycle

Buttons created by AddLargeButton() and AddSmallButton() follow a consistent lifecycle:

  1. Creation: A new ToolStripButton is instantiated with the provided text and optional image. Scaling, text-image relationship, and auto-size are set per the factory method.
  2. Addition: The button is added to Items collection, which triggers the ToolStrip layout engine.
  3. Metrics Application: ApplyMetrics() is called to set the correct margins on all items including the new button.
  4. Configuration: The calling code (typically BeepRibbonControl.Commands.cs) configures further properties like Enabled, Visible, ToolTipText, Tag, Font, Click handler, and accessibility attributes via ConfigureCommandItem().
  5. Rendering: On each paint cycle, BeepRibbonToolStripRenderer paints the button using BeepRibbonRenderer.DrawInteractiveItem() with the current theme colors and interaction state (hovered, pressed, selected, enabled, disabled).
  6. Density/Theme Changes: If Density or the theme changes, all existing large buttons are resized and all items have their margins recalculated.
  7. Disposal: Buttons are cleaned up when the ribbon rebuilds its command tree (ClearRibbonTree()) by disposing images and clearing items.

Popup State Tracking

The group exposes a simple popup tracking mechanism:

  • A boolean _popupOpen field tracks internal state.
  • The IsPopupOpen property setter only fires events when the value actually changes, avoiding redundant notifications.
  • The CloseChildPopup() method provides a clean API for dismiss purposes.
  • Typical usage in parent ribbon code: subscribe to DropDownOpened / DropDownClosed on dropdown buttons and set group.IsPopupOpen accordingly.

Examples

Example 1: Creating Groups via the Ribbon API

// Create a tab page and add groups with commands
var tabPage = ribbon.AddPage("Home");

// Clipboard group with large and small buttons
var clipboardGroup = ribbon.AddGroup(tabPage, "Clipboard");
var pasteLarge = clipboardGroup.AddLargeButton("Paste", pasteImage);
var cutSmall = clipboardGroup.AddSmallButton("Cut", cutImage);
clipboardGroup.Items.Add(new ToolStripSeparator());
var copySmall = clipboardGroup.AddSmallButton("Copy", copyImage);

// Font group with dropdown
var fontGroup = ribbon.AddGroup(tabPage, "Font");
fontGroup.AddLargeButton("Bold", boldImage);
fontGroup.AddLargeButton("Italic", italicImage);
fontGroup.AddLargeButton("Underline", underlineImage);

// Change density dynamically (e.g., in response to window size)
clipboardGroup.Density = RibbonDensity.Touch;

Example 2: Data-Driven Groups (via CommandItems)

// Build a command tree - the ribbon creates BeepRibbonGroup instances automatically
var fontCmdGroup = new SimpleItem { Text = "Font", Name = "font_group" };
fontCmdGroup.Children.Add(new SimpleItem {
    Text = "Bold", ActionID = "bold", ImagePath = "bold.png",
    IsCheckable = true
});
fontCmdGroup.Children.Add(new SimpleItem {
    Text = "Italic", ActionID = "italic", ImagePath = "italic.png",
    IsCheckable = true
});
fontCmdGroup.Children.Add(new SimpleItem {
    Text = "Underline", ActionID = "underline",
    IsCheckable = true
});

var paragraphGroup = new SimpleItem { Text = "Paragraph", Name = "para_group" };
paragraphGroup.Children.Add(new SimpleItem { IsSeparator = true });
paragraphGroup.Children.Add(new SimpleItem {
    Text = "Bullets", ActionID = "bullets", ImagePath = "bullets.png"
});
paragraphGroup.Children.Add(new SimpleItem {
    Text = "Numbering", ActionID = "numbering", ImagePath = "numbering.png"
});

var homeTab = new SimpleItem { Text = "Home", Name = "tab_home" };
homeTab.Children.Add(fontCmdGroup);
homeTab.Children.Add(paragraphGroup);
ribbon.CommandItems.Add(homeTab);

// The ribbon automatically:
// 1. Creates TabPage("Home"), BeepRibbonGroup("Font"), BeepRibbonGroup("Paragraph")
// 2. Determines large/small button layout based on available width and density
// 3. Configures checkable buttons for Bold/Italic/Underline
// 4. Normalizes separators (leading/trailing/duplicate separators removed)
// 5. Handles overflow with "More" dropdown if needed

Example 3: Monitoring Popup State

// Track popup open/close events for UI feedback
clipboardGroup.PopupOpened += (s, e) =>
{
    statusLabel.Text = "Clipboard dropdown open";
    statusLabel.ForeColor = Color.DarkBlue;
};

clipboardGroup.PopupClosed += (s, e) =>
{
    statusLabel.Text = "Ready";
    statusLabel.ForeColor = SystemColors.ControlText;
};

// Wire up dropdown buttons to the popup state tracker
var fontPickerDropdown = new ToolStripDropDownButton("Font...", fontImage);
fontPickerDropdown.DropDownOpening += (_, __) => fontGroup.IsPopupOpen = true;
fontPickerDropdown.DropDownClosed += (_, __) => fontGroup.IsPopupOpen = false;
fontGroup.Items.Add(fontPickerDropdown);

Example 4: Programmatic Theme and Density Switching

// Apply a custom theme
var customTheme = new RibbonTheme
{
    GroupBack = Color.FromArgb(245, 245, 250),
    Text = Color.FromArgb(30, 30, 40),
    GroupSpacing = 12,
    ItemSpacing = 6
};

foreach (TabPage page in ribbon.Tabs.TabPages)
{
    foreach (var group in page.Controls.OfType<BeepRibbonGroup>())
    {
        group.ApplyTheme(customTheme);
    }
}

// Switch to touch density for tablet mode
void EnterTabletMode()
{
    ribbon.Density = RibbonDensity.Touch; // Propagates to all groups automatically
    ribbon.Refresh();
}

void ExitTabletMode()
{
    ribbon.Density = RibbonDensity.Comfortable;
    ribbon.Refresh();
}

Example 5: Accessibility Configuration

// Set larger touch targets for accessibility compliance
var stylesGroup = ribbon.AddGroup(designTab, "Styles");
stylesGroup.MinTouchTargetWidth = 48; // Larger than default 44px

// Apply accessible names to buttons created via the group
var normalBtn = stylesGroup.AddLargeButton("Normal", normalImage);
normalBtn.AccessibleName = "Apply Normal paragraph style";
normalBtn.AccessibleDescription = "Resets text formatting to the Normal style";
normalBtn.AccessibleRole = AccessibleRole.PushButton;

var headingBtn = stylesGroup.AddLargeButton("Heading 1", headingImage);
headingBtn.AccessibleName = "Apply Heading 1 style";
headingBtn.AccessibleDescription = "Formats the selected text as Heading 1";

Separator Management

The group handles ToolStripSeparator items with special spacing logic in ApplyMetrics(). Separators serve as visual dividers between clusters of related commands within the group:

  • Horizontal Margin: Separators receive a horizontal margin of GroupSpacing / 2 (minimum 2px), creating clearly visible gaps between command clusters.
  • Vertical Margin: Set to verticalPadding + 2 (minimum 4px), ensuring the separator line is vertically centered within the group.
  • Rendering: The BeepRibbonToolStripRenderer handles painting separators via OnRenderSeparator(), drawing either vertical or horizontal lines in theme.Separator color. Vertical separators span from y=4 to y=Height-4; horizontal separators span from x=4 to x=Width-4.

ToolStripControlHost Support

While the group primarily hosts buttons and separators, the parent ribbon also adds gallery controls via ToolStripControlHost. This is used to embed BeepRibbonGallery (a UserControl) directly within the tool strip item layout:

var host = new ToolStripControlHost(gallery)
{
    AutoSize = false,
    Width = gallery.Width,
    Height = Math.Max(30, groupHeight - 2),
    Margin = new Padding(1),
    Padding = Padding.Empty
};
group.Items.Add(host);

Control hosts receive the standard theme font via ConfigureCommandItem() for consistent typography, and the hosted control's ApplyTheme() method is called during ribbon theme updates to keep the embedded control visually synchronized with its parent group.

Architecture

BeepRibbonGroup is implemented as a single cohesive class in one source file:

FileResponsibility
Ribbon/BeepRibbonGroup.cs
(171 lines)		Full group implementation. Constructor sets GripStyle.Hidden, DockStyle.Top, RenderMode.ManagerRenderMode, Stretch = true, AutoSize = false, and initial Height = 48. Contains density-based sizing calculations in GetLargeButtonWidth() and ApplyDensity(), button factory methods AddLargeButton() and AddSmallButton(), spacing engine ApplyMetrics(), popup state tracking with IsPopupOpen, and theme integration via ApplyTheme().

The group relies on an external renderer for visual output. The renderer is always BeepRibbonToolStripRenderer (a nested private class inside BeepRibbonControl.Renderer.cs) which extends ToolStripProfessionalRenderer and delegates all painting to the static methods BeepRibbonRenderer.DrawToolStripSurface(), BeepRibbonRenderer.DrawInteractiveItem(), and separator/text rendering overrides. This separation of concerns keeps the group focused on layout and state management while the renderer handles all visual styling.

The group is always a child of a TabPage within BeepRibbonControl._tabs. During the BuildFromSimpleItems() process, the ribbon creates groups via AddGroup(), sets their Density from the ribbon, applies the current theme, and then populates them with buttons, separators, dropdowns, and gallery control hosts based on the SimpleItem command tree. The group's Tag property stores the SimpleItem group node for later reference by the ribbon's responsive layout engine.

Construction Sequence

When BeepRibbonControl.AddGroup() creates a group, the following sequence occurs:

  1. A new BeepRibbonGroup is instantiated with its default constructor (48px height, hidden grip, top dock, stretched, non-auto-size).
  2. The group's Text property is set from the method's title parameter (displayed in the group's header).
  3. Density is set from the parent ribbon's current density value.
  4. The Renderer is assigned a new BeepRibbonToolStripRenderer instance referencing the parent ribbon.
  5. ApplyTheme() is called with the ribbon's current theme, setting colors, font, and initial metrics.
  6. The group is added to the tab page's Controls collection at index 0 (bottom of the docked stack).
  7. The ribbon's command builder then populates the group by calling BuildGroupCommands(), which adds buttons, separators, dropdowns, and gallery hosts.
