Beep Tooltip System

A fully-featured tooltip and popover framework with 20 semantic types, 7 layout variants, 8 animations, 7 built-in validators, guided-tour support, pinnable popovers, and a unified painter pipeline — all integrated with the Beep theme system.

Stable v1.0.164 TheTechIdea.Beep.Winform.Controls.ToolTips

Overview

The Beep Tooltip System is a layered, singleton-managed framework. The simplified architecture is:

  • ToolTipManager — singleton façade; create, attach, update, and dismiss tooltips with one call.
  • ToolTipConfig / PopoverConfig — immutable (or mutable) configuration objects passed to the manager.
  • ToolTipInstance — the runtime wrapper that owns the CustomToolTip window, show/hide animation, and lifetime.
  • CustomToolTip — the WinForms Form subclass rendered on screen. Delegates all drawing to an IToolTipPainter.
  • IToolTipPainter / ToolTipPainterFactory — strategy pattern; chooses the correct painter per ToolTipLayoutVariant.
  • BeepPopover / BeepPinnedTooltip — persistent non-auto-dismiss windows; extend CustomToolTip.
  • BeepTourManager / BeepTourBuilder — guided-tour engine built on top of the tooltip stack.
  • ToolTipExtensions / ToolTipBuilder — extension methods and fluent builder for attaching tooltips to any Control.

Painter Pipeline

All rendering is delegated through IToolTipPainter. ToolTipPainterFactory selects the correct painter based on ToolTipLayoutVariant.

Programmatic tooltip at cursor

private async void OnGridCellHover(object sender, CellHoverEventArgs e)
{
    var pos = grid.PointToScreen(e.CellBounds.Location);

    _hoverKey = await ToolTipManager.Instance.ShowTooltipAsync(new ToolTipConfig
    {
        Title     = e.ColumnHeader,
        Text      = e.CellValue?.ToString() ?? "(empty)",
        Type      = ToolTipType.Default,
        Duration  = 0,          // keep until cell leave
        Placement = ToolTipPlacement.Top,
        Position  = pos
    });
}

private async void OnGridCellLeave(object sender, EventArgs e)
{
    if (_hoverKey != null)
    {
        await ToolTipManager.Instance.HideTooltipAsync(_hoverKey);
        _hoverKey = null;
    }
}

Best Practices

  • Use extension methods for simple cases. control.SetTooltip("text") is always sufficient for plain hover tooltips — no manual ToolTipManager calls needed.
  • Use ToolTipBuilder for one-off styled tooltips. The fluent API avoids repeatedly constructing ToolTipConfig objects.
  • Use ToolTipType semantically. Choose Error for errors, Warning for cautions — the type drives accessible color selection including WCAG contrast enforcement (MinContrastRatio).
  • Set Duration = 0 for popovers and pinnable tooltips. Never let an interactive or pinned tooltip auto-dismiss.
  • Use PersistOnHover = true (default) for all rich tooltips. This satisfies WCAG 1.4.13 — users must be able to move the mouse over the tooltip.
  • Wire TriggerMode = Click for destructive confirmation popovers. Hover-triggered confirmation dialogs are confusing; click-triggered ones are expected.
  • Prefer LoadPreviewAsync for heavy images in Preview tooltips — the skeleton placeholder displays immediately while the image loads.
  • Call HideAllTooltipsAsync() when opening a modal dialog to prevent orphaned tooltips underneath.
  • Disable animations during tests: ToolTipManager.Instance.EnableAnimations = false;
  • Always set TriggerMode = Manual when managing lifecycle yourself (grid cells, map pins, etc.) so the manager does not interfere with your MouseEnter/Leave logic.
  • Tour: call Properties.Settings.Default.TourSeen after TourCompleted to avoid showing the tour to returning users.
