Accessibility Guide

Build accessible applications with Beep Controls

Overview

Beep Controls are designed with accessibility in mind. This guide covers how to ensure your applications are usable by everyone, including users with visual, motor, or cognitive disabilities.

Keyboard Navigation

All Beep Controls support keyboard navigation:

  • Tab / Shift+Tab — Move focus between controls
  • Enter / Space — Activate buttons and toggles
  • Arrow Keys — Navigate within lists, grids, and menus
  • Escape — Close dialogs and popups

Set the TabIndex property to control the focus order:

nameTextBox.TabIndex = 0;
emailTextBox.TabIndex = 1;
submitButton.TabIndex = 2;

Screen Reader Support

Set accessible names and descriptions for controls:

submitButton.AccessibleName = "Submit Form";
submitButton.AccessibleDescription = "Submits the registration form";

searchBox.AccessibleName = "Search";
searchBox.AccessibleRole = AccessibleRole.Text;

Color Contrast

Beep themes are designed to meet WCAG 2.1 AA contrast ratios. When creating custom themes:

  • Minimum contrast ratio of 4.5:1 for normal text
  • Minimum contrast ratio of 3:1 for large text (18px+)
  • Test your themes with a contrast checker tool
  • Never rely on color alone to convey information — use icons or text labels too

High Contrast / Large Fonts

Beep Controls automatically respect system high contrast settings. To support variable font sizes:

// Allow controls to resize with font changes
label.AutoSize = true;

// Use relative sizing where possible
panel.Padding = new Padding(
    (int)(8 * scaleFactor),
    (int)(8 * scaleFactor),
    (int)(8 * scaleFactor),
    (int)(8 * scaleFactor)
);

Focus Indicators

Beep Controls show visible focus indicators by default. Never disable them. If customizing, ensure the focus ring is clearly visible:

// BaseControl paints a focus rectangle automatically
// To customize, override in your control:
protected override void OnGotFocus(EventArgs e)
{
    base.OnGotFocus(e);
    Invalidate(); // Triggers repaint with focus indicator
}

Checklist

  • All interactive controls are reachable by keyboard
  • Focus order is logical (set TabIndex)
  • All images have AccessibleName set
  • Color is not the sole indicator of state
  • Dialog boxes can be dismissed with Escape
  • Custom themes meet WCAG contrast requirements
  • Forms work at 150% and 200% DPI scaling
