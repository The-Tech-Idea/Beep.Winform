Overview

The Beep theming system provides a unified way to style every control in your application. All BaseControl descendants automatically subscribe to the active theme and repaint themselves when the theme changes. You do not need to manually set colors on individual controls; instead, controls derive their appearance from the theme they belong to.

The theming stack has three layers:

  • IBeepTheme — the contract that defines all color and style tokens
  • BeepThemesManager — the singleton that holds the active theme and broadcasts changes
  • BaseControl — the base class that reads from the active theme and paints itself

How Themes Cascade

When a control paints itself, it does not hard-code colors. Instead, it follows a resolution chain that determines the final color or style value:

  1. Control-level override — if a property like CustomBackColor is set directly on the control, it wins.
  2. Parent container override — if a parent BeepPanel or BeepCard has a local style, it propagates to children.
  3. Active theme — the global theme registered in BeepThemesManager.
  4. Fallback defaults — hard-coded sensible defaults if no theme is set.
// Control-level override — takes precedence over the theme
myButton.CustomBackColor = Color.LightBlue;
myButton.CustomForeColor = Color.DarkBlue;

// Theme-level styling — applies to all controls NOT overridden
BeepThemesManager.ActiveTheme.BackColor = Color.WhiteSmoke;

BeepThemesManager

BeepThemesManager is the central registry. You can register themes by name, switch between them, and query the current theme. Apply a theme before any form is created: 

static void Main()
{
    ApplicationConfiguration.Initialize();

    // Register built-in themes
    BeepThemesManager.RegisterTheme("MaterialBlue", new MaterialBlueTheme());
    BeepThemesManager.RegisterTheme("MaterialDark", new MaterialDarkTheme());
    BeepThemesManager.RegisterTheme("HighContrast", new HighContrastTheme());

    // Activate one
    BeepThemesManager.ApplyTheme("MaterialBlue");

    Application.Run(new MainForm());
}

Access the active theme from anywhere:

var theme = BeepThemesManager.ActiveTheme;
Color primaryColor = theme.PrimaryColor;
Color successColor = theme.SuccessColor;
Font defaultFont = theme.GetFont(BeepFontKind.Body);

Switching Themes at Runtime

Changing the active theme triggers an automatic repaint cascade across all open forms. Every BaseControl receives an OnThemeChanged notification. 

private void LightModeButton_Click(object sender, EventArgs e)
{
    BeepThemesManager.ApplyTheme("MaterialBlue");
}

private void DarkModeButton_Click(object sender, EventArgs e)
{
    BeepThemesManager.ApplyTheme("MaterialDark");
}
Performance note: Switching themes invalidates every control. Avoid switching in a tight loop or on a timer. Only switch in response to deliberate user action.

Creating a Custom IBeepTheme

Implement IBeepTheme to define your own color palette and style tokens. The interface exposes semantic colors so controls can remain agnostic about the visual look. 

public class OceanTheme : IBeepTheme
{
    public string ThemeName => "Ocean";
    public Color PrimaryColor => Color.FromArgb(0, 105, 148);
    public Color SecondaryColor => Color.FromArgb(0, 151, 178);
    public Color AccentColor => Color.FromArgb(0, 200, 200);
    public Color SuccessColor => Color.FromArgb(46, 139, 87);
    public Color WarningColor => Color.FromArgb(255, 165, 0);
    public Color DangerColor => Color.FromArgb(220, 53, 69);
    public Color BackColor => Color.FromArgb(240, 248, 255);
    public Color ForeColor => Color.FromArgb(25, 25, 35);
    public Color SurfaceColor => Color.White;
    public Color BorderColor => Color.FromArgb(173, 216, 230);
    public Color DisabledBackColor => Color.FromArgb(224, 224, 224);
    public Color DisabledForeColor => Color.Gray;
    public Color FocusBorderColor => Color.FromArgb(0, 120, 215);
    public Color ShadowColor => Color.FromArgb(40, 0, 50, 80);

    public Font GetFont(BeepFontKind kind) => kind switch
    {
        BeepFontKind.Heading => new Font("Segoe UI", 18f, FontStyle.Bold),
        BeepFontKind.Subheading => new Font("Segoe UI", 14f, FontStyle.Bold),
        BeepFontKind.Body => new Font("Segoe UI", 10f),
        BeepFontKind.Caption => new Font("Segoe UI", 8f),
        _ => new Font("Segoe UI", 10f)
    };

    public int BorderRadius => 6;
    public int ControlHeight => 32;
    public Padding ControlPadding => new Padding(8, 4, 8, 4);
}

Register and apply:

BeepThemesManager.RegisterTheme("Ocean", new OceanTheme());
BeepThemesManager.ApplyTheme("Ocean");

BaseControl.ApplyControlStyle

Every BaseControl calls ApplyControlStyle() during initialization and after theme changes. This is where the control reads the active theme and maps tokens to WinForms properties: 

protected override void ApplyControlStyle()
{
    var theme = BeepThemesManager.ActiveTheme ?? BeepThemesManager.DefaultTheme;

    BackColor = theme.BackColor;
    ForeColor = theme.ForeColor;
    Font = theme.GetFont(BeepFontKind.Body);

    // For controls with borders
    _borderPen?.Dispose();
    _borderPen = new Pen(theme.BorderColor, 1f);

    // For controls with backgrounds
    _backBrush?.Dispose();
    _backBrush = new SolidBrush(theme.SurfaceColor);

    Invalidate();
}

This method is the hook for any custom painting logic. Always call base.ApplyControlStyle() first to pick up the standard token mappings, then add your control-specific styles.

FormStyle vs Theme

FormStyle is a per-form property that governs chrome — the title bar, border, min/max/close buttons, and window-level appearance. The theme governs the content area — control colors, fonts, and interior styling.

ScopePer-form (chrome)Application-wide (content)

public class DashboardForm : BeepiFormPro
{
    public DashboardForm()
    {
        FormStyle = FormStyle.Material;           // Chrome: Material-style title bar
        // Content colors come from BeepThemesManager.ActiveTheme
    }
}

Theme Change Notification

If your control caches GDI+ resources (brushes, pens, fonts), you must recreate them when the theme changes. Override OnThemeChanged: 

private SolidBrush _headerBrush;
private Pen _dividerPen;

protected override void OnThemeChanged()
{
    base.OnThemeChanged();

    _headerBrush?.Dispose();
    _dividerPen?.Dispose();

    var theme = BeepThemesManager.ActiveTheme;
    _headerBrush = new SolidBrush(theme.PrimaryColor);
    _dividerPen = new Pen(theme.BorderColor, 1f);

    Invalidate();
}

Access the theme change event directly if you have non-UI components that need to react: 

BeepThemesManager.ThemeChanged += (s, e) =>
{
    // e.OldTheme, e.NewTheme
    logger.Info($"Theme changed to {e.NewTheme.ThemeName}");
    RebuildImageCache();
};

Best Practices

  • Set the theme early — in Program.Main() before any form is created.
  • Never hard-code colors in controls — read from BeepThemesManager.ActiveTheme or override ApplyControlStyle.
  • Cache GDI+ resources and recreate on OnThemeChanged — never create brushes in OnPaint.
  • Use ImagePath instead of Image — the theming system can tint and style images consistently.
  • Test with both light and dark themes — ensure every control is legible in both modes.
  • Provide a high-contrast theme for accessibility compliance.
  • Do not switch themes rapidly — each switch triggers a full repaint cascade.
