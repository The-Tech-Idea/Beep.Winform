## Program Notes

- The plan emphasizes commercial-grade selector behavior, not just feature count.
- Figma-style UX rules are treated as first-class product requirements.
- DevExpress-inspired clarity and state handling are used as a quality reference.
- Phase 1 execution should start with the file-by-file matrix and review notes.
- Phase 1 task breakdown provides the execution order for implementation.
- Phase 2 execution should start from the interaction checklist and matrix.
- Phase 3 execution should start from the visual variant checklist and matrix.
- Phase 4 execution should start from the integration checklist and matrix.
