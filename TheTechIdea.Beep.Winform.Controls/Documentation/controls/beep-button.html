Overview

BeepButton is a modern, themeable button control that extends the standard Windows Forms Button with React-style variants, Material Design principles, and advanced features like SVG icon support, ripple effects, and comprehensive theming capabilities.

Built on the BeepControl foundation, BeepButton provides multiple visual variants, size options, and color schemes that automatically adapt to your application's theme while maintaining consistent behavior and accessibility.

üé® React-Style Variants

Contained, Outlined, Text, and Floating Action Button variants

Basic Usage

Simple Button

// Create a basic button
var button = new BeepButton
{
    Text = "Click Me!",
    Size = new Size(120, 36),
    Location = new Point(10, 10)
};

// Add to form
this.Controls.Add(button);

Styled Button with Variants

// Create a primary contained button
var primaryButton = new BeepButton
{
    Text = "Primary Action",
    UIVariant = ReactUIVariant.Contained,
    UIColor = ReactUIColor.Primary,
    UISize = ReactUISize.Large,
    Size = new Size(150, 48),
    Location = new Point(10, 10)
};

// Create an outlined secondary button
var secondaryButton = new BeepButton
{
    Text = "Secondary Action",
    UIVariant = ReactUIVariant.Outlined,
    UIColor = ReactUIColor.Secondary,
    UISize = ReactUISize.Medium,
    Size = new Size(140, 40),
    Location = new Point(170, 10)
};

this.Controls.AddRange(new Control[] { primaryButton, secondaryButton });
üí° Quick Tip

Use the UIVariant, UIColor, and UISize properties for consistent styling across your application. These properties automatically apply appropriate colors, borders, and sizing based on your active theme.

Event Handling Examples

var button = new BeepButton
{
    Text = "Save Document",
    UIVariant = ReactUIVariant.Contained,
    UIColor = ReactUIColor.Success
};

// Click event handler
button.Click += async (sender, e) => {
    button.IsLoading = true;
    button.Text = "Saving...";
    
    try
    {
        await SaveDocumentAsync();
        ShowSuccessMessage("Document saved successfully!");
    }
    catch (Exception ex)
    {
        ShowErrorMessage($"Failed to save: {ex.Message}");
    }
    finally
    {
        button.IsLoading = false;
        button.Text = "Save Document";
    }
};

// Hover effects
button.MouseEnter += (s, e) => {
    button.UISize = ReactUISize.Large; // Slightly larger on hover
};

button.MouseLeave += (s, e) => {
    button.UISize = ReactUISize.Medium; // Return to normal size
};

Theming

BeepButton integrates seamlessly with the Beep theming system, automatically adapting colors, fonts, and visual effects based on your application's active theme.

Applying Themes

// Apply theme to individual button
button.Theme = "DarkTheme";
button.ApplyTheme();

// Apply theme to all buttons in a container
private void ApplyThemeToButtons(Control container, string themeName)
{
    foreach (Control control in container.Controls)
    {
        if (control is BeepButton button)
        {
            button.Theme = themeName;
            button.ApplyTheme();
        }
        
        // Recursively apply to nested containers
        if (control.HasChildren)
        {
            ApplyThemeToButtons(control, themeName);
        }
    }
}

Best Practices

üé® Visual Design

  • Use Primary for main actions: Reserve primary colored buttons for the most important action on a screen
  • Maintain visual hierarchy: Use Contained > Outlined > Text variants to establish importance
  • Consistent sizing: Use the same UISize within related button groups
  • Meaningful icons: When using SVG icons, ensure they clearly represent the action

‚ö° Performance

  • Batch theme changes: Apply themes to multiple buttons at once using container-level operations
  • Optimize SVG icons: Use simple, optimized SVG paths for better rendering performance
  • Avoid excessive animations: Limit ripple effects and hover animations for better responsiveness

‚ôø Accessibility

  • Provide tooltips: Use ToolTipText for buttons with icons only
  • Keyboard navigation: Ensure buttons are accessible via Tab navigation
  • High contrast support: Test buttons with high contrast themes
  • Focus indicators: Ensure focused buttons have clear visual indicators
‚ö†Ô∏è Avoid Common Pitfalls

Don't use too many primary buttons on one screen - this reduces the visual hierarchy and confuses users about the main action.

‚úÖ Pro Tip

Use the IsLoading property for async operations to provide immediate user feedback and prevent double-clicks.
