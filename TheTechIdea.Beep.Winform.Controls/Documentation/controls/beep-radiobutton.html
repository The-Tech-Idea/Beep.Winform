Overview

BeepRadioButton is an advanced radio button group control that displays multiple mutually exclusive options in a single control. Unlike traditional radio button groups that require multiple individual controls, BeepRadioButton manages all options internally and provides a unified interface for selection handling.

Built on the BeepControl foundation, it supports both horizontal and vertical layouts, image icons for each option, hover effects, and seamless integration with the Beep theming system. The control uses the SimpleItem model for flexible option management.

Key Features

üéØ Single Control Management

All radio button options managed within a single control instance

üìê Dual Orientations

Support for both vertical (stacked) and horizontal (side-by-side) layouts

üñºÔ∏è Image Support

Optional images for each option with SVG support and theme-aware coloring

üé® Interactive Feedback

Hover effects and visual selection indicators for enhanced user experience

üîó SimpleItem Integration

Uses SimpleItem model for flexible option data with text, values, and metadata

üé≠ Theme Integration

Complete integration with Beep theming system for consistent styling

Basic Usage

Simple Radio Button Group

// Create a basic BeepRadioButton
var radioGroup = new BeepRadioButton
{
    Size = new Size(200, 120),
    Location = new Point(10, 10),
    Orientation = RadioButtonOrientation.Vertical
};

// Add options
radioGroup.Options.Add(new SimpleItem { Text = "Option 1", Value = "opt1" });
radioGroup.Options.Add(new SimpleItem { Text = "Option 2", Value = "opt2" });
radioGroup.Options.Add(new SimpleItem { Text = "Option 3", Value = "opt3" });

// Set default selection
radioGroup.SelectedValue = "Option 1";

// Handle selection changes
radioGroup.SelectedValueChanged += (sender, e) => {
    Console.WriteLine($"Selected: {e.SelectedValue} (Index: {e.SelectedIndex})");
};

// Add to form
this.Controls.Add(radioGroup);

Programmatic Selection Management

// Select by value
radioGroup.SelectedValue = "Option 2";

// Get selected item
var selectedItem = radioGroup.GetValue() as SimpleItem;
if (selectedItem != null)
{
    Console.WriteLine($"Selected item: {selectedItem.Text}");
}

// Get selected index
int index = radioGroup.SelectedIndex;
Console.WriteLine($"Selected index: {index}");

// Check if anything is selected
if (radioGroup.SelectedIndex >= 0)
{
    Console.WriteLine("Something is selected");
}

// Clear selection
radioGroup.Reset();
üí° Usage Tip

Use the SimpleItem pattern to separate display text from underlying values, making data binding and business logic much easier to manage.

Layout Orientations

BeepRadioButton supports two layout orientations for different UI scenarios:

üìã Vertical (Stacked)

Options arranged vertically, ideal for forms and long option text

‚û°Ô∏è Horizontal (Side-by-side)

Options arranged horizontally, perfect for toolbars and compact layouts

Vertical Layout (Default)

var verticalRadio = new BeepRadioButton
{
    Orientation = RadioButtonOrientation.Vertical,
    Size = new Size(200, 150) // Height accommodates stacked options
};

verticalRadio.Options.AddRange(new[]
{
    new SimpleItem { Text = "Priority: High", Value = "high" },
    new SimpleItem { Text = "Priority: Medium", Value = "medium" },
    new SimpleItem { Text = "Priority: Low", Value = "low" }
});

// Good for:
// - Forms with limited width
// - Long option text
// - Traditional radio button feel

Horizontal Layout

var horizontalRadio = new BeepRadioButton
{
    Orientation = RadioButtonOrientation.Horizontal,
    Size = new Size(400, 50) // Width accommodates side-by-side options
};

horizontalRadio.Options.AddRange(new[]
{
    new SimpleItem { Text = "Yes", Value = "yes" },
    new SimpleItem { Text = "No", Value = "no" },
    new SimpleItem { Text = "Maybe", Value = "maybe" }
});

// Good for:
// - Short option text
// - Toolbar-style selection
// - Compact layouts
üìè Layout Tip

Choose orientation based on available space and option text length. Vertical works better for longer text, while horizontal is ideal for short options.

Event Handling Examples

var radioGroup = new BeepRadioButton();

// Basic selection handling
radioGroup.SelectedValueChanged += (sender, e) => {
    Console.WriteLine($"Selection changed to: {e.SelectedValue}");
    Console.WriteLine($"Selected index: {e.SelectedIndex}");
    
    // Get the full SimpleItem if needed
    var selectedItem = radioGroup.Options[e.SelectedIndex];
    Console.WriteLine($"Selected item value: {selectedItem.Value}");
};

// Advanced selection handling with business logic
radioGroup.SelectedValueChanged += (sender, e) => {
    switch (e.SelectedValue)
    {
        case "High Priority":
            SetHighPriorityMode();
            break;
        case "Medium Priority":
            SetMediumPriorityMode();
            break;
        case "Low Priority":
            SetLowPriorityMode();
            break;
    }
    
    // Update other controls based on selection
    UpdateRelatedControls(e.SelectedValue);
    
    // Validate form if needed
    ValidateForm();
};

Survey Question Interface

public BeepRadioButton CreateSurveyQuestion(string question, string[] options)
{
    var questionLabel = new BeepLabel
    {
        Text = question,
        Font = new Font("Segoe UI", 12, FontStyle.Bold),
        AutoSize = true
    };

    var radioGroup = new BeepRadioButton
    {
        Size = new Size(400, options.Length * 35),
        Orientation = RadioButtonOrientation.Vertical,
        Location = new Point(questionLabel.Left, questionLabel.Bottom + 10)
    };

    // Add options
    for (int i = 0; i < options.Length; i++)
    {
        radioGroup.Options.Add(new SimpleItem 
        { 
            Text = options[i], 
            Value = $"option_{i + 1}" 
        });
    }

    return radioGroup;
}

Data-Bound Radio Group

public class Priority
{
    public string Name { get; set; }
    public string Code { get; set; }
    public string IconPath { get; set; }
    public int Level { get; set; }
}

public void CreateDataBoundRadioGroup()
{
    // Sample data
    var priorities = new List
    {
        new Priority { Name = "Critical", Code = "CRIT", IconPath = "icons/critical.svg", Level = 1 },
        new Priority { Name = "High", Code = "HIGH", IconPath = "icons/high.svg", Level = 2 },
        new Priority { Name = "Medium", Code = "MED", IconPath = "icons/medium.svg", Level = 3 },
        new Priority { Name = "Low", Code = "LOW", IconPath = "icons/low.svg", Level = 4 }
    };

    var priorityRadio = new BeepRadioButton
    {
        Size = new Size(300, 150),
        Location = new Point(20, 20),
        Orientation = RadioButtonOrientation.Vertical
    };

    // Bind data to radio group
    foreach (var priority in priorities)
    {
        priorityRadio.Options.Add(new SimpleItem
        {
            Text = priority.Name,
            Value = priority.Code,
            ImagePath = priority.IconPath,
            Item = priority // Store reference to full object
        });
    }

    // Handle selection
    priorityRadio.SelectedValueChanged += (s, e) => {
        var selectedOption = priorityRadio.Options[e.SelectedIndex];
        var priority = selectedOption.Item as Priority;
        
        if (priority != null)
        {
            Console.WriteLine($"Selected priority: {priority.Name} (Level: {priority.Level})");
            UpdateTaskPriority(priority);
        }
    };

    this.Controls.Add(priorityRadio);
}

Theming

BeepRadioButton integrates seamlessly with the Beep theming system, automatically applying theme colors to radio indicators, text, and background elements.

Theme Application

var radioGroup = new BeepRadioButton
{
    Theme = "DarkTheme"
};

// Theme automatically applies:
// - PanelBackColor (background)
// - LabelForeColor (text and radio indicators)
// - Theme fonts (if UseThemeFont = true)

// Apply theme manually if needed
radioGroup.ApplyTheme();

Custom Styling

// Override specific colors
radioGroup.BackColor = Color.FromArgb(248, 249, 250);
radioGroup.ForeColor = Color.FromArgb(33, 37, 41);

// Custom font
radioGroup.TextFont = new Font("Roboto", 11f, FontStyle.Regular);
radioGroup.UseThemeFont = false;

Theme-Responsive Features

  • Radio Indicators - Automatically adapt to theme accent colors
  • Text Colors - Use theme foreground colors for optimal contrast
  • Background Colors - Apply theme background colors for consistency
  • SVG Icons - Theme-aware coloring for vector graphics
  • Hover Effects - Theme-appropriate highlight colors
üé® Theming Tip

The control automatically applies theme colors to SVG images when using the ImagePath property, ensuring consistent icon coloring.

Best Practices

üé® User Experience

  • Clear options: Use descriptive, mutually exclusive option text
  • Logical ordering: Order options logically (alphabetical, by frequency, or importance)
  • Default selection: Provide sensible default selections when appropriate
  • Appropriate spacing: Ensure adequate spacing between options for easy selection
  • Consistent orientation: Use the same orientation for similar radio groups

üìê Layout Guidelines

  • Vertical for forms: Use vertical orientation for traditional form layouts
  • Horizontal for toolbars: Use horizontal orientation for toolbar-style selections
  • Size appropriately: Ensure the control size accommodates all options
  • Consider text length: Choose orientation based on option text length

üìä Data Management

  • Use SimpleItem properly: Leverage Text for display and Value for data binding
  • Store object references: Use the Item property for complex data objects
  • Handle selection events: Always respond to SelectedValueChanged events
  • Validate selections: Check for valid selections in forms

‚ôø Accessibility

  • Keyboard navigation: Ensure proper tab order and keyboard accessibility
  • Screen readers: Use meaningful option text for screen reader compatibility
  • High contrast: Ensure sufficient contrast for radio indicators
  • Touch targets: Provide adequate touch target sizes for mobile scenarios
‚ö†Ô∏è Design Warning

Ensure mutual exclusivity of options. Each option should represent a distinct choice that cannot coexist with others in the same group.

‚úÖ Success Pattern

Always handle the SelectedValueChanged event to respond to user selections and maintain application state consistency.
