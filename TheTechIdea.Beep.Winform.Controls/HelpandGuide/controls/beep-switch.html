Overview

BeepSwitch is a modern toggle switch control that provides an elegant alternative to traditional checkboxes. It features a cylindrical (capsule) design with a sliding toggle knob, customizable On/Off labels, background images, and support for both horizontal and vertical orientations.

Built on top of the BeepControl base class, BeepSwitch includes theme integration, data binding capabilities, smooth animations, and gradient backgrounds for a polished, modern appearance suitable for settings panels, preferences dialogs, and interactive dashboards.

Key Features

?? Capsule Design

Modern cylindrical toggle with smooth rounded edges and gradient backgrounds

?? Dual Orientation

Support for both horizontal and vertical switch orientations

??? Custom Labels

Configurable On/Off labels positioned around the switch

??? Background Images

Optional background images for On and Off states with clipping support

?? Data Binding

Full data binding support with automatic property synchronization

?? Theme Integration

Automatic theme color application and gradient support

? Smooth Interaction

Responsive click interaction with visual feedback

?? State Management

Reliable state tracking with change event notifications

Orientations

BeepSwitch supports two orientation modes for different layout requirements:

?? Horizontal Orientation

?? Off label on left
?? Switch in center
?? On label on right
?? Knob slides left/right

Best for: Settings rows, preference lists

?? Vertical Orientation

?? On label on top
?? Switch in center
?? Off label on bottom
?? Knob slides up/down

Best for: Side panels, compact layouts

Basic Usage

Simple Toggle Switch

// Create a basic BeepSwitch
var toggleSwitch = new BeepSwitch
{
    OnLabel = "On",
    OffLabel = "Off",
    Checked = false,
    Size = new Size(120, 50),
    Location = new Point(10, 10)
};

// Handle state changes
toggleSwitch.CheckedChanged += (s, e) => {
    MessageBox.Show($"Switch is now: {(toggleSwitch.Checked ? "On" : "Off")}");
};

// Add to form
this.Controls.Add(toggleSwitch);

Vertical Switch with Custom Labels

Event Handling Examples

var notificationSwitch = new BeepSwitch
{
    OnLabel = "Enabled",
    OffLabel = "Disabled",
    Checked = true
};

// Primary event - state change
notificationSwitch.CheckedChanged += (s, e) => {
    var enabled = notificationSwitch.Checked;
    Console.WriteLine($"Notifications {(enabled ? "enabled" : "disabled")}");
    
    // Update application settings
    Settings.NotificationsEnabled = enabled;
    Settings.Save();
    
    // Update UI state
    UpdateNotificationUI(enabled);
};

// Visual feedback
notificationSwitch.MouseEnter += (s, e) => {
    notificationSwitch.Cursor = Cursors.Hand;
};

notificationSwitch.MouseLeave += (s, e) => {
    notificationSwitch.Cursor = Cursors.Default;
};

Examples

Settings Panel Switches

public void CreateSettingsPanel()
{
    var settingsPanel = new Panel 
    { 
        Dock = DockStyle.Fill, 
        Padding = new Padding(20) 
    };

    var settings = new[]
    {
        new { Label = "Dark Mode", Property = "DarkModeEnabled", Value = true },
        new { Label = "Auto Save", Property = "AutoSaveEnabled", Value = false },
        new { Label = "Sound Effects", Property = "SoundEnabled", Value = true },
        new { Label = "Animations", Property = "AnimationsEnabled", Value = true }
    };

    int yPosition = 20;
    foreach (var setting in settings)
    {
        // Create label
        var label = new Label
        {
            Text = setting.Label,
            Location = new Point(20, yPosition + 15),
            Size = new Size(150, 20),
            TextAlign = ContentAlignment.MiddleLeft
        };

        // Create switch
        var toggle = new BeepSwitch
        {
            OnLabel = "On",
            OffLabel = "Off",
            Checked = setting.Value,
            Location = new Point(180, yPosition),
            Size = new Size(100, 40),
            Tag = setting.Property
        };

        toggle.CheckedChanged += (s, e) => {
            var propertyName = toggle.Tag.ToString();
            var value = toggle.Checked;
            
            // Save setting
            SaveUserSetting(propertyName, value);
            
            // Apply immediately
            ApplySetting(propertyName, value);
        };

        settingsPanel.Controls.AddRange(new Control[] { label, toggle });
        yPosition += 60;
    }

    this.Controls.Add(settingsPanel);
}

Feature Toggle Dashboard

public class FeatureToggleDashboard : UserControl
{
    private Dictionary featureSwitches;

    public FeatureToggleDashboard()
    {
        InitializeFeatureToggles();
    }

    private void InitializeFeatureToggles()
    {
        featureSwitches = new Dictionary();
        
        var features = new[]
        {
            new { Name = "Beta Features", Key = "beta_features", Enabled = false },
            new { Name = "Advanced Analytics", Key = "analytics", Enabled = true },
            new { Name = "Real-time Sync", Key = "realtime_sync", Enabled = false },
            new { Name = "Experimental UI", Key = "experimental_ui", Enabled = false }
        };

        var tableLayout = new TableLayoutPanel
        {
            ColumnCount = 3,
            RowCount = features.Length + 1,
            Dock = DockStyle.Fill,
            CellBorderStyle = TableLayoutPanelCellBorderStyle.Single
        };

        // Add headers
        tableLayout.Controls.Add(new Label { Text = "Feature", Font = new Font("Arial", 10, FontStyle.Bold) }, 0, 0);
        tableLayout.Controls.Add(new Label { Text = "Status", Font = new Font("Arial", 10, FontStyle.Bold) }, 1, 0);
        tableLayout.Controls.Add(new Label { Text = "Toggle", Font = new Font("Arial", 10, FontStyle.Bold) }, 2, 0);

        for (int i = 0; i < features.Length; i++)
        {
            var feature = features[i];
            
            // Feature name
            var nameLabel = new Label
            {
                Text = feature.Name,
                TextAlign = ContentAlignment.MiddleLeft,
                Dock = DockStyle.Fill
            };

            // Status label
            var statusLabel = new Label
            {
                Text = feature.Enabled ? "Enabled" : "Disabled",
                TextAlign = ContentAlignment.MiddleCenter,
                ForeColor = feature.Enabled ? Color.Green : Color.Red,
                Dock = DockStyle.Fill
            };

            // Toggle switch
            var toggle = new BeepSwitch
            {
                OnLabel = "On",
                OffLabel = "Off",
                Checked = feature.Enabled,
                Dock = DockStyle.Fill,
                Tag = feature.Key
            };

            toggle.CheckedChanged += (s, e) => {
                var key = toggle.Tag.ToString();
                var enabled = toggle.Checked;
                
                // Update status label
                statusLabel.Text = enabled ? "Enabled" : "Disabled";
                statusLabel.ForeColor = enabled ? Color.Green : Color.Red;
                
                // Update feature flag
                FeatureFlags.SetEnabled(key, enabled);
                
                // Log change
                Logger.Info($"Feature '{feature.Name}' {(enabled ? "enabled" : "disabled")}");
            };

            featureSwitches[feature.Key] = toggle;

            tableLayout.Controls.Add(nameLabel, 0, i + 1);
            tableLayout.Controls.Add(statusLabel, 1, i + 1);
            tableLayout.Controls.Add(toggle, 2, i + 1);
        }

        Controls.Add(tableLayout);
    }

    public void SetFeatureState(string featureKey, bool enabled)
    {
        if (featureSwitches.TryGetValue(featureKey, out var toggle))
        {
            toggle.Checked = enabled;
        }
    }
}

Data Binding

BeepSwitch supports comprehensive data binding for seamless integration with data models:

Simple Data Binding

public class UserSettings
{
    public bool NotificationsEnabled { get; set; } = true;
    public bool DarkModeEnabled { get; set; } = false;
    public bool AutoSaveEnabled { get; set; } = true;
}

// Create bound switches
var settings = new UserSettings();

var notificationSwitch = new BeepSwitch
{
    OnLabel = "On",
    OffLabel = "Off",
    DataContext = settings,
    DataSourceProperty = nameof(UserSettings.NotificationsEnabled)
};

// Refresh binding to load initial value
notificationSwitch.RefreshBinding();

// Changes to switch automatically update data context
notificationSwitch.CheckedChanged += (s, e) => {
    // Update the data model
    var propInfo = settings.GetType().GetProperty("NotificationsEnabled");
    propInfo?.SetValue(settings, notificationSwitch.Checked);
    
    Console.WriteLine($"Settings updated: Notifications = {settings.NotificationsEnabled}");
};

Filter Creation

// BeepSwitch can create filters for data queries
var activeOnlySwitch = new BeepSwitch
{
    OnLabel = "Active Only",
    OffLabel = "All Items",
    BoundProperty = "IsActive",
    Checked = true
};

// Create filter from switch state
var filter = activeOnlySwitch.ToFilter();
// Result: AppFilter { FieldName = "IsActive", FilterValue = "True", Operator = "=" }

// Use in data queries
var filteredData = dataSource.Where(filter.ToExpression());

Advanced Binding with Validation

public class ConfigurationViewModel : INotifyPropertyChanged
{
    private bool _advancedMode;
    
    public bool AdvancedMode
    {
        get => _advancedMode;
        set
        {
            if (_advancedMode != value)
            {
                _advancedMode = value;
                OnPropertyChanged();
                ValidateConfiguration();
            }
        }
    }

    public event PropertyChangedEventHandler PropertyChanged;
    
    protected virtual void OnPropertyChanged([CallerMemberName] string propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
    
    private void ValidateConfiguration()
    {
        // Validation logic
        if (AdvancedMode && !HasAdvancedLicense())
        {
            MessageBox.Show("Advanced features require a premium license.");
            AdvancedMode = false;
        }
    }
    
    private bool HasAdvancedLicense()
    {
        // License validation logic
        return LicenseManager.CurrentLicense?.Type == LicenseType.Premium;
    }
}

// Bind switch to view model
var viewModel = new ConfigurationViewModel();
var advancedSwitch = new BeepSwitch
{
    OnLabel = "Advanced",
    OffLabel = "Basic",
    DataContext = viewModel,
    DataSourceProperty = nameof(ConfigurationViewModel.AdvancedMode)
};

// Two-way binding with validation
viewModel.PropertyChanged += (s, e) => {
    if (e.PropertyName == nameof(ConfigurationViewModel.AdvancedMode))
    {
        advancedSwitch.Checked = viewModel.AdvancedMode;
    }
};

Theming

BeepSwitch integrates with the theme system for consistent styling:

Automatic Theme Application

var themedSwitch = new BeepSwitch
{
    OnLabel = "Light",
    OffLabel = "Dark",
    Theme = "ModernDark"
};

// Automatically applies:

Theme-Responsive Features

  • Gradient Backgrounds - On/Off states use theme gradient colors
  • Text Colors - Labels inherit theme foreground colors
  • Border Styling - Switch outline adapts to theme border colors
  • Toggle Knob - Knob colors and effects follow theme settings

Custom Theme Colors

Best Practices

?? UI/UX Guidelines

  • Label clarity: Use clear, concise labels that describe the action or state
  • Orientation choice: Use horizontal for lists, vertical for compact sidebars
  • State indication: Ensure On/Off states are visually distinct
  • Feedback: Provide immediate visual feedback when toggled
  • Consistency: Maintain consistent switch styles throughout your application

? Performance

  • Image optimization: Use appropriately sized background images
  • Event handling: Avoid heavy operations in CheckedChanged handlers
  • Batch updates: Group multiple setting changes to avoid excessive events
  • Memory management: Dispose of switches properly when removing from forms
  • Animation efficiency: Let the control handle its own smooth toggle animations

? Accessibility

  • Keyboard support: Ensure switches can be toggled via keyboard (Space key)
  • Screen readers: Provide meaningful labels and state descriptions
  • Color contrast: Maintain sufficient contrast for text and backgrounds
  • Size considerations: Make switches large enough for easy interaction
  • Focus indicators: Ensure clear visual focus indicators for keyboard navigation

?? Technical Considerations

  • Data binding: Use proper data binding for automatic model synchronization
  • State persistence: Save switch states to user preferences or configuration
  • Validation: Implement proper validation for critical toggle actions
  • Error handling: Handle exceptions gracefully in event handlers
  • Thread safety: Ensure UI updates happen on the main thread
?? Critical Operations Warning

Be cautious with switches that trigger irreversible actions. Consider confirmation dialogs or undo mechanisms for critical operations.

? Reliable State Management

BeepSwitch automatically handles state management and visual updates, making it reliable for settings and preferences with proper event notifications.
